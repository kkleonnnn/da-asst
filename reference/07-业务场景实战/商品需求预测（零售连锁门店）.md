## 正文

### 一句话方法论
小店没有大库存，卖多少进多少，因此"备货量"几乎等于"对未来需求的预测"。预测做得好坏，直接决定两类潜在损失：备多了商品过期/挤占货架（货损 + 机会成本），备少了供不应求（错失销售）。所以需求预测的本质是一道**时序预测**题，而它"可简单可复杂"——关键是把三件事讲清楚：**选什么模型、喂什么特征、用什么指标评价**；并且要意识到，**预测对象是一家店还是一个平台，会让上述选择发生质变**。

### 视角一：站在单家小店

问题设定：拿一家店某商品过去 N 周的销量序列，预测下一周（第 N+1 周）的销量。

**最直接的做法（滑动窗 + 时序特征）**
把过去一长段销量序列用长度 N+1 的窗口滑动切分：每个窗口里前 N 个数据作为输入特征（时序特征也叫 Lag 特征，即"滞后特征"），第 N+1 个数据作为要拟合的目标。这样就把"时序预测"转成了一个标准的"用 N 个历史值预测下一个值"的监督学习问题，再用 MAE / MSE / R² 等指标评估拟合偏差，决定是否上线。看似简单，但有三个关键点要拿捏。

**关键点 1：模型怎么选（从简单到复杂的一条谱系）**

- **简单移动平均（SMA / MA）**：取过去 N 周均值当作下周预测值，思路朴素（股票里的 MA5/MA10/MA30 同理）。N 不宜取太大，否则太久远的数据会拖累当前精度；缺点是预测对真实变化会**滞后一拍**，对销量波动频繁的商品不够准。
- **ARIMA**：可看作对 MA 的改进——除了原始的 N 期序列特征，还把前后期的**差分结果**作为新特征一并纳入，再用线性回归拟合第 N+1 期。因为引入了"变化量"信息，多数情况下优于纯 MA，但用好它需要一定统计学基础。
- **Prophet（考虑周期性的专用模型）**：商品销量常有周期性。一条思路是直接选用能兼容多种周期的模型（另一条思路是衍生更多周期特征）。Prophet 把序列加性拆成三块再相加拟合：

  `y(t) = g(t) + s(t) + h(t) + e(t)`

  其中 g(t) 是**趋势项**，s(t) 是**季节性项**（按季节维度构造的子序列），h(t) 是**节假日项**（按假期维度构造的子序列），e(t) 是误差。各子序列用线性等模型拟合后求和，得到最终预测。
- **通用机器学习模型（GBDT / XGBoost / LightGBM）**：时序问题并不一定要用"时序专用模型"。树模型在实际场景中非常主流，非线性拟合能力强，**不仅能预测 N+1，对 N+2、N+3 期也有不错的精度保证**，是很多需求预测的首选。
- **深度模型（LSTM / DeepAR 等）在单店视角下往往不划算**：单店能提供的数据有限，而深度模型参数空间大，当参数量级接近样本量级时**过拟合风险很高**，预测结果常不如树模型且不稳定。结论：单店数据少，需要"经验约束"来收敛，深度模型不是好选择。

**关键点 2：特征怎么造（时序特征之外的世界）**

销量不只由自身历史决定，还和大量外部因素有关：
- **时间特征**：季节交替（如换季时冷饮销量大幅波动）、星期效应（写字楼周边小店工作日饮料销量高于周末）、节日前后（如春节前后乳品销量波动）。
- **天气特征**：雨天饮料、雨具等品类销量变化。
- **热点事件特征**：周边临时举办大型活动会拉动饮料、小食等需求。

把这些特征融入模型有两种典型做法：
- 对 MA / ARIMA / Prophet 这类模型，可把它们的**预测输出当作一个特殊特征**，与时间/天气/事件特征并列，再接一个新的拟合模型预测 N+1。
- 对 GBDT 等机器学习模型则灵活得多，直接把这些特征连同 Lag 特征一起喂进去训练即可。

此外，时序数据本身能**自动衍生**大量特征：除了最大值、最小值、均值这类直观统计量，还能算出最小二乘回归斜率、均方根、傅里叶变换分量等。可借助成熟的时序特征自动提取工具批量生成（Python 有现成库），代价是衍生特征多、运行耗时较长，需权衡。

**关键点 3：评价指标怎么选（由商品成本结构决定）**

回归常用指标有 MAE、MSE、RMSE、MAPE、MSPE、RMSPE、R²。重点理解三者在销量预测中的取舍：
- **MAE（绝对误差）**：对"差了几件"的绝对量敏感。
- **MAPE（相对误差百分比）**：对"差了百分之几"敏感。
- **举例对比**：某高成本、单品销量小的商品预测 2 件、实际 3 件（差 1 件，相对误差 50%）；某饮料预测 99 件、实际 100 件（同样差 1 件，但相对误差仅 1%）。如果小店里多是前一类**单价高、走量小**的商品，用 MAPE 更合适——它会以牺牲少量高销量商品的绝对误差为代价，更关注低销量商品的相对偏差。
- **R²（决定系数）**：当上述两类商品都大量存在时，R² 是一个不错的折衷选择。

> 选指标的核心不是"哪个公式更高级"，而是"哪种误差对这门生意更致命"——备货错 1 件对一件熨斗和一瓶饮料的意义完全不同。

**单店建模的代码骨架（用 GBDT 举例，中性示意）**
```python
import numpy as np
import pandas as pd
from sklearn.ensemble import GradientBoostingRegressor
from sklearn.metrics import mean_squared_error

data = pd.read_csv("data.csv")

# 时序问题切分训练/测试集：不要随机切，应以某个时间点为界，
# 前段做训练、后段做测试（这里用 80/20 比例示意）
train_size = int(len(data) * 0.8)
train, test = data[:train_size], data[train_size:]

# 特征工程：假设 fea1/fea2/fea3 为特征、label 为目标。
# 归一化、标准化、特征泛化等操作（如需要）放在这一步。
feats = ["fea1", "fea2", "fea3"]
X_train, y_train = train[feats].values, train["label"].values

model = GradientBoostingRegressor(
    n_estimators=100, learning_rate=0.1, max_depth=3, random_state=42
)
model.fit(X_train, y_train)

y_pred = model.predict(test[feats].values)

rmse = np.sqrt(mean_squared_error(test["label"], y_pred))
print(f"RMSE: {rmse}")
```
> 注意预测值落地时要取整（如 round），毕竟商品按件卖；即便对走量小的商品牺牲 50% 以内的相对精度，仍会采用取整后的整数预测值。

### 视角二：站在零售平台

问题设定：一个平台在一座城市/一个省/全国拥有成百上千甚至上万家小店（类似连锁便利店平台），要预测各门店未来一段时间各商品的销量。与单店相比有两个本质差异：

- **评价口径变了**：平台的准确率往往是**多门店准确率的综合考量**，而非单店一条曲线的优劣。
- **特征更丰富**：门店数量多，门店之间常存在较强相关性。例如春运高峰前交通枢纽门店销量攀升，可能预示随后其他地区门店的销量变化，因此**多个门店的销量序列可以放在一起联合建模**——既可当作无差别特征并列输入，也可在模型结构上做门店间的前后串联。
- **深度模型开始大有用武之地**：单店受限于样本量与参数空间拉不开差距、过拟合风险高；而平台级样本量巨大，与参数空间量级形成了较好的权衡。此时 RNN / LSTM / DeepAR / WaveNet / N-Beats / Transformer 等模型，往往能融合多维度特征、得到更优的预测效果。

### 一句话总结
同一道"零售需求预测"题，**数据量级决定模型上限**：单店数据稀，老老实实用 MA / ARIMA / Prophet / 树模型，深度模型容易过拟合；平台数据厚，深度时序模型与门店间相关性才真正变成资产。无论哪种视角，模型、特征、评价指标三件事都要围绕"这门生意里哪类备货错误最贵"来定。

### 常见坑与边界
- **不要用随机切分做时序训练/测试集**：必须按时间先后切，否则会用"未来"预测"过去"，回测虚高。
- **MA/Prophet 类模型对突变滞后**：销量波动剧烈或受突发事件主导时，纯时序模型反应慢，需补充事件特征或人工干预。
- **单店盲目上深度模型 = 过拟合**：数据量没到规模就上 LSTM/Transformer，结果往往既不准又不稳定。
- **指标别只看一个**：高单价小走量与低单价大走量混杂时，单看 MAE 或单看 MAPE 都会偏，必要时用 R² 折衷或分品类分别评估。
- **自动特征衍生要权衡成本**：批量衍生时序特征虽强，但特征爆炸会显著拉长训练时间，需筛选。
