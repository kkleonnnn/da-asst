## 正文

### 一句话方法论
内因（产品体验、引导、性能等）是流失分析的重点，因为它可调可控；但用户流失也常受外部环境影响——政策、经济、竞品、市场——这些大多不可控，却必须看懂。这套框架借五个经典管理/营销模型，给"外因"提供一组互补的观察视角：先用 SWOT 框定大盘，再用 PEST 看宏观、波特五力和 4P 看竞争、用户体验五要素回到产品对标，最后落到优化建议。

### 五个模型各管什么
- **SWOT（总览定位）**：从优势、劣势、机会、威胁四象限认识产品。前两者是内部，后两者是外部。在流失外因分析里，重点是"威胁"这一象限——通常表现为某种外部不利趋势削弱了产品竞争力。SWOT 的作用是先判断流失的外部矛头大致指向哪里。
- **PEST（宏观环境）**：从政治、经济、社会、科技四个层面扫宏观因素。政治如目标地区的政策法规变动；经济如人均可支配收入、整体景气；社会如人群结构变化；科技如新技术、新平台对老产品的替代。
- **波特五力（竞争格局）**：把行业竞争简化为五个力——同业现有竞争、新进入者威胁、替代品威胁、买方议价力、供方议价力。流失外因里最常见的是"同业竞争"和"替代品"在抢人。
- **4P（竞品对标·营销侧）**：从产品、价格、促销、渠道四要素对比竞品。用户流失往往是竞品在某个 P 上更有吸引力——更低价、更猛促销、更优渠道供给。
- **用户体验五要素（竞品对标·产品侧）**：从战略层、范围层、结构层、框架层、表现层五层拆解产品，与竞品逐层对比，找出体验差距并转成优化项。表现层是 UI 观感，框架层是交互操作，结构层是信息架构与功能流程，范围层是核心功能/满足什么需求，战略层是定位与商业模式。

### 五个模型怎么串起来用
不要把五个模型并列硬套，它们有先后与分工：
1. 先用 **SWOT** 圈出外部威胁的大方向；
2. 用 **PEST** 排查宏观面（政策/经济/社会/科技）是否有冲击；
3. 若矛头指向竞品，用 **波特五力 + 4P** 从竞争格局和营销要素拆竞品如何抢走了用户；
4. 最后用 **用户体验五要素** 把竞品和自家产品逐层对标，把"外部差距"翻译成"内部能改的优化点"，让分析有落脚处。

### 中性示例
某出海工具类 App 留存整体下滑，内因排查后产品体验无明显劣化。转外因：SWOT 锁定"外部威胁"。PEST—政治层面发现目标市场出台了限制类政策，给了本地竞品涌入的窗口。波特五力/4P—本地竞品以更低价格和更激进的促销快速起量。用户体验五要素对标—竞品在表现层（本地化 UI）和框架层（更顺手的交互）更贴合当地习惯。结论：外部政策叠加竞品在价格与本地化体验上的优势导致流失，优化建议落在本地化体验与价格策略上。

### 常见坑 / 边界
- **外因多不可控，别强行"解决"**。这套框架的产出常是"理解为什么流失"，而非"立刻能改的动作"；真正能落地的改动多来自最后一步对标出的产品优化点。
- **与数据结合度低**。这些是定性视角，不要指望它们直接给出指标贡献度；它们的价值是补全内因看不到的盲区。
- **别面面俱到地填满每个格子**。模型是提示思考维度，不是问卷，挑与当下流失相关的几格深挖即可。
- **顺序按矛头走**。如果一眼能看出不是竞品而是宏观冲击，就别在波特五力上耗时间。
