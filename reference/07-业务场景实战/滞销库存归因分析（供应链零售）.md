# 滞销库存归因分析（供应链/零售）

## 正文

### 一句话方法论
滞销库存不是单点问题，而是"需求不确定 → 信息沿链路逐级放大（牛鞭效应）→ 采购/物流/销售各环节偏差叠加"的系统性结果；要先用一套库存监控指标把"有没有积压、积压多严重"量化出来，再沿采购、物流、售后三个环节逐层归因，最后才谈处置。

### 为什么会积压：牛鞭效应
供应链理想状态是"零库存、不缺货、零浪费"，但现实中信息在传递时会失真并被层层放大：越靠近上游，对终端需求的感知越模糊，每一级为了不缺货都会"合理地"加单备货，结果整条链路集体超额采购，需求一旦回落就大面积积压。

典型放大链条（中性造例）：
- 决策层定下年增长目标（如同比 +30%）；
- 销售管理层据上年数据评估，认为至少要在上年基础上加备 50%；
- 一线销售为规避缺货责任、又听到上游原料涨价/供应商承诺返利的消息，进一步上报更高的备货量；
- 供应商收到多个下游的放大订单后，为防违约同样扩大生产。

每一级单独看都"理性"，叠加起来就是整条链的过量库存。要破解，本质是信息共享与协同（建立共享库存/需求数据的链路视图），是管理与人性的问题，而非单纯靠更精的公式。

### 第一步：先把库存口径厘清（库存是动态量，不是静态查询值）
常见误区是把库存当成"仓库现有数量"这一静态值。实际库存随出入库实时变化，入库台账只反映即时状态，还要把已采购但在途、以及已接订单但尚未发出的部分一起算进来。可用库存应做动态加减：

```
可用库存数量 = 即时库存数量 − 待付（已接单待发）数量 + 在途数量
```

> 造数示例：某商品即时库存 100、待付 20、在途 30 → 可用 110；另一商品即时 200、待付 50、在途 40 → 可用 190。做库存分析时按业务场景选取合适口径，不要只看货架数。

### 第二步：库存监控指标体系
对每个商品/每个入库批次，建立以下核心指标用于监控与预警：

| 指标 | 定义 | 计算 |
|---|---|---|
| 可售天数 | 当前库存还能撑多少天销售 | 当前库存 ÷ 最近日均销量 |
| 库龄 | 该批货自入库至今在仓时间 | 当前日期 − 入库日期 |
| 库存周转天数 | 库存平均周转一次需多少天 | 日均存货金额 ÷ 日均销货成本 |
| 库存周转次数 | 一年周转几次 | 全年存货周转次数 =（日均存货金额×360）÷ 产品年销售成本金额；周转天数 = 360 ÷ 周转次数 |

口径要点：
- **可售天数**的"日均销量"通常取最近一周到半月的均值；遇大促等异常销售期要剔除，避免高估去化速度。
- **库龄**衡量"压了多久"，**可售天数**衡量"还能卖多久"，两者一起看才完整。

判读规则（造例）：
- 商品 A、B 可售天数（如 60 天、35 天）均 > 各自周转天数（如 30 天），说明储备充足、暂不需补货，但需防积压风险，库存量更大的更要盯紧。
- 若某商品平均周转 30 天、而某批次库龄已显著高于此（如 108 天），就是已经出现积压的信号 → 列为优先优化对象，暂停其采购，直到现有滞销被有效处理；库龄在正常范围内（如 23 天）的批次则可不动。

### 第三步：沿环节归因——库存为何积压
库存滞销的根在"销售不畅"或"采购/生产维持了超过原计划的供给"。从供应链运营角度，可沿三个环节逐层下钻分析。

#### ① 采购环节（拆成采购前 / 中 / 后）
- **采购前**：依据历史销售与需求预测制定采购计划。
  `采购批量 = 平均日需求量 × 采购周期 + 安全库存量 − 实际库存量`
  公式只是骨架，具体要按行业场景定义安全库存等参数；同时这一步最易受牛鞭效应影响（采购方不知供应商真实供给、销售计划又不透明），需靠信息共享缓解。
- **采购中**：跟踪采购订单执行，重点是交期。延迟交付可能拖停生产/销售（尤其季节性、促销窗口期）；但交期过早会提前占用资金并产生库存。应设预警指标，监控订单完成率、协商交期 vs 实收交期、异常批次，及时介入。
- **采购后**：按商品/供应商/区域/仓库多维度复盘采购效果，指导下期。
  `订单完成率 = 入库数量 ÷ 订单量`；`计划达成率 = 入库数量 ÷ 计划入库数量`
  > 造例：某商品计划 79、实际入库 27、订单量 53 → 完成率 51%、达成率 34%（供给严重不足）；另一商品计划 58、入库 74、订单量 168 → 完成率 44%、达成率 128%（采购超计划但仍未满足订单，口径需结合下单率一起看）。
- **供应商评估**：结合历史采购订单数、价格水平、交货质量、交期时效、信誉等多指标打分，将供应商分级（如高质量供应商 / 交货准时 / 价格竞争力强等标签，综合为 A 级 / B 级），用画像指导采购决策、择优合作或激励改进。

#### ② 物流环节
物流交付直接关联库存动态平衡，导致积压的常见原因：
- **运输延误**：天气、交通拥堵、运输工具故障等意外，使货卡在起点或中转，积压在前段。
- **配送效率低**：配送线路规划不当、配送中心管理或人员不足，导致货到不了消费者手中，增加库压。
- **包装/装卸问题**：操作不当致运输途中损坏或丢失，引发退货、返修，拖累周转。

监控指标（造例字段）：已交付订单数、逾期交付订单率、待交付订单率、逾期待交付率、破损退货率等。
若逾期交付率偏高，再往下钻收货-仓储-拣选-发货-退货全链路：是物流端发货慢，还是仓储拣货后压货、在途时间长、距离/车辆/成本问题？是否存在货到客户却迟迟未签收？逐层定位优化空间。

#### ③ 售后环节
- 最常见是退换货：退换流程不明确、退换处理不及时；可维修产品维修周期过长（尤其配件短缺时）；售后响应慢、服务质量差，都会引发客户不满、招致更多退换货，反向增加库压。
- **退换货分析**：收集退换货数量、退货原因分类占比、退货产品类型、退货时间分布等，对原因做详细分类统计；进一步分析退货客户的评价，了解对商品的不满点。
- 退换货属于逆向物流，要单独监控运输环节、设计运输路线、规划退货仓的地域分布，同样靠数据精细化运营。

### 第四步：积压库存的处置策略
诊断出积压后，结合销售数据制定专项营销去化：
1. 促销活动、折扣销售，加速去化周转；或与其他畅销品捆绑销售带货。
2. 重新审视产品线，对积压品做重构、赋予新用途或新功能，提升市场竞争力。
3. 拓宽渠道（如上多个电商平台），把库存推向更广市场。

### 常见坑 / 边界
- **不要把库存当静态数**：忽略在途与待发，会严重误判可用库存。
- **可售天数的分母要去噪**：用含大促的销量算日均，会低估积压风险。
- **单指标会误导**：订单完成率、达成率、下单率必须组合看，单看任一项都可能得出相反结论。
- **公式只是骨架**：采购批量、安全库存等参数必须按行业实际标定，照搬通用公式算不出准确采购量。
- **根因常在跨部门信息断层**：牛鞭效应本质是信息不共享 + 人性避责，技术指标只能暴露问题，根治要靠链路协同机制。
- 本框架是"抛砖引玉"式的通用思路，落地需结合具体业务场景做更深拆解。
