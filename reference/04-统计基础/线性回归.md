## 正文

### 一句话方法论
线性回归用一条线性方程去刻画自变量 X（单个或多个 X1…Xp）对因变量 Y 的平均影响；用最小二乘法（OLS）把这条线"拟合"到数据上，靠各自变量前的系数衡量"这个指标对 Y 的影响是正是负、有多大"，再用显著性检验和拟合优度指标判断这条线到底靠不靠谱。它本质上度量的是**相关关系**，而不是因果。

### 1. 回归到底在回答什么
回归分析研究的是变量之间的"依赖关系"——能否用一个或多个自变量去预测因变量的取值。最常见的就是线性回归，它假设 Y 与 X 大体呈直线关系。

在实际工作中，很多这类问题在画一张散点图后就能用肉眼大致看出来有没有关系；统计学的价值在于把这种关系**量化并描述其强弱**，而不是停留在"看起来相关"。

**关键提醒：相关不等于因果。** 即使回归显示"活动参与率高的用户付费转化也高"，也只能说明两者相关，不能推出"因为参与活动所以付费高"。要谈因果，需要更严谨的因果推断设计，单靠回归系数得不出来。

### 2. 模型形式：从简单到多元

**简单线性回归**（单自变量）：

```
Y = β0 + β1·X + ε
```

- **Y**：因变量（被解释、被预测的变量）
- **X**：自变量（解释变量）
- **β0**：截距项，X=0 时 Y 的基准值
- **β1**：回归系数，X 每变动一个单位，Y 平均变动多少
- **ε**：误差项，模型未能解释的随机部分

**多元线性回归**（多个自变量）把一个因变量 Y 同时跟多个自变量 X1, X2, …, Xp 关联起来：

$$Y = \beta_0 + \beta_1 X_1 + \beta_2 X_2 + \cdots + \beta_p X_p + \varepsilon$$

- β0 是截距；
- β1…βp 是各自变量的回归系数，符号代表影响方向（正系数表示正向影响，负系数表示负向影响），绝对值大小代表影响强弱；
- ε 是误差项，通常假设服从正态分布 N(0, σ²)。

举个业务化的例子：把某天的订单量当作 Y，把当天的活跃用户数 X1、可用车辆数 X2、降雨量 X3 当作自变量，回归出来的 β1、β2、β3 就分别告诉你这三个指标各自和订单量的相关方向与强度。建模的核心任务，就是从样本数据里**估计出这些系数**。

### 3. 最小二乘法（OLS）怎么估出系数
**思路**：找一组系数，使预测值与真实值之间的总误差最小。把每个点的残差（实际值 − 预测值）平方后求和，得到残差平方和 RSS：

```
RSS = Σ ( Yi − (β0 + β1·Xi) )²    （i 从 1 到 n）
```

之所以取平方而不是直接相加，是为了避免正负残差互相抵消。OLS 的目标就是让 RSS 最小。

**求解**：把 RSS 分别对 β0 和 β1 求偏导，令偏导为 0（极值点导数为零），得到两个方程联立求解，得出解析解：

```
β1 = Σ (Xi − X̄)(Yi − Ȳ) / Σ (Xi − X̄)²
β0 = Ȳ − β1·X̄
```

直观例子：若 X 序列为 (1, 2, 3, 4)，均值 X̄ = 2.5，那么第一个点的离差 (Xi − X̄) = 1 − 2.5 = −1.5，按公式逐项代入即可手算出系数。理解这一步有助于把握 β1 本质上是"X、Y 协同变动的程度 ÷ X 自身的离散程度"。多元情形同理，只是用矩阵形式一次解出所有系数。

**实务做法**：手算只为理解原理，真实工作里用 Python（`statsmodels` 的 `OLS` 或 `sklearn` 的 `LinearRegression`）或 Excel 一键求解。

#### Python 最小示例（用造数演示，非真实业务数据）
```python
import numpy as np
import statsmodels.api as sm
from sklearn.linear_model import LinearRegression
import matplotlib.pyplot as plt

# 造一组带噪声的线性数据
np.random.seed(0)
X = np.random.rand(100, 1) * 10           # 自变量
Y = 2.5 * X + np.random.randn(100, 1) * 2 # 因变量 = 2.5X + 噪声

# 方式一：statsmodels，能直接打印完整回归报告
X_const = sm.add_constant(X)              # 添加截距项
results = sm.OLS(Y, X_const).fit()
print(results.summary())

# 方式二：sklearn，拿系数更直接
lr = LinearRegression().fit(X, Y)
print("截距:", lr.intercept_, " 回归系数:", lr.coef_)

# 可视化拟合直线
plt.scatter(X, Y, label="Data")
plt.plot(X, lr.predict(X), color="red", label="Regression Line")
plt.legend(); plt.show()
```

**Excel 做法**：通过"数据分析"工具库做回归（若菜单里没有，到 选项 → 加载项 → 选择"分析工具库"添加）；也可以在散点图上加趋势线，设置趋势线类型、显示拟合公式和 R²，这本身就是一种回归结果。

拿到系数后，理论上即可用新的 X 预测新的 Y——但这一步要等下面的检验通过才靠谱。

### 4. 参数怎么估的另一面：OLS 与 MLE 的区别

多元线性回归（和简单线性回归一样）默认用**最小二乘法（OLS）**估参数：它最小化预测值与实际观测值之间的残差平方和，把损失函数对每个参数求导、令导数为零，能解出**解析解**，相对简单。

另一套常见思路是**极大似然估计（MLE）**：先假设数据来自某个概率分布，构造在给定参数下观测数据出现的似然函数，再最大化它（同样对参数求导令其为零）。MLE 适用面更广，能处理二项分布、泊松分布等多种模型，但通常没有解析解，要靠数值方法迭代求解，更复杂。时间序列里的 ARIMA 用的就是 MLE。

两者的关系：当误差项服从正态分布时，OLS 可以看作 MLE 的一个特例。这也解释了为什么同是"回归"，模型的适用条件（要满足哪些假设、什么情况下能用）会有差异——差异根源就在参数估计的数学逻辑不同，逻辑本身建立在特定条件成立的前提上。

### 5. 别急着用：五项基本假设与显著性检验
能解出系数不代表它们"是对的"。OLS 系数本身不保证有统计显著性，必须做假设检验；而检验之前，先要清楚 OLS 成立依赖哪些前提。

#### 5.1 OLS 成立的五项基本假设及其诊断方法
这五项假设是为了让模型对变量关系的刻画尽量"准"。满足这些假设时，OLS 给出的是最优线性无偏估计（BLUE：Best Linear Unbiased Estimator），即无偏且方差最小。需要提醒的是：检验假设不应只当成建模前的例行公事、然后直奔结果；很多时候验证假设的过程本身就在帮你把变量关系看得更清楚。

**1) 线性关系**
因变量与自变量之间应是线性关系。若实际关系非线性，系数会有偏差、刻画失真。
- 散点图：画 Y 与每个 X 的散点，看是否有线性趋势；
- 残差图：画残差是否随机分布，若呈现明显规律则线性假设可疑。

**2) 误差项独立（无自相关）**
假设各观测的误差项相互独立。若误差之间存在相关（往往意味着还有重要的影响因素没被纳入），系数估计会不准，假设检验结论可能失效。
- Durbin-Watson 检验：检测序列自相关，统计量接近 2 表示独立，远小于或远大于 2 表示存在自相关；
- 残差随时间图：时间序列数据里看残差是否有趋势或周期。

**3) 同方差（方差齐性）**
假设误差项的方差在所有观测上恒定，不随 X 变化而放大或缩小。若存在异方差（例如因变量与自变量量纲不同，随自变量增大误差分布也变化），标准误会被低估或高估，进而影响系数显著性检验的可靠性。
- 残差图：看残差在预测值整个范围内扩散是否大致一致，若呈喇叭形扩散则提示异方差；
- Breusch-Pagan 检验：专门检测异方差。

**4) 误差项正态性**
假设误差项（残差）近似服从正态分布，这对 t 检验、F 检验的准确性很关键。若误差非正态，置信区间和 p 值的解释可能失效。
- QQ 图：画残差的分位数图，点贴近 45 度线说明接近正态；
- Shapiro-Wilk 检验：检验残差是否服从正态分布。

**5) 无多重共线性**
多个自变量之间应相对独立、不高度相关。若自变量高度相关，回归系数会变得不稳定、模型难以解释。
- 方差膨胀因子（VIF）：值越高说明该自变量与其他自变量相关性越强，经验上 VIF > 10 视为严重多重共线性；
- 相关矩阵：直接看自变量两两相关，找出高度相关的变量。

**现实提醒**：真实商业数据几乎不可能完美满足上述每一条。一旦存在异方差、多重共线性等问题，OLS 估计可能偏差或方差过大，影响系数的准确性和稳定性。因此实务中要配合假设检验和模型诊断来发现并修正问题，而不是盲目相信跑出来的系数。

#### 5.2 系数与模型是否显著
**t 检验（单个系数）**：检验某个回归系数是否显著不为零，即该自变量对因变量是否真有影响。
- 原假设 H0：βi = 0（无显著影响）
- 备择假设 H1：βi ≠ 0（有显著影响）
- 统计量 `t = βi / SE(βi)`，其中 SE 是系数估计值的标准误。
- 由 t 统计量算出 p 值，若 p < 显著性水平（常用 0.05），则拒绝原假设，认为该系数显著。

**F 检验（整体模型）**：检验所有自变量系数是否同时为零（若全为零，模型整体无效）。
- 原假设 H0：β1 = β2 = … = βp = 0
- 备择假设 H1：至少有一个系数不为零
- 统计量 `F = (SSR/p) / (SSE/(n − p − 1))`，其中 SSR 为回归平方和（被解释的变异），SSE 为误差平方和（未被解释的变异），n 为样本量，p 为自变量个数。

### 6. 拟合优度：R² 与调整后 R²
**R²（决定系数）** 衡量模型对因变量变化的解释能力，取值 0~1：

```
R² = 1 − SSE / TSS    （TSS 为总平方和）
```

当 SSE 趋近 0（拟合几乎无误差）时 R² 趋近 1，拟合越好。

**R² 的陷阱**：每多加一个自变量，模型总能找到更贴近数据的拟合、让 SSE 下降，于是 R² 会被"人为抬高"，哪怕新变量对 Y 几乎没有实际作用。变量越堆越多时，R² 就会失真。

**调整后 R²** 在解释能力之外，对模型复杂度（自变量个数）施加惩罚：

```
R²_adj = 1 − (1 − R²)(n − 1) / (n − p − 1)
```

直观理解：自变量个数 p 越大，分母 (n − p − 1) 越小，惩罚项越重，调整后 R² 相对原始 R² 被压低。因此评估模型时通常看调整后 R²，而不是裸 R²。

#### 推导脉络（便于记忆）
把 SSE 和 TSS 分别按自由度标准化（SSE 用 n−p−1，TSS 用 n−1），代回 R² 定义式，即可得到上面的调整后 R² 公式。

### 7. 把假设违背当成线索来读

检验假设的真正价值，往往不在"通过 / 不通过"，而在违背时它在提示什么：

- **发现异方差**，常意味着因变量和自变量量纲差异大；对相关变量做对数处理后，它们之间的正负相关关系会刻画得更准。
- **发现多重共线性**，意味着自变量之间彼此有关联。比如把"用户数"和"车辆数"同时放进模型，若"车均活跃用户"近似恒定，那用户数多自然车辆数也多，两者高度相关，此时可以去掉其中一个。

把能验证的都验证、能修正的都修正之后，模型才更可能真实反映变量间关系，这时再去做后续的相关性解读或预估才比较稳妥。

### 8. 完整模型诊断：清单与可复用脚本（中性造数）

跑完回归后应做一套诊断，逐项核对前面的假设是否被违反。下面先给一份"逐项核对"的最小清单脚本，再给一个把"建模 + 五项假设检验 + 可视化 + 修改建议"打包起来的函数。

#### 8.1 逐项诊断清单（最小示例）
```python
import numpy as np, pandas as pd
import statsmodels.api as sm
from sklearn.datasets import make_regression
from statsmodels.stats.outliers_influence import variance_inflation_factor
import matplotlib.pyplot as plt
from scipy import stats

# 造 3 个自变量的数据
np.random.seed(42)
X, Y = make_regression(n_samples=100, n_features=3, noise=10)
data = pd.DataFrame(X, columns=['X1', 'X2', 'X3']); data['Y'] = Y

# 拟合
X_const = sm.add_constant(data[['X1', 'X2', 'X3']])
results = sm.OLS(data['Y'], X_const).fit()
print(results.summary())

# 残差正态性：Q-Q 图
sm.qqplot(results.resid, line='45'); plt.title('Q-Q plot for residuals'); plt.show()

# 线性与同方差：残差 vs 拟合值散点
plt.scatter(results.fittedvalues, results.resid)
plt.axhline(0, color='red', linestyle='--')
plt.xlabel('Fitted values'); plt.ylabel('Residuals'); plt.show()

# 多重共线性：VIF
vif = pd.DataFrame()
vif['Feature'] = X_const.columns
vif['VIF'] = [variance_inflation_factor(X_const.values, i) for i in range(X_const.shape[1])]
print(vif)

# 误差自相关：Durbin-Watson
print("Durbin-Watson:", sm.stats.stattools.durbin_watson(results.resid))

# 残差正态性：Shapiro-Wilk
stat, p = stats.shapiro(results.resid)
print("Shapiro-Wilk p 值:", p)  # p>0.05 接近正态，否则拒绝正态假设

# 模型整体显著性：F 检验 p 值
print("F 检验 p 值:", results.f_pvalue)  # <0.05 模型整体显著

# 调整后 R²
print("调整后 R²:", results.rsquared_adj)
```

**诊断要点对应关系**：
- Q-Q 图 / Shapiro-Wilk → 残差正态性
- 残差 vs 拟合值散点 → 线性与同方差
- VIF → 多重共线性（一般 VIF 偏大提示共线性严重）
- Durbin-Watson → 误差自相关
- F 检验 p 值 → 模型整体是否显著
- 调整后 R² → 惩罚复杂度后的拟合优度

#### 8.2 打包成可复用函数（含修改建议）
```python
import pandas as pd
import numpy as np
import statsmodels.api as sm
import seaborn as sns
import matplotlib.pyplot as plt
from scipy import stats
from statsmodels.stats.outliers_influence import variance_inflation_factor
from statsmodels.stats.diagnostic import het_breuschpagan


def multi_regression_analysis(X, Y):
    # 添加常数项（截距）
    X = sm.add_constant(X)

    # 建立并拟合 OLS 回归模型
    model = sm.OLS(Y, X).fit()
    print(model.summary())

    # 多重共线性：VIF 检验
    print("\n---- 方差膨胀因子 (VIF) ----")
    vif_data = pd.DataFrame()
    vif_data['feature'] = X.columns
    vif_data['VIF'] = [variance_inflation_factor(X.values, i)
                       for i in range(X.shape[1])]
    print(vif_data)

    # 提取残差与拟合值
    residuals = model.resid
    fitted_values = model.fittedvalues

    # 可视化验证：残差图 / QQ 图 / 残差分布
    plt.figure(figsize=(12, 10))

    plt.subplot(2, 2, 1)
    sns.scatterplot(x=fitted_values, y=residuals)
    plt.axhline(0, color='red', linestyle='--')
    plt.title('残差图')
    plt.xlabel('预测值')
    plt.ylabel('残差')

    plt.subplot(2, 2, 2)
    sm.qqplot(residuals, line='45', fit=True, ax=plt.gca())
    plt.title('QQ 图')

    plt.subplot(2, 2, 3)
    sns.histplot(residuals, kde=True)
    plt.title('残差分布图')
    plt.xlabel('残差')

    # 自相关：Durbin-Watson
    dw_stat = sm.stats.durbin_watson(residuals)
    print(f"\n--- Durbin-Watson 统计量 ---\nDurbin-Watson 值：{dw_stat:.2f}")

    # 异方差：Breusch-Pagan
    print("\n--- Breusch-Pagan 异方差检验 ---")
    bp_test = het_breuschpagan(residuals, X)
    labels = ['Lagrange Multiplier', 'p-value', 'f-value', 'f p-value']
    bp_results = dict(zip(labels, bp_test))
    for k, v in bp_results.items():
        print(f"{k}: {v:.4f}")

    plt.tight_layout()
    plt.show()

    # 根据检验结果给出修改建议
    print("\n--- 修改建议 ---")
    if vif_data['VIF'].max() > 10:
        print("存在多重共线性，考虑移除高 VIF 变量或使用正则化方法（岭回归、Lasso）。")
    if dw_stat < 1.5 or dw_stat > 2.5:
        print("存在误差自相关，考虑使用时间序列模型（如 ARIMA）或加入自回归项。")
    if bp_results['p-value'] < 0.05:
        print("存在异方差，考虑对数据做变换（如对数变换）或使用稳健标准误。")
    shapiro_test = stats.shapiro(residuals)
    if shapiro_test.pvalue < 0.05:
        print("残差不服从正态分布，考虑数据变换或使用非参数方法。")
    else:
        print("模型假设检验无明显问题。")


# 生成示例数据（中性造数）
np.random.seed(42)
X = pd.DataFrame({
    'X1': np.random.rand(100),
    'X2': np.random.rand(100),
    'X3': 2 * np.random.rand(100) + 0.5 * np.random.rand(100),  # 与 X1/X2 略相关
})
Y = 3 + 1.5 * X['X1'] + 2.2 * X['X2'] + 1.7 * X['X3'] + np.random.randn(100) * 0.5

# 运行分析
multi_regression_analysis(X, Y)
```

### 常见坑与边界
- **把相关当因果**：再漂亮的相关系数、线性关系，在数学上也只是相关。回归系数显著只说明统计相关，不能直接当因果证据；即便加上某个变量后业务结果真的变好，也不能据此断定是因果，更不应直接拿回归结论去指导真实业务决策。
- **预测通常不准**：多元线性回归更擅长"判断与量化指标间的相关关系、描述关系深浅"，而不是给出可靠的点预测；把它当描述工具比当预测工具更稳妥。
- **只看 R² 不看调整后 R²**：堆变量会虚高 R²，应以调整后 R² 评估。
- **跳过假设检验直接用系数**：系数能解出来不等于可靠，需 t/F 检验。
- **忽视假设违反 / 别为了建模而建模**：真实数据常违反同方差、独立、无共线性等前提，必须做残差诊断与 VIF 等检查后再下结论；假设检验不是走过场，违背假设时它提供的诊断信息（量纲、共线、遗漏变量等）往往比模型本身更有价值。
- **超范围外推**：用远超样本 X 取值范围的新 X 去预测，外推风险高。
