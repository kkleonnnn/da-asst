## 正文

### 一句话方法论
描述性统计量是用来「刻画」一份数据长什么样的数值——它从集中趋势、离散程度、分布形状三个角度概括数据。业务里几乎所有常用指标，本质上都是某个统计量套在某个样本上的具体化；把这层关系想通，指标体系、SQL 聚合函数、指标下钻就都串起来了。

### 为什么先学统计量
分析的第一步是拿到一版「合适」的数据。理解抽样、样本，是为了科学地选出真正回答问题所需的数据；选错样本或选错统计量，结论就会跟着错。所以在面对任何一版数据时，先问两件事：**样本选得对不对**、**用哪个统计量来描述它**。一句值得记住的原则是：观点背后的数据口径，往往比观点本身更值得推敲。

### 用一个生活化的例子理解
判断「我的工资算高还是低」，可以层层逼近，这个逼近过程本身就是数据分析的缩影：

1. **先看平均数**：大家平均一万，我两万，对比平均水平我偏高。
2. **意识到平均数怕极值**：一个人月入百万、一个人零收入，平均下来五十万，看着不低却失真。于是换用**分位数**——50 分位（中位数）能说明「有一半人不超过这个值」，比平均数更抗极端值。
3. **缩小样本再比**：把样本限定到「同城 + 同工作年限 + 同岗位 + 同量级公司」后，再算这群人的中位数，结论可能从「超过 90% 的人」变成「只超过 10% 的同类人」。

关键启示：**调整样本**和**调整统计量**，会得到完全不同的结论。分析的本质，就是在不断校准样本与统计量的过程中，逼近真正想回答的命题。

### 常见统计量速查

| 统计量 | 含义 | 算法（中文公式） | 对应的业务场景示例 |
|---|---|---|---|
| 平均数（均值） | 数据的中心位置 | 所有观测值之和 ÷ 观测值个数 | 日均订单量＝总订单量 ÷ 天数 |
| 中位数 | 数据的中间点，抗极值 | 升序排列后取最中间的值 | 每日活跃用户数的中位数 |
| 众数 | 出现频率最高的值 | 统计各取值出现次数，取最多的 | 最常被观看的视频类别 |
| 方差 | 数据的离散（波动）程度 | 每个值与均值之差的平方，再取平均 | 每日学习时长的波动 |
| 标准差 | 离散程度，与原数据同量纲 | 方差的平方根 | 客单价（如打车费用）的标准差 |
| 四分位数 | 反映分布，把数据四等分 | 升序后取第 25%、50%、75% 位置的值 | 用户消费金额的分布 |

要点：均值、中位数、众数描述**集中趋势**；方差、标准差描述**离散程度**；分位数/四分位数描述**分布形状**。标准差比方差实用，因为它和原始数据是同一个单位。

### 统计量与 SQL 聚合函数是一一对应的
理解统计量后再看 SQL 聚合函数会轻松很多——聚合函数就是统计量在数据库里的具象化：

```sql
-- 订单总数（计数）
SELECT COUNT(*) FROM orders;
-- 订单金额总和
SELECT SUM(order_amount) FROM orders;
-- 平均值
SELECT AVG(order_amount) FROM orders;
-- 最大 / 最小
SELECT MAX(order_amount) FROM orders;
SELECT MIN(order_amount) FROM orders;
-- 方差 / 标准差
SELECT VARIANCE(order_amount) FROM orders;
SELECT STDDEV(order_amount) FROM orders;
-- 按订单拼接客户姓名（GROUP_CONCAT 为 MySQL 方言）
SELECT order_id, GROUP_CONCAT(customer_name) FROM orders GROUP BY order_id;
```

跑完聚合，一版数据里就能得到一堆统计量：货品总量、每日平均销量、总销量、单品均价……它们都是统计量的不同表现。

### 业务指标 = 统计量套在某个样本上
工作中的常见指标，本质都是「对某段时间/某个样本做某种聚合」：

| 指标 | 口径 | 统计量本质 |
|---|---|---|
| 总订单数 | 某时间段内全部订单数量 | 对订单 ID 计数 |
| 活跃用户数 | 某时间段内使用产品的独立用户数 | 对去重用户 ID 计数 |
| 新增用户数 | 某时间段新注册用户数 | 对新注册用户 ID 计数 |
| 留存用户数 | 某时间段再次使用产品的用户数 | 对回访用户 ID 计数 |
| 人均使用时长 | 每个用户平均使用时长 | 使用时长总和 ÷ 独立用户数 |
| 人均消费金额 | 每个用户平均消费 | 消费金额总和 ÷ 独立用户数 |
| 转化率 | 完成某行为的用户占比 | 完成行为用户数 ÷ 总用户数 |

### 指标如何串成一个分析故事
单个指标只是名词，分析是把它们串成因果链。例如一个完整的拆解叙事可能是：

> 总订单数在下降 → 背后是活跃用户数减少 → 但人均付费金额没变（说明不是某类用户单独流失，而是各结构用户都在走）→ 这些用户流失前的人均使用时长在持续走低 → 即他们在逐渐失去使用意愿。

指标是把图表、看板、结论串起来的线索。基于这样的结论，再不断衍生新指标去解释、去衡量活动与策略效果，最后沉淀成监控看板。

### 从均值过渡到分布：指标下钻
当发现某个均值类指标（如人均时长）下降时，一个自然动作是**拆开分布看**：前 20% 用户的人均时长和人数占比是否也降了？后 20% 呢？这一步就是从「均值」过渡到「分位数/分布」，指标也随之下钻了一层。结论会从「平均使用时长下降」细化为「头部用户使用时长减少是主因」，从而更聚焦。

由此可以更原子化地理解两件事：
- **RFM 模型**本质上是三类统计量（最近一次消费、消费频次、消费金额）的展现，其「高/中/低」分层依据正是对统计量做分位划分。
- **分组统计**（按维度切分样本、各组分别给统计量来描绘分布）类似分层抽样，是完成指标下钻的标准手法，也直接关联到「指标异动分析」这类经典问题——不过真正解决异动还需更多专门方法。

### 常见坑
- **只看均值不看分布**：均值易被极值带偏，遇到右偏/长尾数据应优先看中位数和分位数。
- **样本与统计量不匹配**：先确认样本口径（时间范围、是否去重、人群限定），再选统计量，否则结论可能整体错位。
- **方差/标准差量纲混淆**：方差单位是原数据的平方，不便直接解读；对外沟通通常用标准差。
