## 正文

### 一句话方法论

正式分析前先「逐字段」地把数据看一遍：**最小分析粒度是字段**，了解数据就要从了解每一个字段开始。判断一个字段该怎么看，先看它属于哪种**测量尺度**；看的方法分两类——离散字段看枚举值的频数频率，连续字段看统计特征和分布（必要时分箱）。这是一件「磨刀不误砍柴工」的事。

注意：只用单一字段做的是**描述**，还算不上「分析」（分析要有比较）；但描述是分析的地基，跳过它，后面挖价值会既不全面也不高效。

### 四种测量尺度：决定一个字段能做什么运算

判断字段类型，本质是判断它被哪种尺度衡量，这直接决定了它能做哪些运算、适合哪些方法。

| 尺度 | 特点 | 能做的运算 | 例子 |
|---|---|---|---|
| 定类（名义） | 取值是有限的几个类别，**彼此无大小** | 只能数频数、判相等 | 性别、部门、商品品类、颜色 |
| 定序（顺序） | 取值有限且**有大小之分**，但**说不清差距多大** | 可排序，不能算间隔 | 满意度（很满意/一般/不满意）、等级（优/良/中/差） |
| 定距（间隔） | 可比大小、可算差值，但**没有绝对零点**，不能比倍数 | 可加减，不可乘除 | 摄氏温度（20℃比10℃高10℃，但不是「两倍热」） |
| 定比（比例） | 在定距基础上**有绝对零点**，倍数有意义 | 加减乘除都可 | 长度、重量、时长、金额、计数 |

经验对应：**定类、定序常衡量离散值；定距、定比常衡量连续值**。两点提醒：① 当定序的「序」失去业务意义，它就退化成定类；② 同一字段的尺度可能随业务解读而变，不要只看数据形态、要结合业务判断。

### 看离散字段：枚举 + 频数频率

离散字段（定类 / 定序）取值集中在一个不大的有限集合里，所以思路很简单：**列出所有取值，给出每个值的频数和频率**，基本就掌握了它的全部信息。对定序字段，按值的大小排列展示会更直观。

> 中性示例：某字段「会员等级」取值 {普通, 银卡, 金卡}，频数 6200 / 2900 / 900，频率 62% / 29% / 9%。一眼就能看出结构是否健康、有没有异常占比。

### 看连续字段：统计特征 + 分布

连续字段（定距 / 定比）取值近乎无穷，不能逐个看，要靠**统计特征**来刻画。常用的有：

- **集中趋势**：平均值、中位数、众数。最常用的是平均值（如「人均在线时长」）；除算术平均外，还有调和、几何、平方平均等变体，按场景选。
- **离散程度 / 形态**：标准差、偏度、峰度等（用于看分布是否对称、有没有长尾）。

关键提醒：算这些统计量时一定要**结合业务事实**思考，否则只是得到一堆看着唬人却脱离业务的数字，达不到「认识数据」的目的。

看连续字段分布有两条常见思路：
1. **分布拟合**：把字段值和常见分布（正态、卡方、t、F 等）做拟合，若标准化后的均值、标准差、偏度、峰度与某分布高度一致，就可以用这个分布及其参数来代表该字段。
2. **分箱**：把连续值离散化成一个个区间（见下节）。

### 分箱：把连续值切成区间

分箱就是把连续字段转成有限的区间（多数情况下变成定序）。它既能帮你认识数据形态，也能消除噪声、防止字段过于灵敏影响模型稳定性，还是特征工程里常用的离散化手段。分箱有三类思路：

- **直接分箱**：不参考任何额外信息。
  - *等深分箱*：每个箱里的样本数量相同（深度 = 样本数）。
  - *等宽分箱*：每个箱覆盖的区间跨度相同（宽度 = 区间跨度）。
  - 例：对 {4,7,10,13,14,18,23,30,32} 分 3 箱。等深 → [4,7,10] / [13,14,18] / [23,30,32]（各 3 个）；等宽 → [4~13] / [14~23] / [24~32]（每段跨度 10，箱内数量可不等）。用直方图看「深 / 宽」最直观。
- **最优分箱**：借助数据本身或一列「导向属性（标注）」来切。
  - 按数据聚集程度切（聚类思想）：把天然挨得近的值归到一箱。
  - 按标注切：每切一刀都让各箱里**标注的分布差异尽可能大**（如风控里按「是否逾期」切，让各箱的逾期 / 未逾期比例差别最大）。衡量差异常用熵（切前算一次熵，在相邻值间试切、取切后加权熵下降最多的点），以及 WOE、IV 等指标。
- **业务分箱**：直接用成熟的业务认知定切点。如 0~100 分按每 10 分一档、年龄按少年 / 青年 / 壮年 / 老年切。

> 三类的取舍，本质是 **Data Talk（让数据指标表现最优）vs Business Talk（让结论更容易被业务认可）**的权衡：偏算法的人倾向前者，偏业务的人倾向后者，最终还是看具体场景。

### 顺手做的事：识别异常值与数据清洗

逐字段看的过程中会自然暴露脏数据。异常值（重复、错误、无效、缺失、干扰值）大多要结合业务才谈得上「异不异常」。两个常见切入面：

- **业务面**：重复 ID（数据重传导致，注意有些表 ID 本就允许重复）、违反一致性约束的错误 / 无效值（如性别栏出现「男/女」以外的值、0~1 评分出现负数或大于 1）。
- **数据面**：干扰性离群值。常用 **IQR 法**：排序后取下四分位 Q1、上四分位 Q3，IQR = Q3−Q1；以 Q3 + k·IQR 为上界、Q1 − k·IQR 为下界（k 一般取 1.5~3），界外即视为离群干扰值。

处理脏数据没有万能解，常见策略：**丢弃**（脏数据少时，或类似比赛去掉最高最低分防干扰）、**指代**（用最大 / 最小 / 边界 / 均值 / 众数或插值填补）。但也别一刀切：有些「异常」本身携带信息（如某传感器在极端天气下缺失值变多，缺失数量反而是天气信号），这时把它转成模型可用的格式即可，不必当噪声删掉。

### 常见坑 / 边界

- **忽略尺度直接算**：对定类字段求平均、对定距字段谈倍数，都是没意义的运算。
- **统计量脱离业务**：算出一堆漂亮数字却不结合业务解读，等于没认识数据。
- **盲目套等宽 / 等深**：分箱方式要服务目的——认识形态、风控区分、还是迎合业务认知，目的不同选法不同。
- **把单字段描述当分析**：逐字段是地基，真正的「分析」要在此之上引入比较与建模。
