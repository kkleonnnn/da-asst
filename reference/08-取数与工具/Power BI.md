## 正文

### 一、PowerBI 是什么：把 Excel 的四个能力打包

PowerBI 可以理解为四个能力组件的集成体，这四块能力在 Excel 中也以加载项形式存在，PowerBI 相当于把它们整合到一个工具里，覆盖"取数 → 清洗 → 建模 → 可视化 → 发布共享"的完整链路：

| 组件 | 职责 | 一句话定位 |
|---|---|---|
| Power Query | 数据连接与清洗 | 把多源、脏乱的数据整成规整表 |
| Power Pivot | 数据建模 | 建表间关系、写计算、处理大数据集 |
| Power Map | 三维地理可视化 | 在地图上画热力 / 空间分布 |
| Power View | 交互式图表 | 出交互图形与视觉效果 |

在 Office 2016 及以上版本中，Power Map / Power Pivot / Power View 已内置，但需从「文件 → 选项 → 加载项 → COM 加载项」里手动勾选启用；低版本需单独安装 Power Query 插件。

**与 Excel 的核心差异**（决定何时该升级到 BI 工具）：
- **图表类型**：BI 工具原生支持卡片图、漏斗图、仪表图、KPI 图等 Excel 没有的类型，且有可下载的扩展视觉对象库；Excel 图表类型相对单一，要靠插件补足。
- **数据处理量**：BI 工具靠 M 语言（Query）和 DAX（Pivot）处理，能撑大数据集并做多表建模；Excel 处理大表容易卡顿、无响应。
- **交互与联动**：BI 工具的图表天生联动——点选一个视图会过滤其他视图；Excel 实现同样效果要手动加复选框 / 滑块，操作繁琐。
- **自定义能力**：DAX 函数比 Excel 公式适用面更广，能写复杂的上下文相关计算。

**与 Tableau 的取舍**：两者都是主流自助式 BI。PowerBI 与 Office 生态结合紧密、上手与 Excel 相近、对微软系数据源友好；Tableau 在可视化表达的灵活度上历来有口碑。选型主要看团队既有技术栈与数据源，不存在绝对优劣（原文偏向 PowerBI，属作者立场，此处中性处理）。

> 学习重心提示：BI 的价值不在炫酷外观，而在数据建模与计算口径。可视化往往只是"把写好的度量值拖进图表"。学习精力建议向清洗、建模倾斜，可视化少花时间。

### 二、Power Query：清洗的常用动作

进入 Query 编辑器后，一条数据准备链路通常是：

1. **获取数据**：本地文件（工作簿可汇总同簿不同 Sheet，文件夹可合并同目录多个工作簿）、CSV / 文本、数据库（SQL Server / Oracle / MySQL / PostgreSQL 等，可写 SQL 语句直接取）、Web（自 Web 抓网页表格）。
2. **类型转换**：选中字段改数据类型（小数 / 整数 / 货币 / 日期 / 文本 / 布尔等）。OCR 来的网页数值常是文本，需显式转整数 / 小数。
3. **第一行用作标题**：网页 / 抓取数据常缺表头，一键把首行提升为列名。
4. **拆分列**：按分隔符（如 `-`）或字符数拆分。典型场景：薪资区间 `15-28k` 拆成最低 / 最高两列，取最低值更贴近实际起薪；地区 `上海-浦东新区` 拆成省 / 区。
5. **提取**：从文本里取"分隔符之间 / 之前 / 之后"的内容。如从 `1.48万亿(1,483,497,784,867)` 用括号作起止分隔符提出纯数字串，再转整数。
6. **删除行 / 列 / 重复项 / 错误 / 空值**：去多余列、按字段去重（如用户 ID 标唯一）、转换报错的行（如"面议"无法转数值）直接删错误。
7. **逆透视（Unpivot）**：把宽表多列收成长表两列（维度名 + 值）。如把"金 / 银 / 铜"三列奖牌数收成"奖牌品类 + 数量"两列，便于后续按品类聚合或画图。
8. **分组依据**：按维度聚合（如按省份对销售额求和），相当于 GROUP BY。
9. **标准数值运算**：整列同除一个量级（如 GDP 除以 1 亿）方便阅读。
10. **关闭并应用 / 上载**：清洗结果回写到数据模型或表格。

**自定义筛选顺序的技巧**：切片器默认按拼音/字母排序。若想让"春夏秋冬"按自然顺序显示，可另建一张顺序表（季节 + 顺序号），与主表建多对一关系，再对季节字段设置"按列排序 → 顺序列"。

### 三、Power Pivot / DAX：建模与度量值

Power Pivot 相比普通透视表的关键增量：能**建表间关系**、能**做非重复计数**、能写 **DAX** 复杂计算。

- **建关系**：在两表共有主键（如"省份"）上创建关系，常见为一对多。建好关系后即可跨表分析。
- **非重复计数**：普通透视表没有，Pivot 才有。如统计每年"不重复的订单编号数"，避免一单多行被重复计。

**几个高频 DAX 度量值模板**（口径要点写在后面）：

```
销售金额   = SUM('表'[销售额])
累计销售额 = CALCULATE([销售金额], FILTER(ALLSELECTED('表'), SUM('表'[销售额]) <= [销售金额]))
总销售额   = CALCULATE([销售金额], ALLSELECTED('表'))
累计占比   = DIVIDE([累计销售额], [总销售额])
```

- `CALCULATE`：在被修改的筛选上下文里求值，是 DAX 的核心。
- `ALLSELECTED`：取查询中所有行列的上下文，但保留行列筛选之外的显式筛选——常用来算"占当前视图全集的比例"。
- `DIVIDE`：安全除法，分母为 0 时返回备用值 / 空，避免报错。
- 累计 / 同期类还可用时间智能函数（如 `TOTALYTD`）按时间维度累计。

新建度量值与新建列的区别：度量值是按当前上下文动态聚合（放进图表才算）；新建列是逐行落地的物理列。文中 RFM 的逐行 R 值用"新建列"，而 F=`DISTINCTCOUNT(...)`、M=`SUM(...)` 用"度量值"。

### 四、可视化选型：常用视觉对象与适用场景

BI 工具自带 + 可下载的视觉对象很多。选图的本质是"数据结构 + 想表达的关系"对上图形：

| 想表达 | 适用图形 | 字段映射要点 |
|---|---|---|
| 目标 vs 实际差值 | KPI Indicator / 子弹图(Bullet) / 仪表(Gauge) | 实际值→值，目标→目标值；可设阈值色带（红=未达标） |
| 二八 / 累计贡献 | 帕累托图（柱 + 累计折线） | 类目→轴，金额→柱，累计占比 + 0.8 分界线→次轴 |
| 排名随时间动态变化 | 动态条形竞赛(Bar Chart Race) | 类目→Name，值→Value，时间→Period，可设 TopN |
| 分类频率 / 文本权重 | 词云(WordCloud) | 类目→类别，数量→值 |
| 多维度对比一个对象 | 雷达图(Radar) | 维度→类别，数值→Y 轴 |
| 双向分组对比 | 蝴蝶 / 旋风图(Tornado) | 分组→组，对比维（如性别）→图例，数量→值 |
| 二维定位分类 | 象限图(Quadrant) | 两个度量分别给 X / Y 轴，自定义四象限标签 |
| 分布形态 | 直方图(Histogram) / 箱线图(Box-Whisker) | 连续值→值；箱线可关掉均值点显示 |
| 气泡大小对比 | 气泡图(Bubble) | 名称→Bubble Name，度量→Value |
| 地理分布（点 / 形状） | 地图 / 形状地图(Shape Map) | 地名→位置，度量→大小 / 色彩饱和度 |
| 明细表 | 表格 / 矩阵 | 拖字段即可，矩阵可行列交叉 |

**形状地图（自定义行政区）做法**：默认形状是美国地图，要画国内行政区需：① 从地图可视化平台框选区域、复制 JSON API 链接，下载为 `.json`；② 用 mapshaper 把 GeoJSON 转 TopoJSON 导出；③ PowerBI「选项 → 预览功能」勾选"形状映射视觉对象"并重启；④ 在地图设置里把映射类型改为"自定义地图"，导入该 json，位置 / 图例绑地名，度量绑色彩饱和度。

**导入第三方视觉对象**：可视化面板「… → 从文件导入视觉对象」，选 `.pbiviz` 文件。第三方对象非微软提供，仅在信任来源时导入。

### 五、两个内嵌的方法示例（工具承载的分析套路）

#### 1）RFM 客户分层（用 DAX 落地）
RFM 用三个指标刻画客户价值：R(Recency 最近一次交易间隔)、F(Frequency 交易频次)、M(Monetary 交易金额)。

- **打分规则**：每个维度按均值劈成高低两档。注意 R 是"间隔天数"，越小越好，所以 **R≤均值记 2、>均值记 1**；F、M 则 **≥均值记 2、<均值记 1**。
- **组合**：三位组成 8 类（如 222=重要价值会员，111=流失会员），用 `SWITCH` 映射成会员标签。
- **落地步骤**：先算每个用户的最后交易日（`MAXX(FILTER(... EARLIER ...))`），R=固定基准日 − 最后交易日；F=`DISTINCTCOUNT(订单ID)`；M=`SUM(支付额)`；再 `SUMMARIZE` 成"用户 + R/F/M"新表，`AVERAGE` 算各维均值，`IF` 打分，连接符拼出 RFM 码，最后建模连回主表做分组分布、占比、业绩贡献可视化。
- **结论用法**：重要价值会员通常人数与业绩贡献双高，资源应倾斜（VIP / 高折扣）；流失会员占比若大，优先低成本触达挽回。

#### 2）二八 / 帕累托分析
二八法则（帕累托法则）：约 20% 的因素贡献约 80% 的结果，分析应聚焦这关键少数。落地要点：

- 按销售额降序，画"柱（金额）+ 累计占比折线 + 80% 分界线"的组合图，交点左侧即贡献八成收入的关键品类。
- 用 DAX 求"前 20% 款式的销售额占比"：`款式数量 = CALCULATE(DISTINCTCOUNT([货号]), [销售额]>0)`；`20%款式数量 = ROUND([款式数量]*0.2,0)`；`前20%销售额 = SUMX(TOPN([20%款式数量], ALL([货号]), [销售金额], DESC), [销售金额])`；再 `DIVIDE` 出占比。
- 价值：既能识别该重点经营的少数，也能反向思考如何把其余 80% 往这 20% 转化。

### 六、常见坑 / 边界
- **OCR / 抓取数值是文本**：拆分、提取后务必显式转数值类型，否则聚合、画图会异常；转换中无法解析的值（如"面议""null"）要删错误行而非留空。
- **去重要选对唯一键**：按用户 ID 去重前先确认该字段确实唯一，否则会误删多行。
- **R 的方向易错**：RFM 里 R 是"越近越好"，打分方向与 F/M 相反，写 `IF` 时别套同一个模板。
- **累计 / 占比类度量看上下文**：`ALLSELECTED` 与 `ALL` 行为不同，前者尊重外部切片器、后者无视；占比口径错常源于此。
- **切片器排序**：天然按字母/拼音，业务自然序（季节、等级）需借顺序表 + 按列排序。
- **工具不替代口径思考**：图表只是表达层，结论是否成立取决于建模口径与数据质量，别被"拖拽即出图"的便利掩盖了口径校准。
