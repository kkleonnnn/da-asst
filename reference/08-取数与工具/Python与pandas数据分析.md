### 一、选择并搭建编辑环境

常见的几类 Python 编写/运行环境，各有定位：

| 环境 | 定位 | 适合谁 |
|---|---|---|

### 三、字符串：分析里最常处理的脏数据

字符串支持索引与切片，下标从 0 开始，切片区间"含左不含右"；负号表示从右往左数。

```python
a = 'hello world'
type(a)        # str
len(a)         # 长度

a[0:5]         # 'hello'   取前 5 个字符
a[::-1]        # 'dlrow olleh'  整体反转
```

清洗中高频的字符串方法（按用途分组）：

- **查找计数**：`count('o')` 出现次数、`index('o')` / `find('o')` 定位位置、`startswith('h')` 是否以某字符开头。
- **大小写规整**：`upper()` 全大写、`lower()` 全小写、`capitalize()` 首字母大写、`title()` 每词首字母大写。
- **拼接与对齐**：`'-'.join(a)` 用分隔符把字符逐个连起来、`zfill(15)` 左侧补零到指定宽度、`replace('world','boy')` 替换子串。
- **去空白/去边缘字符并拆分**——清洗日期、编号时最常用：

```python
s = '**2023/08/05**'
s.strip('*')           # '2023/08/05'   去两端的 *
s.lstrip('*')          # 只去左边
s.rstrip('*')          # 只去右边
s.strip('*').split('/')# ['2023', '08', '05']  去边缘后按 / 拆成列表
```

字符串还支持 `+`（拼接）和 `*`（重复）运算。

### 四、数值与类型转换

整数与浮点数都支持基本算术：`+ - *`、真除 `/`、地板除 `//`（向下取整）、幂 `**`、取余 `%`。括号优先级最高。

```python
num = 10
num // 2      # 5    地板除
num ** 2      # 100  平方
num % 2       # 0    取余
```

自增/自乘的简写：`num += 1` 等价 `num = num + 1`；`num *= 5` 等价 `num = num * 5`。

**类型转换是清洗的核心动作**：从外部读进来的数字常常是字符串，直接相加会变成"文本拼接"而非"数值求和"，必须先转换。

```python
num1, num2, num3 = '10', '20', '30.0'
print('字符串相连:', num1 + num2 + num3)   # '102030.0'  这是拼接，不是加法！

# 转成数值再相加
num1_int = int(num1)
num2_int = int(num2)
num3_int = int(float(num3))   # 含小数点的字符串要先 float 再 int
print('数值相加:', num1_int + num2_int + num3_int)  # 60
```

> 常见坑：浮点运算会有微小精度误差（如某些幂/取余结果出现一长串 9）；这是二进制浮点表示的固有现象，做金额等精确计算时要留意。

### 五、三大数据结构：列表、元组、字典

这是组织数据的基础容器，区别在"可不可变"和"怎么取值"。

#### 列表 `list`——可变、有序

元素可增删改，按位置（下标）访问，可装任意类型。

```python
lst = [1,2,3,4,5,6,7,8,9]
lst[0]      # 第一个
lst[-1]     # 最后一个
lst[0:4]    # 切片，规则同字符串
lst[::-1]   # 反转
lst[::2]    # 步长 2，隔一个取一个
```

常用方法：

| 方法 | 作用 |
|---|---|
| `append(x)` | 末尾追加一个对象 |
| `extend([x])` | 把另一个列表的元素逐个合并进来 |
| `insert(i, x)` | 在指定位置插入 |
| `remove(x)` | 删除第一个值为 x 的元素 |
| `copy()` | 复制一份 |
| `count(x)` / `index(x)` | 计数 / 找位置 |
| `sort()` | 原地升序排序 |

列表也支持 `+`（合并）和 `*`（重复）。

#### 元组 `tuple`——不可变、有序

可理解为"锁定的列表"，初始化后元素不能改，只能查询。索引、切片、`count`、`index` 用法都和列表一致。**适合存放不该被意外修改的固定数据。**

```python
t = (0, 1, 2, 3, 4)
t[0]        # 0
t[::-1]     # (4, 3, 2, 1, 0)
```

#### 字典 `dict`——键值对，按键取值

以 `键:值` 形式存储。**键必须唯一且不可变（用字符串、数字、元组等），值可重复、可改、可为任意类型。**

```python
d = {'a':10, 'b':20, 'c':30}
d['b']            # 20      按键取值
d.keys()          # 所有键
d.values()        # 所有值
d.items()         # 所有键值对
d.update({'d':40})# 新增/更新
d.pop('a')        # 删除某键
d.get('b')        # 取值，键不存在时不报错（更安全）
```

> 选型直觉：要频繁增删改、顺序重要 → 列表；要防误改的固定序列 → 元组；要靠"名字"快速查值 → 字典。

### 六、三大控制流：条件、while、for

#### 条件 `if / elif / else`

靠**缩进**界定代码块，条件成立才执行对应分支。

```python
score = 75
if score < 60:
    print('不及格')
elif score < 70:
    print('成绩: {}'.format('及格'))
elif score < 80:
    print('成绩: {}'.format('中等'))
elif score < 90:
    print('成绩: {}'.format('良好'))
else:
    print('成绩: {}'.format('优秀'))
# 输出：成绩: 中等
```

#### `while`——满足条件就反复执行

```python
a, i = 0, 1
while i <= 100:
    a += i
    i += 1
print(a)   # 5050  累加 1 到 100
```

`while ... else` 的 `else` 只在循环"自然跑完"（不是被 `break` 中断）时执行。

#### `for`——遍历集合或可迭代对象

`range(起, 止, 步长)` 生成一段整数序列，常配 `for` 用；同样遵循"含起不含止"。

```python
total = 0
for i in range(1, 101):
    total += i
print(total)   # 5050
```

#### 循环控制语句

- `break`：立即结束整个循环；
- `continue`：跳过本次剩余部分，进入下一轮；
- `pass`：什么都不做，仅占位（语法上需要一句但暂时没逻辑时用）；
- `while True` + `break`：实现"先无限循环、满足条件再主动跳出"的模式。

#### 用 pandasql 在 DataFrame 上写 SQL（实用示例）

当你已经把数据读进了一个名为 `df` 的 DataFrame，想按某个数值字段做分箱标签，可以直接写 SQL 的 `CASE WHEN`，无需记 pandas 的多重条件赋值写法：

```python
import pandasql as sql

result = sql.sqldf("""
    select 名称, 维度A, 维度B, 数值,
    case
        when 数值 < 100000 then '档位1'
        when 数值 < 200000 then '档位2'
        when 数值 < 300000 then '档位3'
        when 数值 < 400000 then '档位4'
        else '档位5'
    end as 分组标签
    from df
    where 数值 is not null
""")
```

它的价值在于：把"按阈值分箱并打标签"这种分析里极高频的操作，用人人都熟的 SQL 表达出来，降低从 SQL 转 pandas 的学习成本。

## 第二部分：pandas 实操主线（从清洗到可视化与入库）

### 八、读入与数据体检
拿到一张表，先别急着算，先看清它长什么样。

- **读取**：`pd.read_excel(路径)` 读 Excel，`pd.read_csv(路径)` 读 CSV。读进来得到一个 DataFrame。
- **预览前几行**：`df.head()` 默认看前 5 行，确认列名和数据是否符合预期。
- **看字段结构**：`df.info()` 列出每列的非空计数、数据类型（日期是否被当成日期型、数值是否被当成字符串等），是发现「类型错位」的第一道关口。
- **描述统计**：`df.describe()` 一次性给出各数值列的计数、均值、标准差、最小/最大值与四分位数。例如能快速看出某客群年龄从 22 到 62 岁、均值约 42 岁。

> 体检要点：`info()` 看类型与缺失，`describe()` 看分布与异常极值，两个一起用基本能定位绝大多数数据质量问题。

### 九、排序、筛选与拆分

**排序**——`df.sort_values(by='列名')`
- 默认升序；加 `ascending=False` 改降序。
- 多列自定义方向：`by=['类别列','金额列'], ascending=[True, False]`，即先按类别升序、组内再按金额降序。
- 想原地改写传 `inplace=True`。

**筛选（布尔索引）**——把条件写进 `df[...]`
- 单条件：`df[df['性别']=='男']`。
- 多条件「与」用 `&`，「或」用 `|`，每个条件务必各自加括号：`df[(df['性别']=='男') & (df['年龄']>60)]`。
- 命中一组离散值用 `isin`：`df[df['订单号'].isin(['A001','A002','A003'])]`，比写一长串 `|` 清爽。

**列拆分**——`str.split`
- 形如「华北-河北省-保定」想拆成省、市，可 `df['区域'].str.split(pat='-', expand=True)`，`expand=True` 让结果展开成多列 DataFrame，再用 `.iloc[:,0]`、`.iloc[:,1]` 取各段赋给新列。

### 十、统计运算与分组聚合
- **计数**：`df['某列'].count()` 数非空行数。
- **去重计数**：`df['类别'].unique()` 返回不重复取值数组，`len(df['类别'].unique())` 即类别个数。
- **求和**：`df['销量'].sum()`。
- **分组聚合**：`df.groupby('类别')['订单号'].count().reset_index()` 得到每个类别的订单数；`reset_index()` 把分组键还原成普通列，方便后续出图或导出。把 `count` 换成 `sum`/`mean` 即可做分组求和、求均值。

> 套路：「先 groupby 再 agg」是 pandas 里最常用的分析动作，几乎所有「按某维度汇总指标」的需求都落到这一句。

### 十一、可视化常用图型
共性前置设置（解决中文乱码与负号显示）：
```
plt.rcParams['font.sans-serif'] = ['SimHei']     # 正常显示中文
plt.rcParams['axes.unicode_minus'] = False       # 正常显示负号
```

- **饼图（占比）**：把分组汇总结果 `.plot(kind='pie')`，常用参数 `autopct='%.1f%%'` 标百分比、`explode` 把某块突出、`startangle` 调起始角度。适合看各分类的结构占比，如各区域销量占比。
- **箱线图（分布与离群）**：`plt.boxplot(x=数据)`，`whis` 控制上下须的倍数（默认 1.5 倍四分位距），`showmeans=True` 标均值。用来看某指标的分布形态和离群点——大量点落在箱须之外说明该指标长尾、波动大（如利润）。
- **折线图（趋势）**：用 seaborn `sns.lineplot(data=df, x='日期', y='指标')` 看指标随时间的波动趋势，seaborn 会自动带置信带。
- **词云（高频项）**：用 `wordcloud` 库统计分类频次后 `generate_from_frequencies(频次字典)`，出现越多字越大，可直观看出哪类占比高/低；中文需指定中文字体路径（如 `font_path='simhei.ttf'`）。

### 十二、一键生成探索性分析（EDA）报告
做初步 EDA 时不必手写一堆代码，可用自动化分析库三行出报告：
```
from ydata_profiling import ProfileReport   # 旧名 pandas_profiling，已更名
profile = ProfileReport(df, title='分析报告')
profile.to_file('report.html')
```
生成的 HTML 报告通常含五块：
1. **数据集概览**：行数、列数、变量类型、内存占用。
2. **各变量分布**：每个变量的取值分布、分位数、描述统计——信息量最大的一块。
3. **相关性可视化**：Spearman / Pearson 等相关矩阵热力图、变量两两交互散点。
4. **缺失值可视化**：各字段缺失情况；某字段缺失比例大（如超 30%）就要考虑是否弃用。
5. **样本与重复行**：预览若干行，并标出重复行。

> 定位：EDA 报告是「快速摸底」工具，适合拿到陌生数据时一眼看全貌；正式结论仍需针对性深挖，别把报告直接当分析成果。

### 十三、表格样式美化与导出
pandas 的 `df.style` 链式调用，能做出类似 Excel 条件格式的效果，再导出 Excel：
- **数字格式化**：`df.style.format({'增长率':'{:.2%}', '金额':'{:.0f}'})`，把小数转成百分比、控制小数位。
- **高亮极值/空值**：`.highlight_max()` 高亮最大值、`.highlight_min(color='gray')` 最小值、`.highlight_null(null_color='red')` 空值；空值还可用 `na_rep='空值'` 替换显示文本。
- **色阶**：`.background_gradient(cmap=...)` 按数值大小渐变着色，类似热力图。
- **数据条**：`.bar(subset=[...], align='mid', color=['正色','负色'])`，正负值用不同颜色的条形长度表达大小。
- **隐藏列**：`.hide(axis='columns', subset=['某列'])`（旧版 `hide_columns`）。
- **导出**：链式设完样式后 `.to_excel('out.xlsx', engine='openpyxl', index=False)`，样式随之带入 Excel。

可链式串联，如 `df.style.format(...).highlight_max().highlight_null(...).hide(...)` 一次性套用多种样式。

### 十四、批量解析多份合同/文档并汇总成表
当有上百份格式相同的 PDF（如采购合同）要批量抽取固定字段汇总成 Excel：
1. 逐页读出文本（旧 `PyPDF2` 已弃，建议改用 `pypdf` 的 `PdfReader` / `page.extract_text()`）。
2. 用正则 `re.findall(r'字段名：\s*(.*)', text)` 按字段名匹配出值（合同编号、甲方、乙方、品名、数量、单价、总价等）。
3. 把各字段拼成字典 `append` 进列表，`pd.DataFrame(列表)` 转二维表，`to_excel` 导出。

> 关键在于「文档格式高度统一」——字段名和版式一致，正则才稳定。格式各异的文档需逐类写规则或改用更鲁棒的版面解析。

### 十五、网页表格抓取并存入数据库
对于静态 `table` 标签呈现的网页表格，可走「抓取 → 入库 → 回读」全流程：

1. **抓取**：`pd.read_html(url)` 返回页面内所有 table 组成的列表，取 `[0]` 得到目标表。
   - 常用参数：`header`（标题行）、`skiprows`（跳过行）、`attrs`（按属性定位某张表，如 `attrs={'id':'table'}`）、`parse_dates`。
   - 翻页抓取：观察 URL 的页码参数（如 `?p=2`），用循环拼出各页链接，逐页 `read_html` 后用 `pd.concat` 合并。
   - 局限：只适用于 `table` 标签的静态表格；JS 动态渲染、非 table 结构的数据 `read_html` 取不到。
2. **入库**：用 `pymysql.connect(host, user, password, db, port)` 建连接，`cursor.execute(建表 SQL)` 建表，再迭代 DataFrame 逐行 `insert`（`df.iloc[r, c]` 取值，注意类型转换避免插入报错）。完成后 `commit()` 提交、`close()` 关连接，否则库里看不到数据。
3. **回读**：`pd.read_sql('select * from 表', con)` 把库里数据读回 DataFrame 继续分析。

> 进阶：评论、动态渲染等非 `table` 结构的网页取数，以及取数后的文本/情感分析，见同板块「网页爬取与文本情感分析」。

### 十六、简单用户画像分析（完整小流程示例）
目标：拿一组交易明细，研究性别、年龄与交易金额的关系。串起前面的清洗动作做个范例。

1. **读入与摸底**：`read_excel` 读入，`data.shape` 看行列数，`data.dtypes` 看类型——常发现「用户ID 被当数值、交易金额/日期被当字符串」。
2. **类型转换**：
   - 用户ID 转字符串（它只用于标识、不该参与运算）：`data['用户id'] = data['用户id'].astype(str)`。
   - 交易金额去掉货币符号后转浮点：先 `str` 切片去符号再 `astype(float)`。
   - 交易日期转日期型：`pd.to_datetime(data['交易日期'], format='%Y年%m月%d日')`。
3. **重复判断**：`data.duplicated().any()`，返回 `False` 即无重复行。
4. **缺失处理**：
   - `data.isnull().any()` 看哪些列有缺失；`data.isnull().sum()` 看缺失个数；`/data.shape[0]` 算缺失比例。
   - 决策规则：缺失比例过高（如「受教育水平」缺 64%，远超 30%）的字段直接整列删除——`data.drop(labels='受教育水平', axis=1, inplace=True)`。
   - 缺失比例低的字段保留，按场景填补：类别型用众数 `data['性别'].mode()[0]`、数值型用均值 `data['年龄'].mean()`，再 `fillna(value={...}, inplace=True)`；或直接删缺失行。
5. **可视化结论**：
   - 性别 × 金额：`data.groupby('性别')['交易金额'].sum().plot.bar()`。
   - 年龄 × 金额：先 `pd.qcut(data['年龄'], 5)` 等频分箱，再按箱分组求和出柱状图，看哪个年龄段贡献金额最高/最低。
   - 金额区间 × 人数：先 `describe()` 看金额最小/最大值定边界，再 `pd.cut(金额, bins=[...])` 等距分箱，按箱 `count` 用户数出柱状图。

> 分箱选择：看「不同取值区间内的样本数分布」用等距分箱 `cut`；想让每箱样本数相近、再比较各箱指标用等频分箱 `qcut`。

### 常见坑与边界
- **类型不转就直接算**：字符串数字相加是拼接不是求和；Excel/CSV 里的金额带货币符号、日期是中文格式，读进来都会变字符串，不转类型就无法做数值运算或时间序列，且报错信息常不直观。读入后第一件事就是 `dtypes` / `info()` 核类型并转换。
- **缩进决定逻辑块**：Python 用缩进而非花括号划分代码块，缩进错位 = 逻辑错或报错。
- **切片"含左不含右"**：`range`、列表/字符串切片都遵循此规则，边界算错是最常见的 off-by-one 错误。
- **字典键取值用 `get` 更稳**：`d['x']` 在键不存在时直接报错，`d.get('x')` 返回空值，批处理时更安全。
- **多条件筛选忘加括号**：`df['a']==1 & df['b']==2` 会因运算符优先级出错，每个条件必须单独括起来。
- **缺失值阈值是经验值**：「超 30% 删列」只是常用经验线，关键字段即便缺失多也可能要保留并专门处理，别机械套用。
- **接口随版本漂移**：`pandas_profiling` 已更名 `ydata_profiling`，`PyPDF2.getPage()` 等老接口在新版被 `pypdf` 的新方法取代，`df.style.hide_columns` 改为 `hide(axis='columns')`，照抄老代码可能报错，需对照当前版本文档。
- **`read_html` 只能吃静态 table**：动态渲染或非表格结构的页面取不到，需改用浏览器自动化或接口抓取。
- **pandas 是单机内存方案**：数据量大到内存装不下时，应下沉到 SQL / 分布式计算，而非硬扛。
- **安装环境的版本/界面会变**：本文的安装路径与库版本仅供参考思路，以官方/镜像站当前文档为准。
