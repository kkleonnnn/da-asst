## 元信息
- **适用范围**：把一类重复、机械的表格整理 / 清洗操作交给"能读写文件、能运行代码"的对话式 AI 助手来做的场景，涵盖五类高频任务——行列结构转换（宽转长）、逆透视、批量插入空行、批量插入表头行、一格多值拆多行，以及让 AI 生成 Excel 函数公式。适合不熟悉公式、Power Query 或 pandas，但能用对话式 AI 上传文件的人；也适合分析师临时、小批量地省事。不适用于：数据量极大需进数仓 / ETL 批处理的场景；结构极不规则、需逐块复杂判断的脏数据；对结果准确性要求极高又无法人工抽样复核的关键计算；以及对数据安全敏感、不允许把原始文件上传外部 AI 服务的情况。

## 正文

> ⚠️ 以下全部为改写后的自有表述，示例均用中性造数，不含真实业务数据。

### 一句话方法论
把一类重复性的表格操作（结构转换、补行、补表头、拆单元格、套公式），用一段**结构化提示词**描述清需求、附上原始文件，交给支持文件读写与代码执行的对话式 AI，让它处理后回传一个**新文件**供下载。需求里通常要说死三件事：**对哪一（些）列做什么、关键参数是多少、输出为新文件并给下载入口**。重复需求还可固化成一个"触发词"，下次只发触发词 + 文件即可复用。

---

## 一、通用方法（所有任务共用）

### 1）前提：AI 要能"读写文件 + 跑代码"
所用的 AI 助手必须能接收上传的 Excel 文件、能运行代码 / 处理数据、并能产出可下载的结果文件。纯文本对话型、不支持文件处理的 AI 无法完成这些任务。

### 2）写提示词的通用骨架
不论哪类任务，一段好用的指令大体都包含这几段：
1. **读取文件**：上传表格，让 AI 读成数据框（必要时指明处理哪个 Sheet）。
2. **说清操作对象与参数**：对哪一列 / 哪段数据、做什么变换、关键参数取值多少（如插几行、按什么分隔符、哪列是标识列）。
3. **执行变换**：让 AI 完成核心处理。
4. **导出并给下载链接**：把结果**另存为新文件**，返回下载地址，避免覆盖原始数据。

### 3）把操作固化成可复用的"触发词"
如果同一操作要反复用，可以把上面这段提示词"固化"成一个带触发关键词的指令（如 `#宽转长`、`#插入间隔行`），约定"以后只要发文件 + 这个关键词，就自动执行整套流程"。本质是给 AI 助手预设一个轻量"宏 / 工作流"，把重复需求收敛成一个可记忆的快捷指令，既省去每次重述，也降低参数说错的风险。

### 4）所有任务都适用的通用坑 / 边界
- **结果必须复核**：AI 处理较大表时可能漏行、错位、截断，导出后务必抽查几行与原表对照，确认数值未串位、无丢列再投入使用。
- **核对总行数 / 总列数**：每类任务都有一个可验算的行 / 列数关系（见各节），拿到结果先验算，能快速判断有没有漏处理或多处理。
- **格式可能丢失**：经 AI 处理后，单元格格式、公式、合并单元格、条件格式等可能不被保留，结果多为纯值；对格式敏感的表格要复核。
- **数据隐私与合规**：含真实业务 / 个人数据时，先评估是否可上传第三方 AI 并做脱敏；敏感数据优先用本地的 Power Query / pandas 完成。
- **可复用性低于原生方案**：高频、需版本管理、要进生产管线的需求，用 Power Query 或 pandas 脚本更稳，AI 适合一次性、小批量、临时省事。

## 二、按任务类型分述

### 任务 A：行列转换——宽表转长表（宽转长 / unpivot / melt）

**做什么。** 把一张"一行多列"的宽表，转成"竖向堆叠"的长表：保留一个唯一标识列，把其余各列的列名拍扁成一个"类别"列、把对应单元格的值收进一个"数值"列。

- **宽表（处理前）**：第一列是维度（如季度 Q1～Q4），后面横排 1 月、2 月……12 月共 12 个数值列，整张表是 4 行 × 13 列的矩阵。这种形态适合人眼阅读，但很多图表和透视工具要求"一指标一列"。
- **长表（处理后）**：变成三列——维度、月份、数值。原来 4×12=48 个数值格，被拆成 48 行记录，每行写清"哪个季度、哪个月、值是多少"。这才是透视表、绘图库、数据库友好的"整洁数据（tidy data）"形态。

**为什么用 AI。** 不用记 Excel 的"逆透视"路径，也不用写 pandas 的 `melt`，自然语言描述即可；列很多时（12 个月、几十个指标）手工搬运易错，AI 一次成型。

**提示词要点（在通用骨架上补两点）：**
- **指定哪一列是唯一标识（id，保持不动）**，其余列做 melt / unpivot。
- **给新生成的两列命名**：告诉 AI"类别列""数值列"分别叫什么（如"月份""数值"）。
- 可选：转换前先**规整列名**（如统一为"1 月""2 月"），否则长表的"月份"列取值会参差不齐。

**示例（中性造数）。**
处理前（宽表，节选）：

| 季度 | 1月 | 2月 | 3月 | … | 12月 |
|------|-----|-----|-----|---|------|
| Q1 | 1494 | 1391 | 3756 | … | 4958 |
| Q2 | 1934 | 1817 | 2808 | … | 2296 |
| Q3 | 1399 | 1816 | 4461 | … | 4253 |
| Q4 | 2492 | 4388 | 3821 | … | 1105 |

处理后（长表，节选）：

| 维度 | 月份 | 数值 |
|------|------|------|
| Q1 | 1月 | 1494 |
| Q2 | 1月 | 1934 |
| Q3 | 1月 | 1399 |
| Q4 | 1月 | 2492 |
| Q1 | 2月 | 1391 |
| … | … | … |

**专属坑 / 边界：**
- **核对行数**：转换后行数应等于 标识列行数 × 待转列数（本例 4×12=48）；对不上说明漏列或多带了列。
- **别把标识列卷进去**：忘了指定 id 列，AI 可能把"季度"也当成被展开对象，结果错乱。
- **空值与缺列**：原表有空单元格时，长表会出现空数值行，按需决定保留还是过滤。

### 任务 B：逆透视为长表（unpivot，按 Sheet / 列区间指定范围）

> 逆透视与任务 A 的"宽转长"是同一类数据整形（都对应 pandas 的 `melt`）；区别在使用习惯：这里更强调**按 Sheet 和列区间指定逆透视范围**、并把它固化成提示词反复执行。两节可结合使用。

**做什么。** 把"横着铺开的很多列"折叠回"竖着的一列分类 + 一列数值"，让数据变成"一行一条记录"的规整长表——这是透视表、可视化和建模真正想要的输入格式。常见于把人工录入的报表喂给 BI、透视表或建模工具之前的预处理。

**长表结构（中性造数）。**
原始宽表（节选）：

| 年 | 1月 | 2月 | 3月 | … | 12月 |
|----|-----|-----|-----|---|------|
| 2019 | 1490 | 1390 | 3760 | … | 4960 |
| 2020 | 1930 | 1820 | 2810 | … | 2300 |

逆透视后的长表（节选）：

| 年 | 类别(variable) | 数值(value) |
|----|------|------|
| 2019 | 1月 | 1490 |
| 2020 | 1月 | 1930 |
| 2019 | 2月 | 1390 |
| 2020 | 2月 | 1820 |
| … | … | … |

**可复用逆透视提示词的四步（在通用骨架上细化）：**
1. **加载文件**：读入待处理的 Excel，并指明要处理哪个 Sheet。
2. **指定逆透视范围**：说清哪些列要折叠（如"B 列到 M 列"或"1 月到 12 月"），哪些列作为 ID 列保留。
3. **执行逆透视**：保留首列（如"年"）作为 ID 列，其余指定列做逆透视；在 pandas 中对应 `df.melt(id_vars=['年'], value_vars=[...])`。
4. **导出结果**：把长表另存为新的 Excel 文件。

**专属坑 / 边界：**
- **ID 列选错**：该保留的标识列若也被卷进逆透视，会丢失"这一行属于哪条记录"的归属信息，务必明确哪些是 ID 列。
- **列名当数据**：折叠后原列名会落进类别列，记得把默认的 `variable` / `value` 重命名为有业务含义的名字（如"月份""金额"）。
- **混合类型**：被折叠的多列若类型不一致（有的数值、有的文本），合并到一列后类型会被迫统一，可能引入隐患；逆透视前最好确认这些列同质。
- **空值膨胀**：宽表里大量空单元格逆透视后会变成大量空值行，必要时折叠后过滤掉空值，避免长表虚胖。
- **核对结果**：抽查长表行数是否约等于"原行数 × 折叠列数"，并随机核对几条值。

### 任务 C：每行之间批量插入固定数量空行

**做什么。** 把一张已有的表格，在每条记录之间统一插入若干空行（做打印分隔、留出手工填写空间、给后续分组 / 小计预留位置、便于按记录分块阅读）。适合数据量不大、操作偶发、不想手动逐行插入的场景。

**提示词要点（在通用骨架上补两点）：**
- **插入行数务必写死**（例如"每行之间插入 4 行空行"）；固化成触发词时尤其要把 N 写进规则。
- **强调只在数据行之间插入、表头不插**（明确首行是标题），避免在表头与首行之间也插空行。

**AI 内部通常怎么实现（了解原理，便于校验）。** 常见做法是"辅助列 + 排序"：给每行原始数据一个递增序号（1、2、3…），再为要插入的空行生成排在它后面、却排在下一条记录前面的"占位序号"；把原始行与占位空行拼到一起、按序号排序，原始记录与其后的空行就自然交错；最后删掉辅助列、导出。只要表头不被误当数据行插空行、且空行确实落在相邻两条记录之间，结果即正确。

**示例（中性造数）。** 设原始表为按年份的月度数值表（表头「年 / 1月 / … / 12月」，下有 2019—2022 四行），要求每行之间插入 3 行空行：

| 年 | 1月 | 2月 | … |
|----|-----|-----|----|
| 2019 | 1494 | 1391 | … |
|  |  |  |  |
|  |  |  |  |
|  |  |  |  |
| 2020 | 1934 | 1817 | … |
|  |  |  |  |
|  |  |  |  |
|  |  |  |  |
| 2021 | … | … | … |

（表头保持在最上方不被插空行。）

**专属坑 / 边界：**
- **核对总行数**：新行数 ≈ 原数据行数 ×(1 + N)（首 / 末是否补空行按需约定），快速判断有没有漏插或多插。
- **表头别被插空行**：若 AI 把表头当数据，需在提示里强调首行是标题。

### 任务 D：批量为数据插入表头行

**做什么。** 给只有数据、缺少列名的表格补上字段名标题；或给竖向堆叠的多段数据，在每段上方各补一行表头。目标是：表头列数与数据列数一一对应、不破坏原有数据顺序、已有表头的位置不重复插入。

**操作套路（核心是"先读懂列结构，再决定哪里需要插"）：**
1. **上传并读取数据**：弄清一共多少列、各列数据形态。
2. **确定 / 读取表头内容**：若原表已有可作模板的表头，先读出来存成列名列表；若完全没有，则按列数与含义拟定字段名（如"年、1月、2月……12月"）。
3. **新建一张与原表列结构相同的空表**：以原数据列名 / 列数为基准建空框架，避免列数对不上导致错位。
4. **遍历数据、按需插入表头**：在需要的位置（如每段数据最前面）插入表头行；若某位置当前已是表头（非空 / 已有列名）则跳过、不重复插入；遇空行不处理。
5. **保存为新文件并提供下载。**

**通用提示词（可套用）：**
> 加载包含数据的 Excel 文件。
> 读取文件并获取表头：读出可用作模板的标题行，保存为列名列表（若无表头则按列数拟定字段名）。
> 创建一个与原表列结构相同的空表。
> 遍历数据并在需要的位置插入表头行：如果该位置当前不为空（已是表头）则跳过，空行不处理。
> 将处理后的数据保存为新的 Excel 文件并提供下载。

**示例（中性造数）。**
处理前（只有数值、缺列名）：

| | A | B | … | M |
| --- | --- | --- | --- | --- |
| 1 | 2019 | 1494 | … | 4968 |
| 2 | 2020 | 1934 | … | 2296 |

处理后（在数据上方补齐表头，列与数据一一对应）：

| 年 | 1月 | 2月 | … | 12月 |
| --- | --- | --- | --- | --- |
| 2019 | 1494 | 1391 | … | 4968 |
| 2020 | 1934 | 1817 | … | 2296 |

若需在每段数据上方都加表头（如按年份分段），处理后应在每一段最前面各出现一行字段名，而原始第一行表头位置不被重复插入。

**专属坑 / 边界：**
- **列数必须对齐**：表头字段个数要与数据列数完全一致，否则整体错位；让 AI 以原数据列数为基准生成表头。
- **避免重复插入**：让 AI 判断目标位置是否"已经是表头"，已有则跳过，否则两行标题叠在一起。
- **空行的处理**：遇空白行应跳过，而非误当成数据段开头插表头。
- **字段名的准确性**：无表头时由 AI 拟定的列名只是猜测，需人工核对每列含义（尤其数值列易张冠李戴）。

### 任务 E：一格多值拆成多行（一对多拆分）

**做什么。** 当一列单元格里塞了多个用分隔符（逗号、分号、顿号、空格等）隔开的值时，让 AI 按分隔符切开，把每个值"竖"成独立一行，其余字段原样复制到每一行——即把"一行多值"展开成"多行单值"的规整长表。这有利于做分组统计、去重计数、和其他表关联。

**操作套路：**
1. **指明要拆的列**（如"城市"列）。
2. **指明分隔符**（逗号、顿号、分号、空格等），让它按此切分。
3. **要求转成多行**：明确"拆分后每个值各占一行"而非拆成多列，并保留同一记录的其他字段。
4. **要求产出可下载文件。**

**通用提示词（可套用）：**
> 读取表格中需要拆分的目标列。
> 按指定分隔符拆分该列的值。
> 把拆分后的每个值转换为单独一行，其余字段保持不变。
> 保存为新文件并提供下载。

**示例（中性造数）。**
处理前（一个单元格里多值）：

| 城市 |
| --- |
| 北京,上海,广州,深圳,杭州 |

处理后（拆成多行，一行一值）：

| 城市 |
| --- |
| 北京 |
| 上海 |
| 广州 |
| 深圳 |
| 杭州 |

若原表还有其他列（如"销售员=张三"），拆分后这些列应被复制到每一行，保持每行信息完整。

**专属坑 / 边界：**
- **分隔符要说准**：中文逗号"，"、英文逗号","、顿号"、"是不同字符，混用会切分不干净，最好先确认或要求 AI 把多种分隔符都识别。
- **首尾空格与空值**：拆出来的值可能带空格或出现空字符串，让 AI 顺手 trim 并丢弃空值。
- **其他字段的对应关系**：多列表格拆分时，确认非拆分列是"按行复制"而非错位。
- **核对行数**：拆分后总行数应等于各单元格值个数之和。
- **方向别搞反**：本方法是"拆成多行"（长表）；若业务要的是"拆成多列"（宽表），属另一种处理，提示词要相应改写。

### 任务 F：生成 Excel 函数公式

**做什么。** 让 AI 帮你挑选 Excel 函数、生成公式并校验。尤其适合"知道要算什么、但记不清该用哪个函数 / 参数怎么填"的场景，以及一次要写很多条相似公式的批量处理。效果好坏几乎全看"你有没有把表结构和需求讲清楚"。

**提示词模板（四段式 + 一个触发词）：**
1. **文件描述**：说明这张表有哪些列、各列代表什么、分别在第几列。例：A 列年份，B 列月份，C 列收入金额。
2. **需求描述**：用一句话说清要算什么、范围是什么。例：计算 2010 年 6 月到 2011 年 11 月之间的收入合计。
3. **函数使用**：不必自己指定函数，写"由你选择合适的函数"，把选型交给 AI。
4. **输出公式**：约定一个触发词（如 `#输出公式`）。写好前三段后发触发词，AI 就直接给公式，不再绕圈解释。

把前三段存成模板，日常只改字段值即可复用；若能把数据文件一并发给 AI，它能结合真实表结构给得更准。

**示例：跨年份月份区间求和。** 按上面模板（A 列年、B 列月、C 列收入），让 AI 算"2010 年 6 月至 2011 年 11 月"的收入合计，它会给出一条多条件求和公式：对 C 列求和，同时用 A 列限定年份区间、B 列限定月份区间：

```excel
=SUMIFS(C:C, A:A, ">=2010", A:A, "<=2011", B:B, ">=6", B:B, "<=11")
```

拿到公式后：打开 Excel → 在任一空单元格粘贴 → 回车得结果。

> ⚠️ 这种"年、月分两列、用大小于号卡区间"的写法有个坑：它会把"年份在 2010–2011 之间**且**月份在 6–11 之间"的行全部算进去，而不是严格的"从 2010 年 6 月到 2011 年 11 月这条时间线"。比如 2010 年 11 月、2011 年 6 月都会被纳入。区间跨年且对边界敏感时，更稳妥的做法是把年月拼成一个可比较的值（如 年×100+月）再卡区间，或先抽几行手工验算确认口径。

**公式很多时的批量套路**（"AI 试算 + 校验 → 回填 → 下拉填充"）：
1. 让 AI 先对样例数据做一次试算并给出对应公式，用算出的数对一下公式逻辑是否正确。
2. 确认无误后，把公式搬回 Excel 首行单元格。
3. 用 Excel 的下拉填充（拖拽填充柄）让公式自动套用到整列。

这样既借 AI 校准了公式逻辑，又靠 Excel 自身填充能力省去逐格输入。

**专属坑 / 边界：**
- **AI 会写错参数**：尤其区间边界、绝对 / 相对引用（`$`）这类细节，务必抽几行手工验算后再大面积套用。
- **区间口径要说死**："6 月到 11 月""2010 到 2011"易产生上面的跨年边界歧义，需求里把含不含端点、跨不跨年讲清楚。
- **列位置要交代准**：AI 看不到真实表时全靠文字描述列位置，描述错了公式就引错列。
- **结果以 Excel 实算为准**：AI 给的是公式，真正的数还得回 Excel 跑出来核对，别直接采信 AI 口算的数值。

## 三、不依赖 AI 时的等价实现

这些任务都不是 AI 专属，原生工具同样能做、结果一致，AI 只是降低了门槛。高频或需版本管理时，原生方案更稳：

- **宽转长 / 逆透视**：
  - Excel / Power Query：选"转换 → 逆透视列（Unpivot Columns）"，选中要折叠的列即可。
  - pandas：`df.melt(id_vars=['标识列'], var_name='类别', value_name='数值')`。
  - SQL：用 `UNION ALL` 把每列拼成一列，或在支持的方言里用 `UNPIVOT`。
  - 早期 Excel：可用"多重合并计算数据区域"透视表或公式硬搬，较繁琐，不推荐。
- **插入空行 / 拆分单元格**：Excel 里用"辅助列 + 排序"法插空行；拆分可用"分列"配合转置，或 pandas 的 `str.split()` + `explode()`。
- **批量插表头、批量公式**：pandas 拼接表头行、或 Excel 下拉填充公式。
