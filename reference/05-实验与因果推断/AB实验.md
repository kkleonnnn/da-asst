## 正文

### 一句话总览

AB 实验的本质是一次**假设检验**：把流量随机切成对照组与实验组，用统计方法判断"两组指标的差异是真实存在，还是仅由随机抽样波动造成"。一套完整的 AB 实验要走通五段：**实验前**用功效分析反推样本量（确保真有效果时能检出）；**分流时**选对随机化单元并在必要时用 AA 校验分流是否公平；**设计落地时**想清多改动如何拆组、流量按几比几分、留存/转化用什么口径；**判读时**把"分流不均"和"版本效应"两个搅在一起的因素拆开；**决策时**在核心指标与护栏指标冲突时做"收益—代价"的量化权衡。核心始终是先回答"我要回答什么问题、要排除哪些干扰"，而不是套模板。

## 第一部分 · 实验前的统计原理：假设检验、显著性、功效与样本量

### 1. 为什么实验结论要走假设检验

做一次改动后，实验组的指标几乎总会和对照组有点不同——但这点不同可能只是抽样噪声。假设检验提供一套规则：先假设"两组其实没差别"（零假设），再看观测到的差异在这个假设下出现的概率有多小，从而决定是否推翻它。这样就把"看起来涨了"变成"有统计依据地判断涨没涨"。

### 2. 选指标前先看它的统计属性

要判断该用哪种检验、怎么算显著，先要识别指标属于哪一类：

- **均值类指标**：人均使用时长、人均成交额、人均互动次数等。
- **概率/比例类指标**：次日留存率、付费率、转化率等（取值是"发生/不发生"的占比）。
- **分位数类指标**：使用时长的中位数、付费金额的 25 分位 / 95 分位等。
- **比率类指标**：点击率、消耗率等由两个量相除得到的指标。

日常实验里最常用、也最好处理的是前两类——**均值类**和**比例类**。分位数类、比率类的检验更复杂，常规实验里出现较少，下面的方法主要服务于均值类和比例类。

> 提醒：有些看似常用的指标（如某段周期内的成交总额、订单总量这类"绝对总量"）通常不在直接做显著性检验的范畴内，因为总量受样本规模影响、不便直接对比，一般转成人均/比率形式再检验。

### 3. 单尾还是双尾：默认用双尾

检验的备择假设可以是"实验组 ≠ 对照组"（双尾），也可以是"实验组 > 对照组"或"< 对照组"（单尾）。实践中**建议默认用双尾检验**，原因有二：

1. **让数据替你做决策，而不是替数据预设答案**。做实验本就是想靠数据驱动判断，如果一上来就假定"只会变好"，相当于提前替数据下了结论，削弱了检验的客观性。双尾对涨和跌都保持警惕。
2. **改动的真实方向未必如预期**。我们期待指标变好，但一个改动完全可能带来反向效果。双尾能同时识别正向和负向的显著变化，更贴近现实；单尾只盯一个方向，可能漏掉与预期相反的真实影响。

### 4. T 检验还是 Z 检验：看是否已知总体方差

两种检验的选择主要取决于**样本量大小**和**是否已知总体方差**：

- 不知道总体方差 → 用 **T 检验**。
- 已知总体方差，且样本量 ≥ 30 → 用 **Z 检验**。
- 已知总体方差，且样本量 < 30 → 用 **T 检验**。

实务中样本量通常远大于 30，而样本量较大时 T 分布会非常接近正态分布。因此为了计算方便，**实际业务里多数场景直接用 Z 检验近似即可**，简单且足够稳健。

### 5. 两类错误：检验会犯的两种错

检验结论是基于样本数据的统计推断，不等于事实真相。把"检验结论"和"事实"交叉，会出现四种情况，其中两种是错误：

| | 事实上两组确有差异 | 事实上两组没有差异 |
|---|---|---|
| **检验推断"两组不同"** | 推断正确 | 第一类错误 |
| **检验推断"两组相同"** | 第二类错误 | 推断正确 |

- **第一类错误（弃真 / 假阳性 False Positive）**：事实上两组相同，检验却判定不同——把"没效果"误报成"有效果"。其发生概率记作显著性水平（Significance Level）。把允许犯这种错误的概率控制得很小，是因为它相当于"对小概率事件下了肯定结论"。**工业界常用阈值为 5%**。
- **第二类错误（取伪 / 假阴性 False Negative）**：事实上两组不同，检验却判定相同——真有效果却没识别出来。其发生概率常设为 **20% 左右**，对应的**统计功效（Power）= 1 − β**，即"真有差异时能正确检出"的概率。

### 6. 为什么能用正态分布：中心极限定理

均值类指标的总体分布未必是正态的。比例类指标服从二项分布；很多金额、时长类指标是明显右偏的（大多数用户贡献小额，少数高值用户拉高总量）。看上去无法直接套用正态方法。

**中心极限定理**解决了这个问题：无论原始总体是什么分布，只要从中重复抽取足够大的样本，**这些样本均值本身的分布会趋近正态分布**。也就是说，我们检验的是"两组某个指标的均值之差"，而均值的抽样分布近似正态——于是可以放心地按正态分布的性质展开后续计算。

### 7. 怎么算显著：标准化与 Z 统计量

有了中心极限定理的保证，均值的抽样分布近似正态；再借助正态分布"任何正态都可标准化为标准正态"的性质，把观测到的差异**标准化**成 Z 统计量，再查标准正态分布得到对应的 **P-value**，据此判断是否显著。

**Z 统计量计算公式：**

- 均值类指标（需知道两组样本量、两组均值、两组方差）：

  $$Z = \frac{|\mu_1 - \mu_2|}{\sqrt{\dfrac{\sigma_1^2}{N_1} + \dfrac{\sigma_2^2}{N_2}}}$$

- 比例类指标（需知道两组样本量、两组比例）：

  $$Z = \frac{|P_1 - P_2|}{\sqrt{\dfrac{\sigma_1^2}{N_1} + \dfrac{\sigma_2^2}{N_2}}}$$

  比例类的方差可由二项分布性质直接得出：$\sigma^2 = P \times (1 - P)$，无需单独估计。

**判读规则**：算出 Z 统计量后查表得 P-value。

- P-value < 0.05 → 拒绝零假设，认为两组指标**有显著差异**（改动有效）。
- P-value ≥ 0.05 → 不能拒绝零假设，认为两组**无显著差异**（不能说改动有效）。

### 8. 算例（中性造数，仅示意流程）

设某产品功能做了一次入口改版实验，对照组沿用旧入口、实验组改为新入口，关注两个指标：**互动率**（比例类）与**人均互动次数**（均值类）。

| 组别 | 进组用户数 | 互动率 | 人均互动次数 |
|---|---|---|---|
| 对照组 | 517 | 22.82% | 0.7698 |
| 实验组 | 482 | 23.03% | 0.7988 |
| 差异大小 | — | 0.21pp | 3.76% |
| Z 统计量 | — | 0.0789 | 0.2864 |
| P-value | — | 0.9371 | 0.7746 |
| 显著性 | — | 不显著 | 不显著 |

读法：

- 两组互动率都在 23% 附近，绝对差异约 0.21pp，P-value 远大于 0.05 → 互动率**无显著差异**。
- 两组人均互动次数相对差异约 3.76%，P-value 同样大于 0.05 → 人均互动次数**无显著差异**。

结论：该次改版没有带来统计上可确认的提升。注意"看起来涨了 3.76%"并不等于"真的涨了"——这正是必须做显著性检验的意义：差异在抽样波动范围内时，不能当作真实效果。

### 9. 实验前反推样本量：决定样本量的四个旋钮

显著性检验只解决"实验跑完后差异是否可信"，但实验能不能跑出可信结论，在开跑前就由样本量决定了。样本量太小，即便改动真有效，也大概率检不出来（假阴性）。因此正式实验前要先做一次"功效分析"，反推需要多少样本、多大流量、跑多长时间。样本量由四个量共同决定，调任何一个都会改变所需样本：

- **效应量（Effect Size）**：你希望这次实验**至少能检测到的最小真实差异**。希望能识别的改善越微小，需要的样本量越大；只想识别"大动作"，样本量就可以小。它是最关键、也最容易被忽视的输入——必须由业务判断"多大的提升才值得上线"来给定，而不是事后看数据凑。
- **显著性水平 α（第一类错误率）**：你愿意承受的"把没效果误判成有效果"的概率，常用 0.05。
- **统计功效 Power（= 1 − β）**：真有差异时能正确检出的概率，常设 **0.8**。设得越高（如 0.9），对漏判越苛刻，需要的样本量越大。
- **基线转化率（Baseline Conversion Rate）**：实验所控制的、改动前的原始转化水平，用来和实验组对比。比例类指标必须先知道这个基线，才能推算预期转化与效应量。

> 直觉记法：**想检测的差异越细 → 样本量越大；要求的把握（功效）越高、容忍的误报（α）越低 → 样本量越大。** 四个旋钮里，效应量和基线转化率来自业务，α 与功效来自统计约定。

### 10. 比例类指标的样本量公式

对转化率/付费率这类比例指标，单组所需样本量可近似按下式估算：

$$n = \frac{(z_{\alpha} + z_{\text{power}})^2 \cdot 2 \cdot p_1 (1 - p_1)}{(\text{effect size})^2}$$

其中：

- $p_1$ 为基线转化率，$p_2$ 为预期转化率；
- 效应量（相对提升口径）$=\dfrac{p_2 - p_1}{p_1}$；
- $z_{\alpha}$ 是显著性水平对应的标准正态分位数（双尾用 $1-\alpha/2$），$z_{\text{power}}$ 是功效对应的分位数。

公式直观上说明了三件事：分母是效应量的平方——**想检测的相对提升缩小一半，所需样本量会膨胀约 4 倍**；分子里的 $p_1(1-p_1)$ 是比例指标的方差，基线越接近 50% 方差越大、越费样本；$(z_\alpha + z_{power})^2$ 把"误报容忍度"和"把握度"打包进来，要求越严样本越多。

### 11. 用代码估算（中性示例）

实务中不必手算，统计软件或在线计算器输入上述四个量即可。Python 里可借助 `statsmodels` 或 `scipy` 直接计算：

```python
import scipy.stats as stats

# 1. 设定四个旋钮
alpha = 0.05      # 显著性水平（第一类错误）
power = 0.8       # 统计功效（1 - 第二类错误）
p1 = 0.10         # 基线转化率（示例造数）
p2 = 0.12         # 预期转化率（示例造数）
effect_size = (p2 - p1) / p1   # 相对提升口径的效应量

# 2. 求两个临界 Z 值（双尾）
z_alpha = stats.norm.ppf(1 - alpha / 2)
z_power = stats.norm.ppf(power)

# 3. 套样本量公式（单组所需样本量）
n = ((z_alpha + z_power) ** 2 * 2 * p1 * (1 - p1)) / effect_size ** 2
print(f"每组约需样本量：{n:.0f}")
```

注意示例里的 `p1`、`p2` 均为中性造数，仅演示流程；真实场景应填入自己业务的历史基线和业务上"值得上线"的目标提升。

### 12. 把样本量翻译成"流量 × 时长"

算出"每组需要多少样本"只是第一步，落地时还要换算成可执行的实验排期：

1. 单组样本量 × 实验组数（A/B 两组就是 ×2），得到实验总需样本。
2. 用日均可分流的合格流量去除总需样本，得到**需要跑的天数**。
3. 天数还要向上取整到完整周期（通常按整周），以抹平工作日/周末的用户行为差异，避免周期性噪声混入结论。

如果换算出来的时长长到业务等不起，就要回头调旋钮：要么接受识别更大的效应量（牺牲灵敏度），要么扩大分流比例，而不是硬把实验提前叫停——提前停会让功效不足，结论不可信。

## 第二部分 · 分流与 AA：分流单元（PV/UV）选择与 AA 测试可用性判定

### 1. PV 分流与 UV 分流：先想清"实验单位是什么"

分流单元（randomization unit）指的是把流量切成对照组/实验组时，"以什么为单位投硬币"。常见两种：

- **UV 分流（按独立用户）**：同一个用户每次访问都被稳定地分进同一组。适合衡量"对用户整体行为的长期改变"——留存、活跃、累计付费、跨多次会话才显现的效果。
- **PV 分流（按访问/请求）**：每一次访问（甚至每一次请求）独立投硬币，同一用户在不同访问里可能落到不同组。适合衡量"单次访问内即可观测、且不依赖用户记忆/累积"的效果。

**判断口诀：效果在多长的时间窗里才稳定下来，就用多大的单元。**
- 若评估对象是"用户级、需要多次访问才能体现"的指标 → 用 UV，否则同一人前后落到不同组会污染估计，把长期效应稀释掉。
- 若评估对象是"单次访问即可判定、且各次访问近似独立"的效果 → PV 也可行，且因为样本数量（访问数 >> 用户数）更大，能更快达到统计功效。

#### 典型对应关系（中性示例）

| 评估对象 | 建议单元 | 理由 |
|---|---|---|
| 推荐/匹配等"持续作用于同一用户"的策略，看次日/7日留存、活跃、累计付费 | UV | 同一用户必须长期处于同一策略下，效果才可信；多次访问被打散会削弱影响 |
| 页面元素改版后"本次访问内"的点击率、转化率、停留时长 | PV / UV 皆可 | 单次访问即可观测；若样本紧张可用 PV 加速，但需确认访问间近似独立 |
| 短期片段化的小实验（如某个轮转位、广告投放位的即时表现） | PV 更常见 | 看的是访问次数维度的即时反应 |

#### 三个容易踩的坑

1. **"绝大多数分流默认就该用 UV"是一种常见但不严谨的口径**。日常确实以 UV 为主，但这是默认值而非铁律；当指标是访问级、且需要更大样本时，PV 有其合理性。关键永远是"效果在哪个层级稳定"。
2. **指标层级要和分流层级对齐**。比例类指标和均值类指标（如各种 `sum(pv)/sum(uv)` 形态的"人均/次均"比率、分位数指标）在 PV 分流下的解释会更复杂，计算点估计与方差时要格外小心，不能照搬 UV 下的简单口径。
3. **比快不等于比对**。PV 让累计型指标（如累计留存）下降得快、达到显著更快，但如果业务效果本质上是用户级长期效应，用 PV"凑显著"是不合理的——快是因为单元不对，不是因为效果真大。

> 实务建议：定单元之前先回答两个问题——这次实验/分析的**业务目的**是什么？要看的**核心指标**是用户级还是访问级？目的和指标共同决定单元，而不是反过来。两个指标也常常需要联合解读（例如 UV 看用户规模、PV 看使用频次），不要孤立看单一口径。

### 2. AA 测试是什么、解决什么问题

AA 测试 = 把两组（甚至多组）**不施加任何改动**的用户分流出来，对比它们之间的指标差异。本质是在检验"分流机制本身是否公平"——两组在没有任何处理时，理论上指标应当一致，差异只应来自随机抽样的波动。

它回答的是一个工程/方法层面的问题：**我的随机分流是不是真的随机、组间是否天然可比**，而不是业务层面的"功能有没有效果"。

### 3. 什么情况下值得做 AA

- **对自己的分流机制/数据没有把握时**：典型如新接入的实验平台、新写的分流逻辑、对历史分流结果存疑。
- 当某次实验**出现了数据上的变化，但变化与业务逻辑对不上、拆不出合理解释**时，做 AA 自检，排除"是不是分流本身就有偏"这种底层故障。
- 反过来，**成熟稳定的平台不必每次都做 AA**。一个长期运行、有大流量验证过分流随机性的平台，其分流机制可视为一个客观的随机过程，本身不引入偏差；偏差通常来自用户群体属性本身，而非分流。强行每次都做 AA 既拖慢节奏，也可能是对平台缺乏信心的表现。

> 经验提醒：作为分析师，遇到异常不要一上来就甩给"分流有问题/AA 没做好"。更应该先从策略、功能、用户行为和数据表现本身去找解释，否则 AA 容易变成万能借口，掩盖真正的归因。

### 4. 怎么做 AA：操作步骤

1. **在不上线任何改动的前提下**，按与正式实验相同的机制，把流量切成 A1、A2 两组（理论上和做 AB 一样，只是两组都是"原版"）。
2. **先看分流是否均匀**：两组的用户量/访问量级是否接近预期比例。量级明显失衡是分流故障的第一信号。
3. **再看核心指标是否一致**：两组的关键指标差异应当落在随机波动范围内，不应出现稳定、可复现的系统性差距。
4. 跑足够的观测周期，避免用极短窗口下的偶发波动下结论。

### 5. 怎么判定 AA 通过/不通过

判定标准有两个层次，建议结合使用：

- **逻辑对齐法（定性）**：看出现的差异能否被合理解释、能否拆解。如果差异说不清、拆不开、与已知逻辑矛盾，就说明分流可能有问题。
- **统计显著性法（定量）**：组间差异在统计检验下应当**不显著**。可以用一个直观的经验参照——若把分流随机性看作客观抽样过程，按显著性水平的含义，大约只有控制水平（如约 5%）比例的指标会"误报"为显著差异，这属于第一类错误的正常范围，不必据此否定分流。换句话说，**偶有个别指标显著不等于 AA 失败，但成片、可复现的显著差异则要警惕**。
- 进阶做法：可对组间已存在的偏差做**纠偏/校正**，例如用 CUPED 一类方法，利用实验前的协变量把组间初始偏差扣除，构造方差更小、更敏感的指标，从而在 AA 通过的基础上进一步提升后续 AB 的检验功效。

### 6. 原假设、备择假设与"互换"的辨析

这是常被追问、也最容易被绕晕的点，关键在于"AA 看的是差异，AB 看的是效果"，两者的零假设方向恰好相反：

- **AB 测试的常规设定**：原假设 H0 = "两组无差异（新策略无效）"，备择 H1 = "两组有差异（新策略有效）"。我们希望拒绝 H0，证明有效。
- **AA 测试的设定**：因为两组本就没有任何处理，我们**期望它们一致**。所以 AA 的"理想结果"是**不能拒绝"两组相同"**——即希望检验结果不显著。

所谓"把原假设和备择假设互换"，需要分清两种意思，否则会答错：

1. 如果指的是 **AA 与 AB 之间**：AA 的诉求是"证明无差异（保留零假设）"，AB 的诉求是"证明有差异（拒绝零假设）"，两者的目标天然相反——这不是简单互换符号，而是实验目的不同。
2. 如果指的是在**同一检验内机械地把 H0 与 H1 对调**：经典假设检验框架是"默认原假设成立，用数据去尝试推翻它"，原假设通常设为"无效应/无差异"这一保守、可被证伪的命题。直接对调会破坏这套逻辑——你无法用"不显著"去**证明**两者相等（不显著只代表"证据不足以说不同"，不等于"已证明相同"）。所以严格说，AA 想要的不是"证明相等"，而是"没有证据表明不等"。

> 一句话收口：AA 不是为了证明两组绝对相等，而是为了在上线真正的改动前，确认"分流这件事本身没有引入系统性偏差"，给后续 AB 的因果解读扫清地基。

## 第三部分 · 实验设计与落地：多改动拆组、流量分配、留存/转化口径

实验设计阶段最容易卡住的不是统计，而是落地：**多个改动到底合并测还是拆开测、对照组和实验组按几比几分流、留存和转化按"进组相对天数"还是"自然日历天"来算**。这三件事都有可遵循的判断套路，核心是先想清"我到底想回答哪个问题、要排除哪些干扰"。

### 1. 同一页面同时改了好几处，怎么安排实验

实务里经常是一个页面同时上线多个改动。要不要拆开测、怎么拆，先看这几个改动**彼此之间会不会相互影响**。

**第一步：判断改动之间有没有相互作用。**
从业务逻辑上判断这几个改动有没有彼此影响、相互增强或抵消的可能。如果判断它们**相互独立、互不干扰**，就可以放在一个实验里同时跑、并行评估——这也是功能预期能在线上共存、各自独立生效时的合理做法。判断依据可以借鉴业务方的判断：如果业务认为某功能有效、其内部对效果的预期是各改动效应可以叠加的，那通常已经考虑过它们相互独立、且未来上线后会共存。

**第二步：如果改动之间会相互影响，用分组拆解效应。**
当几个改动**预期会相互影响**、且业务判断"这些功能合起来才有效、效果是叠加出来的"时，正确做法是把效应拆开分组测。以两个策略 A、B 为例，可以分三组：

| 组别 | 配置 | 作用 |
|---|---|---|
| 对照组 | 不上 A、不上 B | 基线 |
| 实验组 A | 只上 A | 测 A 单独的效应 |
| 实验组 B | 上 A + B | 测在 A 基础上叠加 B 的增量 |

这样能挑出"对线上影响更大的那个策略组合"。

**还能不能拆得更细？** 可以。比如想把 A 单独、B 单独、A+B 三种组合都分清楚，就扩成四组：

| 组别 | 配置 |
|---|---|
| 对照组 | 不上 A、不上 B |
| 实验组 A | 只上 A |
| 实验组 B | 只上 B |
| 实验组 C | A + B |

组数越多，能区分的策略组合越细，但每组要分到的流量也越多、对样本量要求更高。拆到几组取决于你想区分哪些组合、以及流量是否够分。

> 经验取舍：能用三组说清"谁贡献更大 + 叠加是否划算"就别盲目上四组；只有当 A、B 单独效应都需要独立结论时，四组才值得。

### 2. 对照组和实验组的流量比例怎么定

**流量比例之所以重要，是因为它直接关系到能否达到最小样本量。** 最小样本量的含义是"每组各自要达到的最小样本量"，所以分流方案要保证**每一组都能攒够各自的最小样本量**。

**算例（中性造数，仅示意逻辑）：**
假设实验计算后得到的最小样本量是每组 100w。

- 如果按 **1:1** 分流，对照组、实验组都能拿到 ≥100w，达标。
- 如果按 **1:2** 分流，实验组 200w、对照组只有 100w；两组都仍然 ≥100w，所以 1:2 **也可以**。

也就是说，只要每组都过了各自的最小样本量门槛，比例不必死守 1:1。

**那为什么实务里默认还是 1:1？**
从实验检测的角度看 1:2 是没问题的，但如果只是为了满足最小样本量而把某一组分得过大，会**白白多占流量、造成流量浪费**。在"恰好满足每组最小样本量"的约束下，两组样本量往往会**自然收敛到 1:1**——这是最省流量、又能保证检验功效的分法。

> 一句话记法：1:1 不是规定，而是"每组都不浪费"下的自然最优解；只有当某组有特殊需要（如新策略想多观察）才偏离 1:1，且偏离后仍要保证小组达标。

### 3. 留存率怎么算口径（实验场景）

留存率指标在实验里有两个关键口径选择。

**1）按"进组相对天数"对齐，而不是按自然日历天。**
建议**按用户进组的相对时间去计算**留存：以"每个用户进入实验组的那一天"为各自的第 0 天，再看其后第 1~N 天是否回访。这样每个用户都对齐到自己的进组起点，口径一致。
（对比：如果按自然日历天统一对齐，先进组和后进组的用户被观察的窗口长度不一样，留存口径会被进组时间分布污染。）

**2）刻意拉长指标窗口，淡化新奇效应。**
"新奇效应"指功能刚上线时用户因好奇而短暂活跃、并非长期真实偏好。要排除它的影响，可以**把指标刻画的天数拉长**——例如更关注较长周期的累计留存，而不是只看头一两天。在很多业务场景里，**更长周期的留存往往更贴合真正想达成的目标**（留得住才是真有效，而非一时新鲜）。

> 落地提醒：留存这类"对齐起点 + 看长期"的指标，相对天数对齐 + 拉长窗口两步要一起做，单独做任一步都可能被进组时间或新奇效应带偏。

### 4. 转化率怎么算口径：按自然日历天 vs 按进组相对天

这是一个高频纠结点：要不要为了排除新奇效应，去分别计算"用户接触新功能第一天的转化率""第二天的转化率"再往后递推，还是直接**按自然日历天**算就够了？

**结论：实务中通常直接按自然日历天算就够了，不必逐个相对天去拆。** 原因有二：

1. **新奇效应的影响其实没那么大。** 实验上线后，老用户往往在超过 70% 的占比上、于实验开始后的**第一个活跃天**就触发并进了组；也就是说绝大多数用户本就集中在"第一天"进组。因此"按进组相对天对齐"和"直接按自然日历天去算"两种方式得到的结果**差异往往不大**。
2. **按自然日历天的算法更通用、更易实现。** 自然日历天口径灵活性更高，数据平台/取数工具对它的支持也更好、去重等处理更顺手。所以一般情况下直接按天计算就能解决绝大多数问题。

> 边界：上面成立的前提是"用户进组高度集中在前一两天"。如果你的实验是缓慢放量、用户进组在时间上拉得很散，那相对天对齐才更有必要——先看进组时间分布再决定口径。

### 5. 跨指标的口径通用提醒

- **"算法越复杂越严谨"是误区。** 相对天对齐、逐日拆转化率这些更"精细"的算法只有在能带来实质差异时才值得；当用户进组集中、或长期指标已能反映目标时，简单的自然日历天口径反而更稳、更易复用。
- **新奇效应的两条应对路径要分清：** 留存类靠"拉长观察窗口"淡化；转化类靠"判断进组是否集中"决定要不要逐日拆。
- **设计阶段先问"我要回答什么问题"：** 多改动拆几组、流量怎么分、口径怎么定，最终都回到"想区分哪些效应、要排除哪些干扰"这一根本问题，而不是套模板。

## 第四部分 · 判读：灰度异动来自版本还是分流不均（含没做 AA 的补救）

没做 AA、灰度后指标异动时，不要急着下"版本有效/无效"的结论。先排除"两组人群本来就不一样"这个最大干扰：**回看用户进组前的行为，确认两组结构可比**；再结合**指标是比例型还是均值型、流量够不够、跑的时间够不够一个完整周期**来判断这个差异可不可信。核心是把"分流不均"和"版本效应"这两个搅在一起的因素拆开。（若已做过规范 AA、分流已确认无偏，可直接读差异，跳过本节的补救步骤。）

### 1. 问题长什么样

典型场景是这样的：实验来不及在上线前做 AA（即不改动任何东西、空跑两组看是否本就一致），直接灰度放量。放量后看渗透、留存、活跃这类指标，发现实验组和对照组有差异，还可能被追问"和上一版比变动了 8%"。这时候真正卡住的不是"有没有差异"，而是：

- 这个差异是**版本改动**带来的（说明功能真有效/真有害），
- 还是**两组人群一开始就分得不均**（说明差异是噪声，是分流问题伪装成了效果）？

没做 AA，等于少了一把"两组本来就一样"的标尺，所以必须用事后手段把这把标尺补回来。

### 2. 补救思路一：事后验证两组人群是否可比

既然事前没空跑，就用事后数据把"两组起点是否一致"验出来。两条路：

**1）回看用户进组前的行为数据（最直接、最该先做）。**
把这两个组里的用户拉出来，看他们**在进组之前那一段时间**的表现——重点看活跃度、人群结构（如年龄分布等画像维度）这类不受本次版本影响的"基线特征"。逻辑是：进组前发生的事和你这次的版本改动无关，如果两组连进组前都长得不一样，那进组后的差异就高度可疑，很可能是分流不均而非版本效应；反之，如果两组进组前高度一致、进组后才分开，差异才更可能真的来自版本。

> 关键点：选来对比的特征必须是**版本影响不到的**（进组前行为、固有画像），否则等于拿被污染的尺子去量。

**2）用倾向得分匹配（PSM）或事后抽样做同质对比。**
如果担心整体两组就是不均，可以用 PSM（俗称"点匹配"）或按特征做事后抽样：**挑出一批特征相同的人，在这批同质人群里再去比指标差异**。这样能把"人群结构不同"这个干扰压下去，剩下的差异更接近版本的净效应。

- 注意：PSM 这类方法在这种场景里更像"加一道保险"。多数情况下，分流平台本身分出来的两组就是同质的（顶多是分流不均，而非系统性偏差），**先做更轻的"进组前回看"往往就够用**，不一定要一上来就上 PSM。

### 3. 补救思路二：看指标是"比例型"还是"均值型"，分别处理

不同类型的指标，对"分流不均"的敏感度不一样，判断方式也不同：

- **比例型指标**（渗透率、留存率、活跃占比这类"占比/比率"）：
  对分流不均其实**比较稳**。只要实验组流量足够大，比率一般不会因为分流稍有不均就明显失真——除非分流平台本身有 bug，或者增量样本太小。所以比例指标看到差异时，**先确认每组流量够不够**，够的话这个比率差异通常可以采信。
- **均值型指标**（人均时长、人均消费这类"均值"）：
  对分布、对离群点更敏感，要**额外看方差**。如果方差很大，均值差异可能只是分布抖动，不能直接当成版本效应。

> 一句话：比例指标主要受"样本量够不够"约束；均值指标还要多看一道"方差大不大、分布稳不稳"。

### 4. 怎么读"相对提升 8%"这个数

被问到的"变动了 8%"通常是**相对提升（uplift）**，即相对上一版/对照组的相对变化，不是绝对值差。判断它可不可信，看三件事：

1. **是不是同期对照 vs 实验组算出来的？** 相对提升应当用**同一时间窗口内**的对照组和实验组去算，而不是拿实验组去和"上一个历史版本"比——历史版本和现在的外部环境、人群都变了，那种跨期对比的 8% 含金量很低。
2. **底层流量够不够大？** 对比例型指标，只要实验组流量足够（如几万、几十万用户级别以上），8% 这种量级的相对增幅一般不会是纯分流不均能凭空造出来的——除非分流平台有问题。流量越大，这个差异越不像噪声。
3. **小流量灰度要格外谨慎。** 当灰度只占全量的极小比例（如 1%~5%），增量样本本身就小，比率更容易抖；这时 8% 更可能是波动，需要靠前面"进组前回看 + 看流量是否达标"来兜底。

### 5. 流量与时间：两个容易被忽略的前提

**1）流量要先过最小样本量这一关。**
判断任何差异之前，先确认实验**有没有攒够最小样本量**。没达标时，看到的差异本身就不该解读。流量很大时，分流不均的影响通常可以忽略、不必过度纠结；流量本就一般甚至偏小时，分流不均才会变成真问题，要更小心。

**2）时间要跑够一个完整的行为周期。**
实验不能看一两天就下结论，要**积累到足够长的实验周期，通常至少一个完整的自然周（约 1 周）**。原因是用户行为有周内节律（工作日/周末差异等），观察窗口跨不满一个周期，指标会被"今天恰好是星期几"带偏。要让结果可比，得攒满相对完整的一个时间周期的数据。

### 6. 判读落地动作清单（按顺序做）

遇到"没做 AA、灰度异动、不知是版本还是分流"的情况，建议按这个顺序排查：

1. **先确认流量是否达到最小样本量、是否跑满约一个完整周期**——没达标就先别解读差异。
2. **回看两组用户进组前的行为与画像**（活跃度、人群结构等版本影响不到的特征），看两组起点是否一致。
3. **分指标类型判断**：比例指标确认流量够即可较放心采信；均值指标再补看方差/分布。
4. **必要时上 PSM / 事后抽样**，在同质人群里复核差异——作为加保险，不一定每次都要。
5. **确认"8%"是同期对照 vs 实验组的相对提升**，而非跨历史版本的对比。
6. **拿不准内部口径时，对齐团队既有惯例**：很多团队对"这种情况怎么判定、按什么规则走"已有内部约定，按约定走通常比自创一套更稳；差异大、确实存疑时，重做一次规范实验（含 AA）也是合理选择。

## 第五部分 · 决策：核心指标与护栏指标冲突时如何取舍

当一个策略让核心指标涨、护栏指标也涨时（典型如转化/时长/GMV 涨了，但投诉率、退款率、客诉等体验类指标也涨了），不要只看「净涨幅够不够大」就拍板，而要先**拆解护栏指标恶化的根因、判断它是否动摇了核心指标本身的可信度**，再用「收益—代价」的量化权衡和预设阈值做上线决策。（前提：核心指标本身已达显著、实验设计无偏；若核心指标未达显著或分流不均/样本污染，应先回到前面几部分做有效性校验，不进入本流程。）

### 1. 为什么不能只看核心指标涨幅

核心指标（如 GMV）是要推高的目标指标，投诉率/退款率这类是**护栏指标（guardrail metric）**——它们的作用是兜底，防止为了短期目标而伤害长期体验或平台健康。一个策略哪怕把目标指标推得很高，只要它是靠「踩护栏」换来的，长期往往不可持续。所以核心指标涨不等于策略成立，护栏一旦报警就必须先解释清楚再谈推全。

### 2. 第一步：先确认核心指标的涨是不是「真涨」

护栏指标的异常，常常是核心指标涨幅本身有水分的信号。要先把目标指标的来源拆开看，确认它是健康增长而非「虚涨」：

- **拆 GMV 的结构**：单价 × 件数（或 流量 × 转化 × 客单价）。如果涨幅几乎全来自单价或客单价被人为抬高（如诱导高价品、捆绑销售），而件数/真实需求没动，这种涨是脆弱的。
- **警惕诱导性玩法**：靠诱导点击的链接、强插的导购素材、加价钩子等把单量做上去，短期 GMV 会涨，但这类增长会同步推高投诉，本质是「拆东墙补西墙」。
- **结论**：如果核心指标的涨主要由这类不健康来源贡献，那么投诉率上涨其实是在揭穿这次增长的质量问题，应倾向不推全，或回炉改策略。

### 3. 第二步：拆解护栏指标（投诉率）上涨的根因

不要停在「投诉涨了」这个总量结论，按多个维度下钻定位是结构性问题还是局部噪声。可参考的拆解角度：

- **按用户群**：是全量用户投诉都升，还是集中在某一类人群（如新用户、低活用户、价格敏感人群）？
- **按渠道/来源**：投诉增量集中在哪个流量入口或场景？
- **按商品/品类**：在电商场景下尤其有效——把投诉率和 GMV 变化按品类对齐看，往往会发现投诉集中在某些「高增长但高投诉」的品类，说明问题是局部而非普遍。
- **按角色链路（商家—商品—用户）**：体验问题常常牵涉多方利益。比如新策略只对部分商家有利、损害了另一部分商家或中小商家的利益，导致他们或其用户产生不满；又如平台、商家、用户三方目标不一致，策略偏向了其中一方。
- **甄别「真投诉」与「薅羊毛式投诉」**：如果策略含大额优惠/补贴，可能吸引来一批本就冲着优惠来、用完即走的低质用户，他们的退款、投诉、钻空子会拉高投诉率，但这未必代表正常用户体验变差。要把这类异常人群单独识别出来，看剔除后核心用户的投诉率是否仍上升。

> 落到可执行的提问上：投诉用户和成交用户是不是同一批人？投诉到底针对的是策略本身，还是商家/商品的固有问题（只是被这次实验放大暴露）？

### 4. 第三步：量化权衡「收益」与「代价」并对照预设阈值

在根因清楚、确认核心指标是真涨之后，做量化取舍：

1. **预估推全后的规模影响**：把实验组的投诉上涨幅度按人群规模外推到全量，估算推全后大盘 GMV 的增量和投诉的增量分别有多大。
2. **设投诉率上涨的可接受阈值**：结合业务经验为护栏指标设一条红线（如「投诉率上涨不超过某个百分点」）。超出阈值即视为不可接受，无论 GMV 涨多少都不推全；在阈值内则可在「换样式/换钩子等更温和的版本」上再做一轮实验，复核增长是否能保留、投诉是否能压回。
3. **算清长期账**：护栏指标恶化的代价往往是滞后的（用户流失、复购下降、平台口碑），不能只用本期 GMV 的短期收益去对冲。

### 5. 决策路径小结

1. 核心指标显著上涨 → 否则先回到实验有效性，不进入本流程；
2. 拆核心指标来源，确认是健康增长还是诱导/虚涨；
3. 多维度拆投诉上涨根因，区分结构性问题 / 局部品类 / 异常人群；
4. 量化收益与代价，对照预设护栏阈值；
5. 通过 → 推全或灰度放量；不通过 → 改策略后再实验，或直接放弃。

## 常见坑 / 边界（全流程汇总）

**统计原理类**
- **不要只看绝对/相对差异就下结论**。差异方向再"好看"，P-value 不达标就不能宣称有效。
- **样本量太小时正态近似不成立**，此时应回到 T 检验，且要警惕功效不足导致的假阴性。
- **默认双尾**，除非有极强先验且能接受漏判反向效果的风险。
- **显著 ≠ 业务有价值**。统计显著只说明差异大概率非偶然，效应量是否值得推广要结合业务体量另做判断。
- 比例类指标方差直接用 $P(1-P)$ 即可，不要再去"估方差"绕弯路。

**样本量与功效类**
- **不设效应量直接开跑**：没定"至少要检测多大差异"，样本量无从算起，最常见的失败是流量给太少、跑出一堆"不显著"却无法解释。
- **功效不足却下"无效"结论**：样本量没达标就说改动没用，可能只是假阴性。判"无效"前要确认实验本身有足够功效。
- **中途偷看就停**：样本没攒够、看到一时"显著"就上线，等价于变相抬高了第一类错误率，结论不可靠。
- **效应量定得过乐观**：把预期提升填得很大会让样本量看起来很省，但真实提升达不到时实验照样检不出，等于自欺。宁可保守。
- **基线漂移**：基线转化率取自历史，若业务近期有大改动或强季节性，旧基线会失真，需用近期可比口径重新估。

**分流与 AA 类**
- **把"绝大多数用 UV"当铁律**：分流单元取决于"效果在哪个层级稳定"，不是默认值；指标层级要和分流层级对齐。
- **用 PV"凑显著"**：累计型指标在 PV 下达到显著更快，但用户级长期效应本质上不该用 PV，快是因为单元不对而非效果真大。
- **把 AA 当万能借口**：遇到异常先从策略、功能、用户行为找解释，别一上来甩给"分流有问题"；成熟稳定平台不必每次都做 AA。
- **误以为"不显著=证明相等"**：AA 想要的是"没有证据表明不等"，而不是"证明两组绝对相等"。

**口径与落地类**
- **"算法越复杂越严谨"是误区**：相对天对齐、逐日拆转化率只有在能带来实质差异时才值得；用户进组集中或长期指标已反映目标时，简单的自然日历天口径反而更稳、更易复用。
- **留存口径只做一半**：相对天数对齐 + 拉长窗口要一起做，单独做任一步都可能被进组时间或新奇效应带偏。
- **多改动盲目上四组**：能用三组说清"谁贡献更大 + 叠加是否划算"就别上四组；组数越多每组流量越紧。

**判读类**
- **把"没做 AA"当成不可补救**：可以用进组前回看、PSM、事后抽样事后补一把"两组是否同质"的标尺，不必直接放弃。
- **用被污染的特征去验同质性**：对比两组时只能用版本影响不到的特征（进组前行为、固有画像），用进组后的指标去验同质性等于循环论证。
- **小流量灰度硬下结论**：灰度占比极小时比率抖动大，uplift 这类数更可能是噪声，必须靠流量达标 + 进组前回看兜底。
- **均值指标只看均值不看方差**：均值型指标对离群点敏感，方差大时均值差异不可直接采信。
- **观察窗口不满一个周期就拍板**：行为有周内节律，跨不满一周的对比口径不稳。
- **"算法越重越对"的误区**：PSM 是保险而非起手式；多数场景先做轻量的进组前回看就能解决，分流平台正常时两组本就大体同质。

**决策类**
- **只盯净涨幅**：把「核心指标涨幅 > 护栏涨幅」当成可上线的充分条件，忽略了两者量纲、长期影响完全不同，不能直接相减比较。
- **不拆根因就下结论**：总量层面护栏涨了就否决、或核心指标涨了就推全，都属于在没看清结构时拍脑袋。
- **忽视利益相关方**：体验类指标的恶化常源于多方利益错位，只从单一视角（如只看平台 GMV）会漏判。
- **阈值拍脑袋**：护栏阈值若没有结合成本/收益结构和历史经验来标定，会让「量化权衡」流于形式。
- **决策本身才是产出**：分析师的价值不只在跑出实验结果，更在于把结果背后的内在逻辑讲清，从而支撑一个能被信服、负责任的决策——决策建议要落在「数据 + 逻辑」上，而非单凭结果数字。

## 全流程总览清单

1. **定目的与指标层级**：先回答"这次实验要回答什么问题、核心指标是用户级还是访问级"。
2. **选分流单元**：用户级长期效应用 UV，访问级即时效应可考虑 PV，并核对指标口径与单元对齐。
3. **估样本量**：用效应量、α、功效、基线四个旋钮反推单组样本量，换算成"流量 × 时长"并向上取整到完整周期。
4. **必要时做 AA**：对新平台/新分流逻辑或异常数据，先查量级均匀 → 查指标一致 → 必要时做纠偏；无成片显著差异、差异可解释即通过。
5. **设计落地**：多改动按是否相互影响决定合测/拆组；流量在"每组达标"约束下自然收敛到 1:1；留存按相对天对齐 + 拉长窗口，转化通常按自然日历天。
6. **跑实验并判读**：攒满样本量 + 完整周期后读差异；没做 AA 时先回看进组前行为、分指标类型判断、必要时 PSM 复核，确认差异来自版本而非分流不均。
7. **决策上线**：核心指标显著上涨且为健康增长 → 拆护栏根因 → 量化收益与代价、对照预设阈值 → 通过则推全/放量，不通过则改策略再实验或放弃。
