## 正文

### 一、什么时候不能做实验

随机分流实验通常是建立因果关系、评估策略效果的金标准，但下面这几类情况它做不了或不该做：

1. **法律 / 伦理约束**：有些干预天然不能拿人去随机分组。比如想验证「吸烟是否更易致癌」，不可能随机指定一组人吸烟、一组人禁烟。
2. **干预由自然 / 外部事件触发**：因果关系来自非人为可控的事件，如自然灾害、政策法案。比如「禁烟法案对香烟销量的影响」，研究者没法人为设置对照。
3. **效应显现周期太长**：有些影响要很久才看得出来，实验难以持续到那个时点。比如「不同教育方式对一个人的长期影响」，可能要数十年才显形。

这类场景只能退到**观测性因果推断**——在真实发生的观测数据里，尽量推断出干预的因果效应。

---

### 二、核心思路：用历史数据造一个「没发生这件事」的平行世界

一个典型业务问题：外卖平台搞了一段时间大促 / 大额补贴，想知道它对平台 GMV 有没有**长期**贡献、贡献多少。

- **理想做法**：随机分流——一半用户发券、一半不发，跑一段时间后比两组人均 GMV。
- **为什么做不了**：通常不能让一部分用户长期享受大额补贴、另一部分长期没有。对人群做差异化补贴会损害公平性，平台还要承担运营风险。

**折中方案**：策略以**全量**方式下发，但只在部分地区上线。例如选定某个大城市全量上大促，再用**没有上这套策略的其他城市**当对照，去**重建一条「如果该城市没上策略、本来会怎么走」的反事实曲线**。真实值与这条预测值之间的差（GAP），就是策略的收益。

> 一句话方法论：在不能做实验、又不得不给出收益数字时，用历史时序建一个模型，模拟「事件没发生」时该指标本应有的走势，再拿真实走势减去它，差额即为事件的因果效应。

---

### 三、模型骨架：贝叶斯结构时间序列（BSTS）

构造一个贝叶斯结构化时间序列模型，对目标时序做分解，形式上大致是：

```
y_t = 趋势项(trend) + 季节项(seasonal) + 回归项(β'·z_t) + 误差项
```

各模块含义：

- **趋势项**：目标时序自身的长期走向。
- **季节项**：周期性波动（如周内 / 季节规律）。
- **回归项**：一组**预测变量 / 协变量** `z_t`，可以是「未上策略城市的同期时序」，也可以是外生变量（如天气）。
- **误差项**：随机扰动。

模型模块是**可拆可配**的：时序模块（趋势、季节性）可根据目标序列特征和预测周期长短，灵活决定是否纳入。

---

### 四、完整流程（以「大城市上大促，参考同体量城市做对照」为例）

#### 1. 选评估城市与对照城市（事前探查）

- 上线前后都要做城市探查，**排除发生过剧烈变化的城市**（受疫情、自然灾害、强竞争冲击等）。
- 对照城市要与评估城市在**规模、体量**上相近。比如评估一个一线大城市，就用其他同级别大城市做参照，而不是拿小城市硬凑。
- 留意**外生变量**：例如某地区的特殊天气（回南天等）可能影响外卖单量，挑对照城市时要尽量让这类因素也可比，或把它纳入协变量。

#### 2. 划分 AA / AB 时间窗并校验精度

把时间轴切成几段，先验证模型可靠、再用它算收益。以「策略上线 3.01–3.07」为例：

- **选协变量（可选）**：纳入对预测精度有帮助的变量（如用户数量、对照城市时序）。
- **AA1 期（求解参数）**：取策略前更早的一段（如 1.14–2.14）来拟合、求解模型参数。
- **AA2 期（精度评估）**：用紧邻策略上线前的一段（如 2.15–2.28）检验模型精度——比对预测值与真实值是否存在显著 GAP。**经验门槛：MAPE < 5% 视为拟合达标**。
  - 达标 → 认为模型可用于估收益；
  - 不达标（存在 AA 差异）→ 说明对照城市选得不好或缺协变量，需**调整对照城市 / 增补协变量**，直到精度合格。
- **AB 期（评估期 / 真正要回答的观测期）**：策略生效后的一段（如大促结束后一周 3.08–3.14），在这段上比真实值与反事实预测值。

> 时间窗经验法则：
> - AA 期越长越好，**至少为 AB 期的 2 倍**，且尽量贴近 AB 期（越近越能代表当下规律）。
> - AB 期一般从策略上线日起算；但评估周期越长，模型误差累积越大，**建议评估周期不超过 1 个月**。

#### 3. 算策略收益

模型通过了 AA2 期精度检验后即视为可靠。AB 期内：

```
策略收益 = 评估城市真实指标 − 模型预测的反事实指标
```

可视化时，常画出真实值与预测值两条曲线，并给预测值附**95% 后验置信区间**（阴影带），直观展示效应是否「钻出」了不确定性范围。

#### 4. 向上汇报时如实标注口径

因为结论不是来自严格随机实验、模型本身带误差，汇报时要**带上评估方法说明**，例如：「策略 A 的 GMV 显著 +5%（全量观测评估）」，而不是把它说成实验级别的精确因果结论。

### 五、优点与不足

**优点**

- **适用面广**：适配各类时间序列——销量、流量、社媒活跃度等，跨领域可用。
- **兼顾长短期**：能刻画事件对时序的长期与短期影响，效应评估更完整。
- **贝叶斯框架可量化不确定性**：给出的是带置信区间的概率化估计，比单点数字更可解释、更可信。
- **窗口灵活**：可自定义事件时间窗、事件前后的控制期，适配不同研究问题。

**不足**

- **对数据质量要求高**：需要足够长、噪声小的时序，且最好没有其他强烈的、未被建模的趋势 / 相关性干扰。
- **强依赖对照组假设**：对照对象选得不当会直接带来偏误——这是结论可信度的命门。
- **解释要谨慎**：即便给了概率估计，也不能轻易宣称是铁板钉钉的因果，仍需结合业务领域知识与其他证据综合判断。

### 六、实操要点

- **现成库可调**：Python 与 R 都有封装好的 Causal Impact 库，使用中可自由配置是否加协变量、是否纳入季节项等。
- **接口约定**：库里通常把 **AA 期作为 `pre_period`、AB 期作为 `post_period`**，传入数据与两段区间即可拟合、运行、出图。常见可视化三联图：
  - 图一：预测值 vs 真实值的走势对比；
  - 图二：两者的逐点差异；
  - 图三：策略的**累积差异**（把每日效应累加，看总收益）。
- **官方算例往往略过精度检验**：很多入门示例直接给 `pre_period` / `post_period` 跑结果，没有 AA 精度校验环节。实操中可以**把 AA1 期当成 `post_period` 反过来跑一遍**，得到模型在「本应无事件」时段的 AA 差异，以此补上精度检验。

最简伪代码示意（区间为造数示例）：

```python
from causalimpact import CausalImpact   # 库名以实际安装为准

# data：第一列为目标序列 y，其后为协变量（如对照城市序列 x0, x1）
pre_period  = [0, 69]    # AA 期：拟合 + 精度校验
post_period = [70, 99]   # AB 期：评估收益

ci = CausalImpact(data, pre_period, post_period)
ci.run()
ci.plot()   # 输出走势 / 逐点差异 / 累积差异三联图
```

> 造数示例（仅演示，不含真实业务数据）：令对照序列为 `x0, x1`、扰动为 `z`，构造实验组 `y = 1.2*x0 + 0.9*x1 + z`，并在后段人为叠加约 +10 的提升，跑出来应能在 AB 期检出这个提升。

### 七、把方法思路抽象到没有建模条件的场景

建模不是「一招打天下」，它只是在**业务成熟度、人员能力**到位时的一个阶段性答案。同一套**反事实**思路完全可以降维落地：

- **有建模条件时**：用模型从数理上证明「数据差异为什么会影响结果」，给出可量化、带置信区间的收益。
- **没条件时（小公司 / 小部门，复盘只给两三天、没有完善实验基础）**：本质思路一样——**挑一个合适的可比样本**做对照，挑选时同样要考虑季节、趋势、历史上是否做过类似优惠等混杂因素，最后做一个「事件前后核心数据的朴素对比」，有效就是有效、没效就是没效。这是很多业务团队在没有实验条件时用的「土办法」，但骨架与 BSTS 一致。

**落地心法**：方法能拿进业务复盘汇报、起到数据部门该有的作用；但若换个环境出现「水土不服」，不要硬套模型，而要**回到方法的本质（构造可比的反事实、用差额估效应）**，据此设计一个更贴合当前业务、人员、时间约束的折中方案。

### 八、常见坑

- **把对照选错**：对照对象不可比（体量差太多、期间发生剧变），整条反事实基线就是错的，收益数字也错。这是最致命的坑。
- **跳过 AA 精度检验直接算收益**：模型还没证明能拟合「无事件」时段，就拿去算 AB 期 GAP，等于拿一把没校准的尺子量东西。
- **评估期拉太长**：周期越长误差累积越大，建议控制在一个月内，长期效应可分段评估而非一口气拉到底。
- **把模型估计当成实验级因果**：观测推断带误差，汇报时必须标注「全量观测评估」口径，区别于随机实验结论。
- **忽略外生变量**：天气、节假日、突发事件等若未纳入或未在对照里平衡，会被误算进策略效应里。
