## 第一部分 · 双边平台与网络效应：对象互相影响时怎么设计

### 核心问题：对象互相影响时，按用户分流会有偏

标准 AB 默认一条关键假设：**给一个对象施加干预，不会影响到别的对象的结果**（即处理无溢出，SUTVA 成立）。但在双边平台上这条常被打破——供需是耦合的。

> 中性示例：某出行平台给一半司机上了新派单策略。被「优待」的实验组司机抢走了更多订单，对照组司机的接单量、乘客的等待时间也随之变化。此时对照组已经被实验组「污染」，两组之差既含真实效应、又含溢出效应，直接对比会高估或低估，结论不可信。

应对思路有两类：要么**承认溢出并把它估出来**（饱和度设计），要么**换一个不会互相污染的分流单元**（按时空切片、按时间轮换等）。下面四种是常用武器。

### 一、随机饱和度实验：把溢出效应直接估出来

适用：用户在局部（地理/时间）会互相影响，但能切出彼此独立的簇。

做法：

1. 把人群切成若干**互相独立的簇**（如地理区域、时间块）。
2. 给每个簇**随机分配一个「饱和度」**——簇内多大比例的对象接受干预（如 0%、25%、50%、100%）。
3. 簇内再按该饱和度随机给个体分组。

怎么评估：若同时存在 0% 与 100% 的簇，直接比这两类簇即得无偏的处理效应；若铺了一整条饱和度梯度，则用「把饱和度作为交互项」的回归，既估出处理效应、又量化溢出强度。还能专门看溢出：比较「高饱和簇里的对照个体」（被很多邻居的干预波及）与「零饱和簇里的对照个体」（没被波及），两者之差就是间接溢出效应。

### 二、Switchback 实验：干预落不到单人时，按时空轮换

适用：干预是**全局策略**、没法拆到单个用户头上（如撮合算法、动态定价、全局调度）。

做法：把**时间和空间切成小块（时空切片）**，在切片层面随机地、交替地切换「用对照策略」还是「用实验策略」——例如某区域 10–11 点用 A、11–12 点用 B，相邻时空像棋盘格一样交替。

怎么保证不互相污染：度量「跨切片率」（有多少撮合/交互发生在切片边界之间），跨切片越少越干净。空间边界常用「以地理格为节点、以撮合关系为边」建图再做谱聚类，找一刀切下去跨界撮合最少、同时样本量又够的划分；时间上用固定时长（如按小时/按天）并检查上一段是否「带飞」下一段。

怎么评估：以时空切片为单位、控制住「星期几/几点」等时段固定效应，回归出处理项；并用 A/A（两边都跑同一策略）验证切片确实独立。

### 三、交叉实验：让同一对象先后承受不同策略，自己做自己的对照

适用：有多个可逆的小改动、时间或样本有限，希望靠「同一对象前后对比」消掉个体间差异。

三个要素：**顺序**（先用哪个后用哪个）、**周期**（每个策略持续多久）、**消除期/洗脱期**（两次切换之间留一段间隔，让上一个策略的延滞效应衰减掉）。

常见设计矩阵：

- **2×2 交叉**：两组分别按 A→B 和 B→A 跑两段；
- **拉丁方**：k 个策略排成方阵，每个策略在每个「顺序」「周期」里各出现一次；
- **强平衡设计**：在拉丁方基础上，让每个策略「跟在其他策略（含自身）之后」的次数也相等，最大限度压制延滞偏差。

评估：拟合含「周期 + 顺序 + 策略 + 延滞（前一策略×当前策略）」的模型做方差分析。误差主要来自三处——周期误差（策略在各周期不平衡）、顺序误差（在各顺序不平衡）、延滞误差（上一策略残留污染当前），靠平衡设计 + 足够长的洗脱期来压。

### 四、多基线实验：强约束下不能分组时，错峰上线

适用：**强约束**场景——分组在伦理/公平/操作上不可行（如教育里不能让一部分学生用更差的方法），或干预效果持久、轮换会因「上一状态遗留」而混淆。

做法：

1. 招募少量对象（如三到六个）。
2. 先让所有对象在「基线期」只测量、不干预。
3. **给每个对象随机分配一个不同的干预起始时间，错峰开启**；每个对象只从对照切到干预一次，之后保持干预，不分组、不轮换。
4. 干预前后持续测量。

评估：每个对象比「干预后均值 vs 干预前均值」得到其效应；因为样本极小，用**随机化检验（置换检验）**而非常规 t 检验——在「无效应」的原假设下反复打乱干预时点、重算统计量，构造零分布，再看真实统计量的位置得出 p 值。必要时先用协变量扣掉一个已知混杂，减小残差、提升把握度；多个对象的结果按重要性等权或加权合并。

### 怎么选（速查）

- 个体局部互相影响、能切独立簇 → **随机饱和度**（顺带量化溢出）。
- 干预是全局策略、落不到单人 → **Switchback**（注意跨切片率与时间带飞）。
- 多个可逆小改动、想用前后对比消个体差 → **交叉实验**（务必留洗脱期）。
- 不能分组（伦理/公平）或效果持久 → **多基线**（错峰上线 + 置换检验）。

## 第二部分 · 提高实验效能：用随机区组与协变量调整缩减方差

### 一句话方法论

能不能跑出显著，取决于「真实效应」「指标噪声」「样本量」三者的拉锯。**真实效应没法改、样本量常受限，那就从降噪声下手**：把指标里那些「跟实验无关、但能被其他变量解释」的波动剥离掉，分母变小，同样的效应就更容易被检出——相当于用更少样本拿到同等灵敏度。

### 先理解三者的关系（为什么缩方差有用）

显著性检验大致是「效应大小 ÷ 噪声」的比拼。在要检出的效应（最小可检测效应 MDE）固定时，**指标方差越大，所需样本量越大**；反过来，方差若大而样本又凑不够，一个真实存在的效应也可能被噪声淹没、跑不出显著。缩减方差就是缩小这个比值的分母，让灵敏度上去，而不必硬堆样本量。

### 手段一：随机区组设计（分层后再随机）

核心思想：在随机分流**之前**，先按某个与指标相关的特征把人群切成若干个「内部更同质」的层（区组），在**每一层内部**都按相同比例分配实验组和对照组。

它为什么降噪：总波动可以拆成「层与层之间的差异」+「层内部的差异」。区组设计把「层间差异」当作已知的结构因素单独扣除，不再混进随机误差里，于是用来做检验的「层内残差」更小，效应就更突出。

> 中性示例：某电商做客单价实验，客单价本身在「高消费/中消费/低消费」用户间差异极大。先按历史消费档把用户分三层，每层内部各自一半进实验组一半进对照组。分析时把「消费档」作为已知因素扣掉，剩下的残差波动明显变小，原本跑不动的效应变得可检出。

### 怎么挑区组变量（关键且容易做错）

一个特征适合当区组变量，要同时满足：

- **与结果指标确有相关性**：能用历史/实验前数据，通过回归或方差分析验证它对指标有预测力。相关性越强，扣掉它降噪越多。
- **不受实验干预影响**：必须是分流前就定下、不会被处理改变的变量（否则会把效应本身也扣掉，引入偏差）。
- **彼此别高度共线**：几个候选变量互相高度相关时不要硬塞一堆，冗余会让模型估计不稳。

### 随机区组实验的流程

1. **定目标与指标**：说清干预、要看的结果指标、什么算成功。
2. **带区组做设计**：用业务直觉 + 历史数据回归，筛出与指标相关、且不受干预影响的特征作区组；用「缩减后的方差」重算所需样本量；在每层内分层随机分配。
3. **校验分流**：用实验前数据做一次 A/A 式检查（回归里同时放入「分组」和「区组」），确认每一层内部实验组对照组确实均衡。
4. **按方差分解出结果**：拟合一个含「分组 + 区组」的回归，把总波动拆成「处理效应 + 区组效应 + 残差」，对处理项做检验。因为区组效应被单独扣走，残差比同等数据上的简单 AB 更小，更易显著。

### 手段二：协变量调整（用实验前数据扣噪声）

把「**同一指标在实验开始前的历史水平**」作为协变量放进回归。每个个体实验后的高低，有很大一部分其实是它实验前就有的基线差异（高消费用户实验后客单价还是高）。把这部分用实验前数据解释掉，残差进一步变小、灵敏度再升。这与「区组」是互补的两种降噪杠杆，常可叠加使用。

### 其他降噪杠杆

- **只分析被触发的人群/时段**：只看真正接触到该功能、或用户真正活跃的时间窗，把大量「根本没机会受影响」的背景噪声排除掉。
- **换一个本身更稳的指标**：在能回答同一业务问题的前提下，挑波动天生更小的衍生指标，比直接用原始噪声指标更易检出效应。

## 第三部分 · 验证策略对长期留存的负向影响：实验设计与观测推断

### 一、问题长什么样

把它抽象成一类常见的商业化 / 体验权衡题：

> 假设「给用户多展示一种打扰更强的曝光（例如跳出当前内容、把人导去外部 App 的广告），会让用户的长期使用时长 / 留存变差」。这个假设怎么验证？影响有多大？

难点有两个，决定了方法选择：

1. **结果指标是「长期」的**——长期使用时长、长期留存，不是当天点一下就能看出来，要观察一段时间才显形。
2. **干预本身有代价**——这类打扰多展示一次，可能当场就掉一点体验/留存，是「明知会亏一点」的动作。所以不能无脑大流量、长周期地开着测。

下面三条路线，按「能不能做随机实验」分层，可单用也可组合互证。

### 二、路线一：标准 AB 实验（能随机时的首选）

**核心思路**：对同一批被曝光人群随机分桶，只改「曝光的是哪种广告」，比两桶的长期使用时长 / 留存。

- **A 桶**：曝光「打扰型 / 外跳类」广告；
- **B 桶**：曝光「留在内部、不外跳」的广告；
- **对比指标**：两桶用户在观察期内的平均 App 使用时长、留存率等长期指标。

**为什么这样切**：把「广告类型」作为唯一变量，其余随机均衡，差异就能归因到广告类型本身。

**会被追问的两个点（也是设计要点）**：

1. **A、B 的成本/收益是否可比？** 如果两种广告本身的变现单价、填充率差很多，单看体验指标会失真——你以为在测「体验差异」，其实混进了「商业价值差异」。处理上要么挑成本相近的两种广告对比，要么把商业收益作为护栏指标一起看，做「体验 vs 收益」的权衡，而不是只盯一个指标下结论。
2. **样本量与实验时长怎么定？** 长期指标的效应往往偏小，需要先按最小可检出效应（MDE）和基线波动估算最小样本量；周期则要覆盖到「长期」效应能显形的时间窗，不能只看开实验头几天。但又因为干预有负向代价，周期不能无限拉长（见路线三）。

> 边界：AB 是金标准，但前提是你**能**对线上流量随机分桶。能随机就优先 AB，下面两条是「不能随机」或「想交叉验证」时的补充。

### 三、路线二：观测数据找相关 + 倾向得分匹配（不能随机时）

没法做随机实验、只有历史观测数据时，退而求其次先看相关，再尽量去掉混杂。

**第 1 步：先看相关方向。**
以「外跳类广告点击 / 外跳类广告点击占总广告点击的比例」为横轴，以长期使用时长 / 留存为纵轴，看散点和趋势：

- 公式上把它拆开理解——某用户的广告点击 = 内部广告点击 + 外跳广告点击；外跳占比 = 外跳点击 ÷ 总点击。
- 先看「外跳占比高的人」和「长期活跃 / 长期使用时长」之间是不是负相关。

**第 2 步：这只是相关，不是因果，要警惕反向与混杂。**
散点能看出关联，但有两个陷阱：

- **反向因果**：可能不是「外跳广告拖垮了留存」，而是「本来就快要流失、黏性低的人，才更容易点外跳广告」。两个方向都能产生同样的负相关。
- **混杂**：重度用户和轻度用户本身行为差异大，直接比不公平。

**第 3 步：用倾向得分匹配（PSM）逼近实验组/对照组可比。**
思路是用可观测特征建模，估计每个用户「会多看 / 多点外跳广告」的倾向概率，再按这个概率把「高外跳暴露」与「低外跳暴露」的相似用户匹配成对，让两组在协变量上尽量同质，再比长期指标。这样能部分剥离「人本来就不一样」带来的偏差，让相关结论更接近因果。

> 这里 PSM 的角色：在无法随机分组时，**用倾向概率把不可比的两群人匹配成「近似可比」**，是观测研究里降混杂的常用手段；但它只能控住「可观测」的混杂，控不住没测到的变量，结论强度低于真随机 AB。（PSM-DID 的完整建模流程与可靠性检验见同板块《倾向得分匹配(PSM-DID)》。）

**配套手段**：也可以做小流量 AB 来交叉验证观测结论，并把「哪些用户的活跃/收入与外跳占比反向最明显」作为后续策略（如对高敏感人群屏蔽该类曝光）的依据。

### 四、路线三：小流量、短周期的实验设计（干预有负向代价时的关键技巧）

当干预**明显有负向影响、会亏钱**时，实验设计要专门为「少亏、还要拿到可信结论」服务：

- **只用小流量测**：因为这是已知有害的动作，不能大面积铺；
- **不能长期跑**：长期开着就是持续在损失（掉留存 / 掉收入）；
- **目标是「达到统计置信所需的最小样本量就停」**：算出在期望的显著性与功效下、最少需要多少流量/多长时间能把效应测出来，到点即收，把代价压到最低。

**结合业务背景做更细的分组（以推荐/广告漏斗为例）**：
若业务里广告经过「召回 → 粗排 → 精排 → 竞价 → 混排」这样的链路，分组可以更贴合机制：

- **对照组**：只投不外跳的内部循环广告（保持现状）；
- **干预组 A**：只投外跳广告（最极端，看上限影响）；
- **干预组 B**：在召回或混排环节，通过素材多样性/探索类策略人为控制外跳与内部广告的混入比例，做更接近真实策略的中间档。

**面试 / 汇报时可加一层「成本-收益」测算**：在一定流量损失下，每天因此少了多少（或多了多少）「内循环广告」、又多带来多少「外循环广告」，把体验损失换算成可比的商业账，让权衡有数。

### 五、方法选择小抄

| 场景 | 首选路线 |
|---|---|
| 能对线上流量随机分桶 | 路线一：标准 AB |
| 只有历史观测数据、不能随机 | 路线二：相关分析 + PSM，必要时小流量 AB 互证 |
| 干预明知有害、不能大跑长跑 | 路线三：小流量 + 短周期，按最小置信样本量即停 |
| 两种广告商业价值不对等 | 任何路线都要把「收益」设为护栏指标，做体验-收益权衡 |

## 常见坑 / 边界（三类场景汇总）

**双边平台 / 网络效应类**
- **「独立簇/独立切片」是前提，也是最难的**。簇之间、切片之间若仍有外溢，估计照样有偏；跨切片率高就别硬上 Switchback。
- **延滞/带飞效应**普遍存在：交叉实验洗脱期不够、Switchback 时间块太短，都会让上一段污染下一段。
- **样本量与统计方法要配套**：多基线样本极小，必须用随机化检验，套常规检验会失真。
- **复杂设计成本高**：能用普通 AB 解决就别上这些；它们是「普通实验失效时」的补救，不是默认选项。

**提效降方差类**
- **区组不是越多越好**。多加一个区组，只有当它的解释力足够大、能盖过「损失的残差自由度」时才真正降误差；预测力弱或与已有区组高度相关的区组，不但不降噪，还可能引入共线、让估计变得不稳。
- **别拿 R² 当目标**。R² 高确实说明区组/协变量在起作用（模型解释了更多波动），但它**不能证明处理效应为真**；真正要看的是处理项的大小与显著性。哪怕 R² 不高，只要效应被稳定检出就行。
- **方差分析有前提**。它依赖回归的常规假设（变量间独立、与残差独立、各组方差齐性、残差近似正态）。对转化率、是否点击这类非连续指标，要换合适的模型，不能照搬。
- **缩方差治不了设计病**。如果分流本身有偏、指标口径有问题，再怎么降噪也只是把错误算得更精确；先保证设计干净，再谈提效能。

**长期负向影响类**
- **只看一个指标**：商业化-体验类的题，单看体验或单看收入都会得出片面结论，必须双指标权衡（一个当核心、一个当护栏）。
- **把相关当因果**：观测数据里的负相关，可能是反向因果（快流失的人才爱点这种广告）或混杂导致，PSM 能缓解但不能根治。
- **周期太短测不出长期效应**：长期指标头几天看不出来，但又不能为了等效应而长期承受负向代价——这正是「小流量 + 最小置信样本量即停」要解决的矛盾。
- **A/B 两侧不可比**：两种广告成本、填充率差太多时，体验差异会被商业差异污染，要么挑可比对象，要么显式控制/校正。
