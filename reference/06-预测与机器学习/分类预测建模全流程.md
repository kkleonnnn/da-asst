## 正文

### 一句话方法论

一个完整的分类预测任务，按「**明确目标 → 数据采集与清洗（构建数据集）→ 特征工程与筛选 → 模型训练与评估 →（可选）多模型融合 → 方案落地**」推进。贯穿始终的一条铁律是：**数据集的构建（清洗 + 特征工程）往往比换模型更能决定效果上限**——「特征工程决定模型上限，调模型只是逼近这个上限」。

### 一、先分清：数据分析 vs 数据挖掘

两者目的不同，理解差异有助于判断该用哪条路。

- **数据分析**偏向「发现问题、解释问题、给方向」，产出多是宏观的指导建议。例如「某渠道新增用户的次留最低，疑因新手引导曝光点击转化率低，建议拆渠道加强引导」——结论作用在群体和策略层面，并不会对每一个具体用户产生直接动作。
- **数据挖掘**偏向「预测趋势与个体行为，支撑可落地的决策」。它能对**每一个个体**给出反应，例如「模型预测某新用户当前流失概率超过 80%」「聚类发现产品内存在若干自然形成的社群」。
- 二者关键差别：数据挖掘多出了**预测、聚类、检测**等能力，能下沉到微观个体，从而直接驱动动作。例如有了流失概率预测后，可对「预测流失概率 > 阈值」的用户定向发放权益挽留——这是单纯出分析报告做不到的。

### 二、明确目标

动手前先想清楚要解决的是什么类型的问题、模型要实现什么效果，因为这直接决定后面选什么模型和算法。

- 练手项目的目标通常已给定（如「输入一位个体的信息，判断其是否属于正类」），相对明确。
- 真实业务里这一步要自己判断：是分类还是回归？预测对象是个体还是聚合量？指标怎么定？这些都会反向约束建模方案。
- 进入一个对标竞赛形式的项目时，除任务本身外，还必须额外确认四件事：**时间线**（各类截止节点，组队尤其要盯紧）、**提交 / 输出格式**（如对每个样本 id 输出一个类别概率、带表头的 CSV，格式错一点就可能整份判废）、**评分指标**（如 AUC，它决定优化方向）、**特殊约束**（如「效率奖」鼓励更轻量省算力的模型，会影响方案取舍）。

### 三、构建数据集（采集 + 清洗）

练手数据通常已较完整，但真实数据脏且缺，需要预处理才能「喂」给模型。

**3.1 特征编码：把非数值特征变成可运算的数**

- 原始数据里很多列不是「数」，而是字符串或编码（如性别、地区、票号），模型无法直接计算，必须把每个维度都转成数学上可运算的形式。
- 入门最直观的方式是 **one-hot 编码**：把一个不可直接运算的类别特征拆成若干列，每列代表一种取值（一种「状态」），用 0/1 表示该样本是否命中。
  - 例：性别原本一列 `male/female`，拆成「是否 male」「是否 female」两列。某个 `male` 样本就编码为 `是否male=1, 是否female=0`。
  - 在 pandas 里可用 `pd.get_dummies(data[features])` 一步生成。

**3.2 缺失值处理**

有选择地对重要维度补缺失值。常见做法由简到精：

- **简单填补**：用默认值、均值、中位数、众数填补。连续变量用均值 / 中位数，类别变量用众数，缺得太厉害的列用占位默认值。
- **借助先验知识填补**：比单纯填均值更准。例如年龄缺失，可先从姓名里的称谓（如不同称谓对应成年男性 / 未婚女性 / 男童）推断年龄段，再在该段内用中位数填补。
- pandas 中 `df['列'].fillna(...)` 可直接填充。

**3.3 顺手要会的其他预处理手法（按需用）**

- **归一化**：把数值压到 [0,1]，如 `MinMaxScaler`。
- **标准化**：转成均值 0、方差 1，如 `StandardScaler`。
- **分箱**：把连续变量切成若干区间（如 `pd.cut`），降低噪声、便于刻画非线性。
- **数据变换**：对长尾 / 偏态变量做对数（`np.log1p`）、平方根变换，压缩量纲、改善分布。
- **异常值处理**：用箱线图或 Z-score 检测离群点；常用 IQR 法把超过 `Q3 + 1.5×IQR` 的值截断（封顶）或剔除。
- **文本特征**：可用词袋、TF-IDF 等；也可从文本里抽取结构化特征（如用正则从姓名提取称谓再 one-hot）。
- **时间特征**：从时间字段拆出年 / 月 / 日 / 星期等可建模的维度。

### 四、特征筛选（编码完不能直接全扔给模型）

一个常见误区：把所有变量都编码后一股脑喂给模型，「维度越多越好」。这会带来两个问题：

**4.1 过拟合**

过拟合指模型在训练集上表现很好，但对没见过的样本预测一般。当选入的特征过多、且与目标相关性不强时容易发生。

- 反例：把「姓名」做成特征，one-hot 后变成「是否张三」「是否李四」…… 模型学到的其实是「叫张三、李四的人会生还」。来一个训练集里没有的「赵四」，姓名特征对他就是纯噪声——**有还不如没有**。

**4.2 维度膨胀与稀疏**

特征越多，模型越大、训练代价越高；若特征太稀疏（大量 0/False），训练和预测性能都会很差。实际应用应尽量精简特征。

**筛选特征的两类方法：**

1. **业务 / 统计直觉 + 透视分析**：先把数据丢进透视表，按单个特征分组看正类比例，找有区分度的维度。例如在某个体存活预测里能观察到：女性正类率高于男性；中青年略高于老人和儿童；等级越高、价位越高正类率越高；带 1–2 个同行者的正类率最高，同行者过多反而下降。进一步还能发现特征间的相关，如编号字段与等级正相关（只有高等级才有编号）、同行者数与年龄正相关——这些重复信息提示可以降维或重构特征。
2. **统计检验量化相关性**：逐个看变量太慢，可用卡方检验、皮尔逊相关系数等批量衡量。把变量两两相关系数画成矩阵，就能看出：哪些变量与目标相关性强（该留）、哪些彼此高度重复（易引发过拟合，可降维或合并）。代码上可用 `SelectKBest(chi2, k=...)` 计算每个 one-hot 特征与目标的卡方值——**卡方值越大、相关性越强**，据此选重要特征。

### 五、模型训练与评估

**5.1 先想清模型在做什么**

分析阶段心里其实已有粗结论（如「某类画像的人更可能是正类」）。模型的价值在于给出**联合概率**：当一个个体同时具备「中年、女性、中等级、带同行者」等多个属性时，他属于正类的概率到底是多少——这种多特征交互下的个体概率，是单纯分析回答不了的。

**5.2 训练（以随机森林为例）**

入门可先用随机森林等开箱即用的分类器，调用现成工具包即可：初始化（设定树的数量 `n_estimators`、最大深度 `max_depth`、随机种子等）→ `fit` 训练 → `predict` 输出每个样本的 0/1 标签。

**5.3 必须切验证集**

练手测试集通常没有标签，光预测不知道对不对、也无从优化。做法类似实验对照：把有标签的训练数据再切成**训练集 + 验证集**（如 8:2，`train_test_split(..., test_size=0.2)`），用验证集上的表现作为调优依据。数据量小时更推荐用**分层抽样的 K 折交叉验证**（如 5 折，保证每折正负样本比例一致），评估更稳健、更能防过拟合。

**5.4 四个常用评估指标**

先定义四种预测结果（以预测正类为视角）：

- TP：预测 1、实际 1（对）
- FP：预测 1、实际 0（错，误报）
- FN：预测 0、实际 1（错，漏报）
- TN：预测 0、实际 0（对）

| 指标 | 公式 | 含义 | 注意 |
|---|---|---|---|
| 准确率 Accuracy | (TP+TN)/(TP+FP+FN+TN) | 整体预测对的比例 | **样本极不均衡时失真**：10 万样本里 99999 正、1 负，全猜正也有 99.999% 准确率，却毫无用处 |
| 精确率 Precision（查准率） | TP/(TP+FP) | 我预测为正的里面有多少真的正 | 关注「别误报」时看它 |
| 召回率 Recall（查全率） | TP/(TP+FN) | 所有真实正样本里被我抓出来多少 | 关注「别漏报」时看它 |
| F1 | 2×Precision×Recall/(Precision+Recall) | 精确率与召回率的调和综合 | 两者都想兼顾时看它 |

不同业务关注的核心指标不同，需先选定一个主指标，再反复「训练—调优」迭代。（部分场景直接用排序类指标，如 AUC。）

**5.5 调优实例（特征 + 参数两条线）**

- 初版把所有觉得相关的特征都放进去训练，结果约为：准确率 0.777、F1 0.667，**召回率偏低**是主要短板。
- **优化特征**：注意到「编号」字段与价位、等级高度正相关（信息重复，易致过拟合），去掉该特征后，准确率升到约 0.788、F1 升到约 0.716——效果明显提升。说明**数据集 / 特征工程的作用极其关键**。
- **优化参数**：数据集本身较小，随机森林不需要太深，把树的最大深度从 5 降到 3，效果再次提升（降低过拟合）。
- 一般规律：后续优化主要围绕**特征工程**（用业务知识构造更多关键特征，收益最大）和**模型参数**展开；对数据从业者而言，算法层面的改进空间通常有限。

### 六、专题：文本分类的完整打法

文本分类是分类预测的一个高频专题（如判断「人写文本 vs 机器生成文本」、情感正负面）。除上面的通用流程外，它有几处独特套路。一个值得记住的结论：**在某些文本判别任务上，经过精心特征工程的传统机器学习方案，反而可能跑赢直接微调大模型的方案**——不要盲目堆大模型。

**6.1 数据量小时：扩增训练数据**

当官方 / 现成标注数据非常少（如标注样本仅一千多条、而待预测样本近万条），这是一个强烈信号——任务在变相要求你自己造 / 找训练数据。

- **造数据**：用提示词工程，调动多个不同的大模型（混用不同家、不同规模的模型，使生成文本风格更多样）批量生成带标签样本。
- **找开源数据**：很多公开平台的贡献者会主动整理并共享数据集，把多个来源汇总可显著扩大规模。
- **导入时统一口径**：不同来源的列名、标签编码各不相同，导入时要逐个改名、对齐字段，并打上 `source` 标记便于追溯。
- **边界提醒**：当大家都用同一批公开数据和公开方案时，起跑线就拉平了；想再提分往往要靠自己额外的数据或独有 trick。

**6.2 文本数据探索（EDA）**

- **文本长度分布**：对比两类样本的文本长度直方图与平均长度，看是否有区分度。
- **n-gram 词频差异**：分别统计「只被 A 类用到」和「只被 B 类用到」的高频 1~5 元词组。这一步常能发现强信号——例如某类文本偏爱固定套话句式、或出现某类专属符号（如某类文本会用 emoji 而另一类按规则不允许出现 emoji，这本身就是强判别特征）。这些差异既加深对数据的理解，也能直接转化为特征。

**6.3 路线 A：微调大模型做分类**

- 思路：取一个预训练大模型（用 4bit 量化加载以省显存），在其上做 **LoRA 微调**，套上序列分类头来输出类别。
- **LoRA（低秩适配）原理**：不动原始权重，而是在权重矩阵旁挂上两个小的低秩矩阵（外加缩放），微调时**只训练这些新增的小矩阵**。好处是可训练参数极少（可低至总参数的千分之几），大幅节省算力、降低过拟合、便于迁移。
- 工程要点：分层抽样的 K 折交叉验证；小 batch + 梯度累积换取等效大 batch、cosine 学习率、warmup、按 step 评估并保留验证集最优权重。这套现已有成熟框架封装，实际代码量并不大。

**6.4 路线 B：传统机器学习 + 重特征工程**

1. **文本纠错**：若发现部分样本存在系统性字符错乱（如某来源里某个字符被整体替换），可用**编辑距离（Levenshtein distance）**做纠错：对「疑似拼写错」的词列出所有距离为 1 的候选；统计全文最频繁的那种字符替换，当其占比超过阈值时，以「该高频修改成本最低」的自定义代价规则对全文批量纠错。实践中纠错常能带来分数提升。
2. **分词**：英文场景可用字节级 BPE（Byte-Level BPE）训练分词器；中文场景换用成熟中文分词库或业务自研词库。
3. **向量化**：用 **TF-IDF**（取 3~5 元词组）把文本转成向量。
4. **模型融合**：训练多个互补的分类器并软投票（soft voting）集成，常见组合：多项式朴素贝叶斯（适合文本计数特征）、SGD 线性分类器、LightGBM（梯度提升决策树）、CatBoost（擅长类别特征、抗过拟合）。

**6.5 关于模型融合的一般认识**

融合的价值在于：不同模型的偏差 / 方差不同、擅长的特征空间不同，组合后能平衡偏差方差、提升鲁棒性、降低对单一模型的过拟合依赖，通常比任何单模型更准。这一思想不限于文本分类，任何分类任务后期提分都可考虑。

### 七、落地与延伸

整个流程的主线是固定的：**数据清洗 → 特征工程 → 模型训练 → 评估调优 →（可选）融合 → 线上应用**。真实业务远比练手复杂：数据量级大需配套 ETL；上线还要考虑模型部署、特征实时计算、监控与迭代。但核心思想不变，先把这条链路完整走通，再逐环节深挖。掌握分类后很多任务可同构处理，例如**情感分析**本质就是把标签分成「正面 / 负面」的文本分类，落地时可把一批评论逐条预测情感倾向，再按时间线或事件维度做整体复盘。

### 常见坑 / 边界

- **不要无脑堆特征**：维度多 ≠ 效果好，无关或高度重复的特征会引入噪声、放大过拟合、增加训练成本。
- **准确率会骗人**：样本不均衡时务必结合精确率、召回率、F1（或 AUC）一起看，并选定与业务匹配的主指标。
- **缺失值填补会引入偏差**：简单均值 / 众数填补方便但可能掩盖结构，重要变量尽量用先验知识更精细地补。
- **特征相关性陷阱**：高度相关的特征信息冗余，往往是过拟合来源，应择一或重构。
- **验证集不能省**：没有可评估的标签数据，调优就是盲调；务必切出验证集或用交叉验证。
- **数据泄漏要看规则**：某些平台「会用真实测试集重跑」的机制下，「在测试集上构建词表 / 重新 embedding」可能是合规优化；但换平台或换评测机制时，这种做法可能构成数据泄漏，务必先确认规则。
- **分布漂移**：训练集（多为自造 / 拼凑数据）与真实测试集分布常有差距，本地交叉验证分数高不代表线上分数稳，要预留对分布差异的鲁棒性。
- **算力与约束**：若设有效率奖或推理时限，要在精度与模型大小 / 推理速度之间权衡，未必越大越好。
- 文中所有具体数值、库名、参数、模型代号均为示例情况，迁移到真实任务时需重新验证。
