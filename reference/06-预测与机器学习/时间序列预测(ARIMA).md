## 元信息
- **适用范围**：单变量、等间隔的业务时间序列短期外推，典型如日订单量、日成交量、日活 / 功能日使用量、周度销量等"主要靠自身历史规律延续"的指标。适合趋势 / 自相关较明显、短期内结构不突变的序列，也适合作为初学者理解时序建模逻辑、或中小团队尚无成熟时序方案时的基础工具。不适用：受外部强冲击主导、长期跨结构突变的预测（应转向带解释变量的模型或机器学习）；有明显周 / 月 / 年周期时应改用带季节项的 SARIMA。金融价格尤其要谨慎——历史拟合好不代表能赚钱。
- **时效**：整理日 2026-06-22；内容时效 长期有效（ARIMA / ARIMAX 为经典方法，statsmodels API 偶有小幅变化，以实际版本为准）。
- **可信度**：高 —— 原理与流程为统计学标准做法（Box-Jenkins 建模法），每一步都有可复算的检验指标支撑；业务落地建议来自实践经验，逻辑自洽但具体取舍需结合自身业务校准。"金融价格可被 ARIMA 有效预测"这一结论可信度低，仅作教学演示。
- **方法论标签**：#时间序列 #ARIMA #ARIMAX #平稳性检验 #差分 #ACF_PACF定阶 #外生变量 #目标预估

## 正文

> 本文同时讲三件事：①ARIMA 三个部件分别在解决什么问题、数学直觉是什么；②为什么必须先懂业务再建模、平稳性检验与 ACF/PACF 怎么帮你定阶、怎么拟合与体检；③纯模型结果在真实工作中往往不能直接用，怎么靠外生变量（ARIMAX）和目标预估场景把它盘活。

### 一句话方法论

ARIMA(p, d, q) 把一条序列拆成三块来描述它自己的"惯性"：**AR 自回归**用过去 p 期的取值，**I 差分**做 d 次差分把序列拉平稳，**MA 移动平均**用过去 q 期的误差项。整套建模就是 Box-Jenkins 流程——**先看图判趋势 → 平稳性检验 → 差分定 d → ACF/PACF 定 p、q → 拟合并体检 → 外推预测**。用业务话讲：拿历史数据预测未来（自回归）；若有周期波动（周末一定比工作日少）或随体量增长的长期趋势，就需要一套公式把它们刻画出来。

### 三个部件的数学直觉

**AR(p)——自回归**：当前时刻的值是过去 p 个时刻值的线性组合。

$$Y_t = \varphi_1 Y_{t-1} + \varphi_2 Y_{t-2} + \cdots + \varphi_p Y_{t-p} + \varepsilon_t$$

其中 $\varphi_1, \dots, \varphi_p$ 是自回归系数，$\varepsilon_t$ 是误差项（白噪声）。

**I(d)——差分**：一阶差分就是当前值减去前一时刻的值，$Y_t' = Y_t - Y_{t-1}$。阶数 d 表示需要差分几次才能让序列平稳，**差分几次 d 就取几**。

**MA(q)——移动平均**：当前时刻的值是过去 q 个误差项的线性组合。

$$Y_t = \varepsilon_t + \theta_1 \varepsilon_{t-1} + \theta_2 \varepsilon_{t-2} + \cdots + \theta_q \varepsilon_{t-q}$$

其中 $\theta_1, \dots, \theta_q$ 是移动平均系数。

三块拼起来再叠加差分，就得到 ARIMA 的整体表达：用 p 个过去值 + q 个过去误差项来预测当前值，并对原序列做 d 次差分。

- **p**：自回归阶数，用多少个过去值预测当前值。
- **d**：差分阶数，需要几次差分让序列平稳。
- **q**：移动平均阶数，用多少个过去的误差项。
- $\varphi$ 和 $\theta$ 都是由模型拟合出来的回归参数。

### 适用前提

- 序列必须**平稳**后才能进 ARIMA（均值、方差、自相关结构不随时间变化）。这是整个方法的硬前提，差分就是为了把非平稳序列拉成平稳。
- 数据需**等时间间隔**、缺口尽量补齐，并把时间列设为索引。
- 只用序列自身历史，不引入其它变量。若效果差，往往是因为序列本身不是"自我延续"型。

### 建模第一步：先懂数据和业务，别上来就敲代码

实际遇到的时序数据通常就是"每天的某个量"——日订单量、日用户量、某功能的日使用量等。**建模前先做描述性分析，叠加业务认知，把这条曲线的波动规律先讲清楚。** 以订单类指标为例，常见规律有：

- 周末订单比工作日低——工作日有大量规律性的通勤需求，周末没有。
- 今年整体比去年高——可能是供给（如投放量）增加，而人均频次基本没变。
- 呈季节性——极端天气（严冬酷暑）订单少，气候适宜的春秋订单多。
- 有寒暑假特征——学生群体的使用占一定比例。

**为什么非要把业务含义先解出来？** 两个原因：

1. **决定建模的调整方向，且这个方向得能被业务解释。** 比如做移动平均，到底取 7 天还是 3 天？可能 6 天和 7 天数值上差不多，但 7 天更贴合"一周"的业务周期，那就选 7。
2. **决定结果能不能被业务逻辑接受。** 比如某些地区可能季节性影响小、降雨影响占比大——这种情况就该考虑引入外生变量（见下文 ARIMAX）。

一句话：**上来就建模是图省事的做法，先把数据和业务背景吃透才是正路。**

### 建模第二步：平稳性检验与差分，确定 d

**先画时序图肉眼判趋势。** 如果能明显看到上升 / 下降趋势或波动中枢漂移，基本可判定非平稳，后面要靠差分处理。

```python
import pandas as pd
import matplotlib.pyplot as plt

df = pd.read_csv("series.csv")
df["date"] = pd.to_datetime(df["date"])
df.set_index("date", inplace=True)
y = df["value"]                 # 目标序列

y.plot(); plt.xlabel("日期"); plt.ylabel("数值"); plt.show()
```

**再用 ADF（Augmented Dickey-Fuller）单位根检验定量判断。** 原假设是"存在单位根（非平稳）"：

- **p 值 > 0.05**：不能拒绝原假设 → 序列非平稳 → 需要差分。
- **p 值 < 0.05**：拒绝原假设 → 序列平稳。
- 也可对比 ADF 统计量与 1% / 5% / 10% 临界值，统计量越负越平稳。

```python
from statsmodels.tsa.stattools import adfuller
result = adfuller(y)
adf_stat, p_value, crit = result[0], result[1], result[4]
```

若非平稳，做一阶差分后再次 ADF 检验。一般做一次差分（d=1）就足以让常见序列变平稳；若还不平稳，再差分一次（d=2）。注意差分会在首行产生缺失值，要 `dropna()`。

```python
y_diff = y.diff().dropna()
adfuller(y_diff)               # 再次检验，确认 p 值 < 0.05
```

### 建模第三步：用 ACF / PACF 定阶（选 p、q）

定阶主要看两张图：自相关函数 ACF 和偏自相关函数 PACF。**对差分后的平稳序列作图。**

**自相关函数（ACF）** 衡量当前值与前几期值的线性相关性，揭示滞后效应的强度。

$$\rho(k) = \frac{\sum_{t=k+1}^{n}(Y_t - \bar{Y})(Y_{t-k} - \bar{Y})}{\sum_{t=1}^{n}(Y_t - \bar{Y})^2}$$

其中 $\rho(k)$ 是滞后 k 的自相关系数，$\bar{Y}$ 是序列均值，n 是序列长度，k 是滞后期。

形象理解：把 $\bar{Y}$ 当基准，$Y_t$ 和 $Y_{t-k}$ 减均值就是各自"离基准的距离"。两者距离一样（即 $Y_t = Y_{t-k}$）时相关性不衰减，等于自己和自己的历史完全相关；随着滞后拉长，用越久远的数据预测当前越困难，自己和更遥远的自己越不相关——这就是"自相关"。本质上，ACF 就是协方差的标准化版本：把协方差除以方差，把结果压到 [-1, 1] 区间，从而不受原始单位和量级的影响。

ACF 的读图要点：

- 若多个滞后期上都显著不为零、且**缓慢衰减**，往往说明序列**非平稳**，需要差分；差分后再求一次 ACF。缓慢衰减意味着序列里有长期依赖或趋势。
- 若 ACF 在某个滞后期后**迅速衰减为零（截尾）**，说明自相关较短，这个截尾阶数可用来定**移动平均阶数 q**。

**偏自相关函数（PACF）** 衡量的是"剔除掉所有更低阶滞后期的影响后"，当前值与某个滞后值的**纯粹**相关性。

$$\varphi_{kk} = \frac{\mathrm{Cov}(Y_t, Y_{t-k} \mid Y_{t-1}, \dots, Y_{t-(k-1)})}{\mathrm{Var}(Y_t \mid Y_{t-1}, \dots, Y_{t-(k-1)})}$$

可以这样理解：分子是 $Y_t$ 与 $Y_{t-k}$ 在控制住中间各阶之后的协方差，分母是条件方差；本质和皮尔逊相关系数 $\rho(X,Y) = \mathrm{Cov}(X,Y)/(\sigma_X \sigma_Y)$ 同源——都是协方差除以标准差。直观上，相当于生成了一组"剔除了低阶滞后影响"的序列，再看它们之间的相关性；**某两阶之间相关性突然消失的地方，就是自回归阶数。**

PACF 的读图要点：

- k=1 时 PACF 等同于 ACF；k>1 时已剔除所有低阶滞后影响。
- PACF 反映某一滞后期对当前值的**直接**关系，不掺其他滞后期。
- 若 PACF 在某阶后**迅速截尾**（衰减到接近零且后续不再显著），这个阶数就是**自回归阶数 p（AR 部分）**；PACF 中的显著峰值帮你定 p。

```python
from statsmodels.graphics.tsaplots import plot_acf, plot_pacf
plot_acf(y_diff, lags=20)
plot_pacf(y_diff, lags=20)
```

> 定阶速记：**PACF 截尾定 p，ACF 截尾定 q，ACF 缓慢不衰减 → 先差分定 d。** 实操中常取"第一个显著超出置信带的峰值"对应的阶数；若图形不清晰，可直接试小阶数（如 p、q 都取 1）作为起点，剩下交给计算机拟合。

### 建模第四步：用 statsmodels 拟合与体检

参数 (p, d, q) 定好后用 `statsmodels` 拟合。**注意把原始未差分序列传给 ARIMA，让 order 里的 d 去做差分**，不要自己先差分再传。

```python
import numpy as np
from statsmodels.tsa.arima.model import ARIMA

# 造一段带季节波动的模拟日订单序列（中性示例数据）
np.random.seed(42)
dates = pd.date_range(start="2023-01-01", periods=180, freq="D")
orders = 50 + 10 * np.sin(2 * np.pi * dates.dayofyear / 365) + np.random.normal(size=len(dates))
data = pd.DataFrame({"Date": dates, "Orders": orders}).set_index("Date")

p, d, q = 2, 1, 2                      # 由 ACF/PACF 定阶得到
model = ARIMA(data["Orders"], order=(p, d, q))
fit = model.fit()
print(fit.summary())

forecast = fit.forecast(steps=10)      # 外推未来 10 天
```

**读 summary 的重点是系数符号、显著性和残差。** 例如拟合出 ar.L1=-1.03、ar.L2=-0.24、ma.L1=0.26、ma.L2=-0.51 这样一组系数：

- ar.L1 为较强负值，说明当前值与滞后一期呈较强负相关；
- ar.L2 为较小负值，说明滞后两期影响较弱；
- ma.L1 正值、ma.L2 较大负值，分别表示一阶、二阶误差项对当前值的正 / 负向影响。

summary 里还要看几个体检指标：

- **系数 p 值（P>|z|）**：若某项（如 ar.L1、ma.L1）p 值远大于 0.05，说明该项不显著，可考虑去掉或调阶。
- **AIC / BIC**：用于在多个候选模型间比较，**越小越好**。可网格搜索多组 (p, q) 选 AIC 最低的。
- **Ljung-Box 检验（残差白噪声）**：p 值大（如 > 0.05）说明残差近似白噪声，信息已被模型吸收干净，是好现象。
- **Jarque-Bera（残差正态性）**：p 值小说明残差不服从正态，预测区间的可信度要打折。

拟合后**务必检查残差**：残差应近似白噪声（无显著自相关）。可用 `plot_acf` / `plot_pacf` 对 `fit.resid` 作图，若残差仍有显著自相关，说明模型拟合不足，要回头重调 p、d、q。

```python
res = fit.resid
plot_acf(res)
plot_pacf(res)
```

做到这一步，相当于完成了一次"课程大作业 / 竞赛建模"级别的工作。

### 建模第五步：外推预测并可视化

用拟合好的模型向前预测若干步，再把预测值接到历史序列后面对比。

```python
steps = 7
forecast = fit.forecast(steps=steps)
idx = pd.date_range(start=data.index[-1], periods=steps + 1, freq="D")[1:]
pred = pd.DataFrame({"predicted": forecast.values}, index=idx)

plt.plot(data.index, data["Orders"], label="历史值")
plt.plot(pred.index, pred["predicted"], "rx-", label="预测值")
plt.legend(); plt.show()
```

**一个常见的教学现象：** 设某指标过去一年呈先升后缓降的走势，当前在 24 上下波动。ADF 原始检验 p≈0.28（非平稳），一阶差分后 p≈2e-30（平稳），故 d=1。拟合 ARIMA(1,1,1) 后向前预测 7 天，结果几乎是一条贴着最后观测值的近水平线（如 24.40、24.38、24.38… 逐步收敛到约 24.37），而 AR、MA 项各自的 p 值偏大、不显著。这反映了 ARIMA 的一个本质特征：**当 AR/MA 项不显著、序列接近随机游走时，最优短期预测就是"延续当前水平"**，预测曲线会很快拉平——模型勉强能跑但解释力弱，不要因为"图看着拟合上了"就高估它。

### 关键认知：纯 ARIMA 结果在真实工作中常常不能直接用

两个现实问题：

1. **纯自回归模型缺乏业务指标**，对业务方而言往往只是"看一眼"的结果，难落到行动。
2. **预测出的订单数本身难直接用**，因为缺少"如何达到这个预测"的策略路径。

应对这两点，有两个实用思路。

#### 思路一：给模型加外生变量（ARIMAX），量化业务因素的影响

ARIMAX 是 ARIMA 的扩展，引入被认为会影响目标变量 $Y_t$ 的外部变量 $X_t$：

$$Y_t = \varphi_1 Y_{t-1} + \cdots + \varphi_p Y_{t-p} + \theta_1 \varepsilon_{t-1} + \cdots + \theta_q \varepsilon_{t-q} + \beta_1 X_t + \beta_2 X_{t-1} + \cdots + \varepsilon_t$$

直观说，就是在原模型基础上**多挂了一条 X 序列**；外生变量这部分的解读，和普通线性回归的系数解读是一样的。X 可以是天气（如降雨量），也可以是各种经济 / 运营指标（如广告支出）。代码上只需在 `ARIMA(...)` 里多传一个 `exog`：

```python
rainfall = np.random.randint(0, 20, size=len(dates))     # 外生变量：模拟降雨量
data["Rainfall"] = rainfall

model = ARIMA(data["Orders"], order=(2, 1, 2), exog=data[["Rainfall"]])
fit = model.fit()

# 预测未来 10 天，需同时给出未来 10 天的外生变量取值（这里用假设值）
future_rainfall = np.random.randint(0, 20, size=10)
forecast = fit.forecast(steps=10, exog=future_rainfall)
```

> ⚠️ 外生变量的数据最好也经过处理（差分或移动平均）。若外生变量波动大、异常值多，容易出现"变量和结果对不上"的情况。典型坑：某地某天暴雨，但该地本就几乎没有供给 / 没有订单，于是出现"有降雨量却没有订单"的脏样本——这种点需要剔除，否则会污染系数。

#### 思路二：把预测结果当成目标制定的"参照之一"，而非唯一答案

把 ARIMA 的预测仅作为目标预估的多个参考之一，是完全可行的。原因在于：目标制定中本来就大量使用同比、环比，也会用移动平均和季节性趋势来预估本季度订单。这些做法通俗易懂，但未必周全。**当建模的参数和结果是干净、可信的时候，可以拿模型结果和自己（业务）预估的结果做对比（甚至对比到月度）**，借此判断到底是模型过拟合了，还是你的人工预估漏掉了某些因素。

### 常见坑与边界

- **不要把差分后的序列再喂给 ARIMA 并同时设 d**，会重复差分。原序列 + 正确的 d 即可。
- **预测曲线很快变平**不是 bug，而是序列近随机游走时的理论结果；这种情况下 ARIMA 几乎没有超额预测价值。
- **AIC 低 ≠ 预测准**。AIC/BIC 只在同一数据上比较模型复杂度与拟合的权衡，真正要看样本外（留出最后一段做回测）的误差。
- **金融价格慎用**：有效市场下价格接近随机游走，ARIMA 对它的"预测"主要是惯性外推，回测看似贴合不代表能指导交易，把它当作学习时序流程的练手案例即可。
- **定阶有主观性**：ACF/PACF 图形模糊时，用网格搜索按 AIC/BIC 自动选阶（如 pmdarima 的 auto_arima）比硬看图更稳。
- **有明显季节性（周 / 月 / 年周期）时应换 SARIMA**（带季节项），普通 ARIMA 抓不住周期。
- ARIMA 是一种**能落地但适用场景有限**的基础模型，价值在于建模过程中你对数据和背后业务本质的理解加深。**模型结果不能全信，要辩证地使用和解读。** 最终给业务交付时，要把这一串问题讲明白才算可信：数据为什么这样波动、未来预估大概多少、为什么和现有预估有差、背后是哪些业务数据的变化、由此给出什么业务指导。
