## 正文

本篇把"数据分析的能力全景"与"从 0 到上手的自学路线"合在一起：先用一张可自查的能力地图说明"要会什么、用来干嘛"，再给出按阶段推进的学习次序，说明每个阶段该掌握哪些主题、练哪些技能。能力地图每个条目只给概览，深入实操另查专题。

### 总览：两条能力线 + 一条推进次序

数据分析师的能力可分成两条线：**业务思维与分析方法**，以及**技术工具与数学模型**。自学的核心判断是——先把业务/分析思维这条线打通（订单为什么涨跌、活动效果怎么衡量、ROI 怎么算、竞品怎么看、下一步运营怎么定），再去补统计基础和工具，避免一上来就陷进语法细节。读书与上课的目的是入门和搭框架，更细的问题留给搜索与实操。

落到可执行的推进次序，大致是：

> **工具起步（SQL / Excel）→ 掌握对比、漏斗、留存等基础分析方法 → 理解假设检验与 AB 测试 → 套用 RFM / AARRR 等业务框架 → 补数仓、埋点等辅助知识 → 系统学统计学这门"内功"。**

下面先铺开能力地图（第一至五部分），再给出与之对应的分阶段自学路线（第六部分）。

## 第一部分　能力地图

把入门知识拆成四块来记：**工具与语言、分析思维与方法、数学模型、辅助知识与求职认知**，外加一张"日常问题地图"用于对号入座。

### 一、工具与语言

三件套覆盖绝大多数日常工作：SQL 取数、Python 做自动化与建模、Excel 做轻量分析与汇报图表。

**SQL（取数主力）**
- 对分析师而言核心是查询。打牢 `select / from / where / group by / having / join` 就能满足约八成查询需求。
- 进阶必会窗口函数：`over(partition by ... order by ...)` 可搭配聚合使用。区别于 `group by` 把所有字段压成一行，窗口函数只对开窗范围内的选定字段做聚合并保留明细行；若对所有指标用完全相同的 `partition by`，效果就退化成不去重的 `group by`。
- 常用聚合：`count / sum / avg / max / percentile_approx`（分位数）、`collect_list / collect_set`（行列归并）。
- 时间函数是高频考点：`current_date / to_date / from_unixtime / date_format / unix_timestamp`，以及 `year/month/day`、`datediff`（日期差）、`date_sub`（向前取 n 天）、`months_between`、`last_day`、`weekofyear` 等。
- 调优常识：给查询列建索引、减少运算量、避免在索引列上直接套函数（否则优化器走全表扫描）、能一句 SQL 写完就别拆多句、用 `where` 替代 `having`（`where` 在聚合前过滤，`having` 在出全部结果后才过滤）、表都起别名并加前缀。
- 典型题型：求"近一个月最大连续登录天数"（用两次 lead 函数把登录序列变成 0/1 串再处理）、求每辆共享车平均出入库时长、求次日/7 日/30 日留存、求当月新增/留存/回流（当月登录且上月登录过=留存，有新增标识=新增，其余=回流，可用 case when 判定）、累计付费（`sum() over()`）、剔除部门最高最低后的平均工资（`rank() over()`）等。

**Python（自动化与建模）**
- 语言基础：输入输出、数据类型（int/float/bool/complex、字符串、列表、元组、字典）、自定义函数、循环。理解"函数=可复用代码块""循环=反复执行同一段逻辑"即可起步。
- 数据分析三大库：NumPy/Pandas 做数组与表格运算（创建、append、diff 求差、isin 判断、delete 删除）；Matplotlib 做可视化。
- 分组聚合心智模型：`df['x'].groupby(df['key'])` 得到的是一个"分组器对象"，再接聚合函数才出结果，例如 `groupby([key1,key2]).mean()`。
- 时间序列处理 = 时间函数 + 时间格式数据，主要靠 `datetime / time / calendar`；`datetime` 以毫秒存储，两个时间相减得到时间差；字符串与时间可用 `strptime` / `dateutil.parser.parse` 互转。

**Excel（轻量分析与汇报）**
- 高频函数：`count / sum / sumif / average / max / vlookup / sumproduct`（两列先乘再求和）。`vlookup` 和 `sumif` 是日常最常用的。
- 数据透视表用于快速汇总。Excel 自带的"数据分析"加载项可做描述统计、相关系数与协方差、线性回归、移动平均。
- 可视化原则：主题色不超过三种；按目的选图——折线看趋势、柱状/直方看量级、饼图看占比、漏斗看转化、组合图看多维。作图流程是"收集合适数据→选对图表→删冗余信息突出重点→适当加辅助线塞入有效信息"。

**可视化分工**：Python 适合自助分析和组内分析；Excel 偏商业图表，用于汇报与业务沟通；Tableau/PowerBI 等 BI 工具满足报表与数据智能需求，本质都是做图表。

### 二、分析思维与方法

这是分析师区别于"取数工具人"的核心，可分成两类：**基础分析方法**和**业务分析框架**。

**基础分析方法**
- **对比分析**：把两个相关指标比较，看规模、水平、速度、协调关系。前提是确保两个指标"可比"。维度上分绝对数/相对数对比；相对数又细分结构相对（部分占整体，如付费用户占比）、比例相对（同一总体不同部分，如性别比）、比较相对（不同地区同性质指标）、强度相对（两个性质不同但相关的总量比）、动态相对（增长率、发展速度）。常见坑：时间窗口不一致（应同比口径，周三比周三、本月比去年同月）、空间差异、样本结构不同（女性多 vs 男性多的付费率不可直接比）、被产品活动策略干扰。
- **漏斗分析**：把用户行为串成链路，每步都有流失，转化曲线形似漏斗。重点看各环节跳出率和原因，用于产品流程优化。
- **帕累托分析（二八法则）**：按指标累计百分比找业务主次。做法是选指标→升序排列→求累计占比→画帕累托图找拐点（如前 10% 用户贡献多少付费）。用于资源配置判断和产品形态研究。
- **关联分析**：发现项集间关系（经典案例是购物篮分析），本质服务于归因——搞清"什么动作能带动目标指标"。简单情形用相关系数即可，工程化可上决策树、马尔可夫链等。
- **留存分析**：核心公式如"次日留存率=次日仍活跃的新用户/当日新增用户"，延伸出 7 日、30 日留存。目标是看清留存趋势与流失原因，服务总用户规模。做法是定口径→分析流失前后行为找突出原因→做业务动作→再用模型提升干预效率。

**业务分析框架**（多为成熟模型，重在套用与汇报）
- **波士顿矩阵（BCG）**：以"市场引力 × 企业实力"两维做四象限分类，多用于竞品分析和汇报，本质是一种分类。
- **杜邦分析**：用财务比率分解净资产收益率（ROE=销售净利率 × 资产周转率 × 权益乘数），评价盈利能力。互联网场景可把"利润=收入-活动成本""资产类比人效"做改造。
- **RFM**：用最近一次消费（R）、消费频率（F）、消费金额（M）给用户分层，常分八类施以不同策略。实战要点是动态划分档次（如取近 30 天某指标前 30% 为高等级）。
- **人货场**：零售分析三模块。互联网里"场"对应 App 本身及各资源位。分析人（来源、属性、消费力、行为）、货（供应商、质量、库存、消耗规律）、场（位置、流量、转化、适配人群），落地于监控预警和供需平衡的资源配置。
- **用户画像**：把用户属性、行为、消费、成本、风控等指标工程化集成，用作虚拟用户代表。核心是有一套灵活的画像工具而非单个画像本身。落地于精细化运营、管理汇报、对外宣传。
- **生命周期 / LTV**：算单用户/单产品全周期收益与成本，投资回报率=用户终身价值/获客成本。最简单可用 RFM 回归预测 LTV，进阶引入更多指标或机器学习，目的是据此重新调配成本与产品方向。
- **AARRR**：获取（Acquisition）、激活（Activation）、留存（Retention）、收益（Revenue）、传播（Referral）。互联网早期增长主线，现在更多结合精细化模型，逐环节找问题。

### 三、数学模型

- **假设检验**：原理是"先假设事情已发生，用统计分布算其发生概率"。步骤：提出原假设 H0（差异由抽样误差引起）与备择假设 H1（存在本质差异）→ 设检验水准 α（常取 0.05 或 0.01，即把 5% 当小概率）→ 选 Z/T/卡方等统计量计算。注意事项：先看资料是否可比（量纲）、统计显著不等于实际有意义（有相关未必有因果）、按资料类型选方法、按经验定单双侧、结论别绝对化（接受或拒绝都可能犯错，P 值并非万能但暂无更好替代）。
- **AB 测试（假设检验的典型应用）**：目的是验证产品改动是否提升目标指标（转化率、ARPU 等）。流程：定目标指标→按目标指标和第一类错误率算所需样本（转化率越低、要求错误率越低则样本越多）→技术上均匀随机分流（避免辛普森悖论）→实验后复盘。常见问题：样本不均匀、样本量不足、未正交分流且无校验机制导致结果不可信、受时间影响大时用时间片轮转法、只观察与测试功能交互的用户。高频考点：第一类与第二类错误的区别、公司更看重哪类错误、流量分层方案、实验组指标不显著怎么办。
- **时间序列**：把同一指标按时间排成数列，分析时间带来的规律并预测未来。四要素：长期趋势 T、季节变动 S、循环变动 C、不规则变动 I。模型有加法（Y=T+S+C+I）和更常用的乘法（Y=T·S·C·I）。工作场景一是找规律（用移动平均看周/季波动，订单下降先看时间差异），二是预测（序时平均、移动平均、趋势预测、指数平滑、季节性预测等，复杂时投喂机器学习）。
- **综合指数评价**：用规则模型把多个独立指标包装成一个集成指标（类似股票里的"上证 50"）。案例：评估创作者质量——先和业务沟通影响因素（发布次数、互动率等），定北极星指标设计公式（如互动率分层计分），再用专家意见或数据（相关性、多重共线性）定权重，最后用于汇报、画像，或作为机器学习目标函数。
- **归因分析**：确定各因素（营销活动、产品特征等）对结果的贡献度，常用于数字营销。方法由浅到深：① 最后一次点击（简单，但无法排除其他因素干扰）；② 线性模型（把最后一次行为加上前 30 天搜索/活动等都作自变量，建多元线性或逻辑回归，再用多重共线性、主成分等优化）；③ 时间衰减模型（在线性模型上加衰减函数，常用指数衰减 w(t)=exp(-λt)，另有线性、幂律、S 型衰减）。深入因果可参考 PSM-DID 等技巧。

### 四、辅助知识与求职认知

**辅助知识**
- 数据库原理：数据库做结构化存储与联机事务处理；SQL 的三级使用——分析师用于查询提取、数据工程师用于数仓建设与表优化、架构师用于库设计与建模。
- 埋点：初级是在关键流程植入统计代码追踪行为；中级多端追踪建用户模型；高级与研发/数据团队合作还原用户画像。埋点规范：尽量全量获取、明确功能需求、与前后端充分沟通文档、与测试一起验收。
- 数据仓库：通过特有存储架构做系统化整理，支撑 OLAP、数据挖掘、决策支持系统；构成包括数据库、元数据、数据仓库管理、访问工具。

**求职与岗位认知**
- 简历：按"教育背景→专业技能→工作经历→项目经历→其他"顺序写。技能写关键词和常用模型别赘述；工作经历每段写清公司/时间/岗位，分点列职能，重点项目放前面；项目经历突出思路和能改善效果的模型。用"什么背景—做了什么—有什么效果"结构化表述，例如"为解决某指标，通过某方法发现原因，给出策略，指标从 X 提升到 Y"。常见误区：在不重要的地方（用了什么 SQL 函数、做了数据预处理）大写特写，会让人觉得思考不深；转行者没突出分析思维；过于流水账、方法论提炼不足（可拿给外行看是否看得懂）。
- 面试流程：自我介绍（突出核心竞争力、别背稿）→SQL/Python 题（一般可口述，两三分钟说完）→简历挖掘（口语化补细节，会被追问质疑，是压力面）→案例分析→反问环节。案例分析常见三类：数据波动类（增长/下降探因，分组涨幅不同的原因）、策略类（指出某 App 改进点，常从业务特性和竞品对比切入）、指标评估类（如何用指标说明问题、制定北极星指标，需对常见指标熟悉且能结构化答题）。
- 面试特点：压力面会在你说成果时质疑，有漏洞就退一步说回去思考，没漏洞就坚持；挖细节会打断你问架构、同事等，既验真实性也考耐心思路。常见扣分项：背稿卡壳、夸大被深究、思考太久且表达不结构化。
- 找岗位：渠道有公司官网、招聘公众号、春秋招、内推、各类招聘 App；看招聘要求时逐条对应自身，多找相近岗位（策略分析师、策略运营等），限制越少要求越低。JD 常见要求与应对：会写 SQL 查询、能展开案例分析（有项目经验优先）、突出某专长、能用 Python 建模（最好有指标改善）、善于沟通推进（突出跨部门协作经历）、有互联网经历（通常是过简历的硬门槛）。

### 五、分析师日常问题地图

把日常工作归成三大类，便于对号入座：
- **异动归因类**：如订单收入波动——先用对比分析判断是否在正常水平（对历史、时间趋势），再做异动归因找出多出的部分由哪块业务贡献；目标达成差异——用预测结果归因加业务复盘，对比预测逻辑与实际值得出原因，再滚动计算下一轮目标。
- **业务分析类**：产品改动（埋点制作跟进验收、上线前搭监控报表、灰测期做 AB 复盘、全量后复盘）、产品方向（当前用户群体与目标是否一致、竞品分析）、运营工作（取数需求、活动效果复盘）。
- **管理汇报类**：月度经营汇总、关键问题及时汇报、成本效益分析（人效、外包人效、活动成本）。

## 第二部分　分阶段自学路线

能力地图是"目录式索引"——遇到具体任务时先在地图上定位它属于哪个模块（取数？某种分析方法？某个业务框架？），再查对应专题深入。本部分则按学习次序，说明从 0 到上手每个阶段该掌握的主题与该练的技能，对应资料可自行寻找相应经典教材或入门读物配合实操。

### 自学的两个心法
- **不要死记指标**。每个行业的指标都不同，但提出问题、推导结论的逻辑是相通的。学习案例时重点看：如何从问题出发、一步步得到结论，而不是背下某个具体指标值。
- **按自身阶段挑深度**。同一主题，初学者和在职者该投入的深度不同；应试只是这个行业的冰山一角，真正值钱的是"数据怎么来、怎么清洗、用什么架构、怎么建模"这条信息工业化链路。

### 阶段一：业务思维与基础入门（先打通这条线）
目标是建立"正常看数据"的逻辑和习惯，先于语法细节。该阶段应掌握：
- 用一款上手门槛低的工具（如自带免费版的可视化/BI 工具）边做边学，建立看数据、读图表的基本直觉；科班背景者可略过最基础部分。
- 通过一批经典分析案例，体会不同业务场景的分析套路与差异，把"从问题到结论"的链路走通。
- 如有明确目标行业，针对性了解该行业的指标体系与数据特征（不同行业关注的指标、数据分布差异很大，部分行业数据并不服从正态分布）。
- 建立"数据分析如何指导经营"的商业视角（老板视角），即便岗位与数据关系不大，也能提升沟通时的站位。
- 理解精益/低成本试错的思路：用小范围测试快速找方向，把数据分析当作"提升效率、降低试错成本"的手段，而非大公司专属。

寻材建议：这一阶段以入门读物和案例书为主，挑一两本读透即可，重在搭框架而非穷尽。

### 阶段二：进阶与在职提升
有了基础认知后，往"业务问题→数学模型"过渡，并把工具用熟。该阶段应掌握：
- 从业务问题过渡到数学模型的工程实践：用户行为建模、AB 实验设计、因果推断、生命周期等模型的原理与落地。
- 数据工业化全流程通识：把"数据从哪来→清洗→架构→分析→建模"这条链路看清，理解行业全貌而不只为应试。
- Python 数据分析能力，按自身定位选深度：偏运营者求广度、够用即可，边学边用；立志做专职分析师者求系统、敢于上代码量。
- 以真实小项目驱动练手（如概率计算、点击率/转化显著性评估、文本与新闻分析、招聘信息处理、社交网络分析等），覆盖概率论、假设检验、聚类、文本比较(NLP)、图分析、简单监督学习等方法，避免死记硬背。

### 阶段三：统计学基础（内功，建议系统学）
统计学是必学的内功，原理熟了在工作中反应更快、判断更准，这也是分析岗常优先要统计学背景的原因。它见效慢但回报长期，不要因为短期"应试用不上"就跳过。可按用途分三层系统学习，对应教材可找相应经典著作：
- **统计学导论层**：覆盖问卷调查、假设检验、时间趋势、简单回归等最基础的方法论，作为权威入门。
- **概率论与数理统计层**：更深入讲概率与各类分布，对假设检验等工具进一步说明；想打牢 AB 测试基础重点补这一层。
- **计量经济学层**：从经济学角度切入统计，尤其在时间趋势的回归与拟合上有用，常应用于公司里的简单数据回归与挖掘。

### 阶段四：工具与专题
配合前面三阶段，按需补齐具体工具与专题，对应资料找各自的经典入门书即可：
- **SQL/HiveSQL**：实际工作和招聘普遍要求，入门一本足够，其余靠实操和搜索补齐。
- **商务图表与可视化**：解决"如何做出专业、有效的商务图表"，配合 SQL 基本能满足分析师日常的可视化需求。
- **用户画像**：把画像的开发与应用了解清楚；分析师重点看应用部分，案例不足可在指标体系相关材料里补全。
- **数据库系统（选看）**：对找工作没有直接帮助，但能让你对数据库的建立、取数、清洗、提效等底层知识心里有数，和开发协作时更顺。

### 常见坑
- 别被资料数量吓到，按自己当前阶段选一两本读透即可，没必要一次性铺开。
- 工具类材料重在"入门搭框架"，不必逐字啃；遇到具体语法/报错直接查资料更高效。
- 统计学这类内功见效慢但回报长期，不要因为短期用不上就跳过。
- 先思维后技术：先把业务/分析思维打通，再补统计与工具，避免一上来陷进语法细节。
